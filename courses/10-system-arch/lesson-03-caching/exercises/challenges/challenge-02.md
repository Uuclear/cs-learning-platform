# 挑战 2：实现高性能缓存系统

## 难度
⭐⭐⭐⭐

## 背景
现代应用需要处理高并发请求，缓存系统必须能够高效处理大量读写操作。简单的单线程缓存可能成为性能瓶颈。

## 任务要求
实现一个支持高并发的缓存系统，包含以下特性：

1. **多级缓存**：L1（本地内存）+ L2（共享缓存）
2. **异步失效**：缓存过期时异步加载新数据，避免阻塞请求
3. **批量操作**：支持批量获取和设置多个键值对
4. **统计监控**：记录命中率、请求量、响应时间等指标

## 具体实现
创建以下组件：

### LocalCache (L1)
- 使用 `threading.local()` 或类似机制实现线程本地缓存
- 支持快速读取，容量较小（如100个条目）
- TTL较短（如1秒）

### SharedCache (L2)  
- 使用线程安全的数据结构（如 `queue.Queue` 或锁保护的字典）
- 容量较大（如10000个条目）
- TTL较长（如60秒）

### CacheManager
- 协调 L1 和 L2 缓存
- 实现 `get(key)`, `set(key, value)`, `mget(keys)`, `mset(items)` 方法
- 处理缓存穿透和雪崩问题

## 性能要求
- 单线程下每秒至少处理10000次读取操作
- 多线程下（4线程）每秒至少处理25000次读取操作
- 写入操作不应显著影响读取性能

## 测试方案
编写性能测试脚本：
1. 单线程基准测试
2. 多线程并发测试  
3. 缓存未命中场景测试
4. 批量操作性能测试

## 提示
- 使用 `time.perf_counter()` 进行精确计时
- 考虑使用连接池模式管理资源
- 对于异步加载，可以使用简单的后台线程或定时器
- 注意内存泄漏问题（及时清理过期数据）

## 评估标准
- 性能指标达标
- 正确处理并发安全
- 代码架构合理，易于扩展
- 包含完整的测试用例和性能报告
