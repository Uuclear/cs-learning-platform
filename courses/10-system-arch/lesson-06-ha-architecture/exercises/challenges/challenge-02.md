# 挑战2：设计高可用电商架构

## 任务描述

为一个中等规模的电商平台设计高可用架构方案。该平台需要支持：

- 用户注册/登录（日活用户约10万）
- 商品浏览和搜索（高峰期QPS约1000）
- 购物车和订单管理（高峰期TPS约100）
- 支付集成（与第三方支付网关对接）

## 要求

1. **绘制架构图**：使用文字描述或ASCII艺术表示系统架构
2. **详细说明**：
   - 各层（Web层、应用层、数据层、缓存层）的冗余策略
   - 故障转移机制和自动恢复策略
   - 数据一致性保证方案
   - 监控和告警策略

3. **关键指标定义**：
   - RTO和RPO目标
   - 可用性目标（SLA）
   - 性能指标（响应时间、吞吐量）

4. **风险分析**：
   - 识别潜在的单点故障
   - 分析级联故障风险
   - 提出缓解措施

## 约束条件

- 预算有限，需要在成本和可用性之间找到平衡
- 团队规模较小（5-10人），运维能力有限
- 必须支持未来业务增长（用户量翻倍）

## 提交格式

以Markdown格式提交，包含清晰的章节结构和详细的说明。
