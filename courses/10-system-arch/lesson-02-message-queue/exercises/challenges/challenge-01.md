# 挑战 01：实现简单的消息代理

## 难度
⭐⭐⭐

## 背景
你正在为一个电商系统设计消息传递机制。系统需要处理订单创建、库存更新和通知发送等操作。

## 要求
实现一个简单的消息代理类 `SimpleMessageBroker`，支持以下功能：

1. **主题管理**：能够创建和管理多个主题（topics）
2. **发布-订阅**：生产者可以向主题发布消息，所有订阅该主题的消费者都能收到消息
3. **消费者回调**：消费者通过回调函数接收消息
4. **线程安全**：确保在多线程环境下正常工作

## 接口定义
```python
class SimpleMessageBroker:
    def subscribe(self, topic: str, callback: Callable[[str], None]) -> None:
        """订阅指定主题"""
        pass
    
    def publish(self, topic: str, message: str) -> None:
        """向指定主题发布消息"""
        pass
```

## 测试场景
- 创建一个主题 "order-events"
- 订阅三个不同的消费者（订单服务、库存服务、通知服务）
- 发布5条订单消息
- 验证所有消费者都收到了所有消息

## 提示
- 使用 Python 标准库的 `queue.Queue` 和 `threading` 模块
- 考虑如何处理消费者处理消息时可能出现的异常
- 确保消息不会丢失（即使消费者处理失败）

## 评估标准
- 功能完整性（40%）
- 线程安全性（30%）
- 代码质量与注释（20%）
- 错误处理（10%）
