# 挑战2：实现离线优先的PWA应用

## 目标
创建一个能够在离线状态下正常工作的PWA应用，支持数据同步和用户通知。

## 要求
1. **核心PWA功能**：
   - 完整的Web Manifest配置
   - Service Worker实现资源缓存
   - 支持从主屏幕安装

2. **离线功能**：
   - 用户可以离线查看最近访问的内容
   - 离线时仍能提交表单（数据暂存）
   - 网络恢复后自动同步暂存数据

3. **用户体验**：
   - 显示当前网络状态（在线/离线）
   - 离线提交时给出明确反馈
   - 同步成功后通知用户

4. **高级功能**：
   - 实现后台同步（Background Sync）
   - 支持推送通知（可选）
   - 数据版本控制，避免冲突

## 技术栈
- Service Worker API
- Cache Storage API
- Background Sync API（如果浏览器支持）
- IndexedDB或localStorage用于本地存储

## 评估标准
- ✅ 应用可以成功安装为PWA
- ✅ 离线时核心功能仍可用
- ✅ 网络恢复后数据正确同步
- ✅ 用户体验流畅，有明确的状态指示

## 提示
- 使用`workbox`库简化Service Worker开发
- 考虑使用队列模式处理离线操作
- 实现适当的错误处理和重试机制
- 测试不同网络条件下的行为

## 扩展挑战（可选）
- 添加数据加密保护用户隐私
- 实现多设备间的数据同步
- 添加性能监控和分析
