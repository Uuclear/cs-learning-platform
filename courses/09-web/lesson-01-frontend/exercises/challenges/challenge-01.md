# 构建个人主页

**难度**: ⭐⭐

## 描述
创建一个完整的个人主页，展示你的技能、项目经验和联系方式。这个页面应该体现现代Web开发的最佳实践，包括语义化HTML、响应式CSS和交互式JavaScript。

## 要求

### HTML 结构要求
- 使用语义化HTML5标签（如 `<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, `<footer>` 等）
- 包含以下主要部分：
  - 个人介绍（头像、姓名、职位描述）
  - 技能展示（使用进度条或图表形式）
  - 项目作品集（至少3个项目，每个包含标题、描述、技术栈标签）
  - 联系表单（姓名、邮箱、消息字段）
- 确保页面结构清晰，便于屏幕阅读器访问

### CSS 样式要求
- 使用现代CSS特性（Flexbox 或 Grid 布局）
- 实现响应式设计，适配移动设备和桌面设备
- 包含悬停效果和过渡动画
- 使用一致的色彩方案和字体选择
- 避免使用内联样式，所有样式放在 `<style>` 标签或外部CSS文件中

### JavaScript 功能要求
- 表单验证（确保必填字段不为空，邮箱格式正确）
- 平滑滚动到页面不同部分
- 返回顶部按钮（滚动一定距离后显示）
- 本地存储表单数据（防止意外刷新丢失）
- 至少一个动态交互功能（如暗黑模式切换、动态计数器等）

### 额外加分项
- 使用CSS变量实现主题切换
- 添加加载动画效果
- 实现图片懒加载
- 添加SEO友好的meta标签
- 通过W3C HTML和CSS验证

## 提示
1. **先规划再编码**：在开始写代码前，先在纸上画出页面布局草图
2. **移动优先**：从移动设备布局开始设计，然后逐步增强到桌面版本
3. **组件化思维**：将页面拆分成独立的组件（如导航栏、卡片、表单等）
4. **测试驱动**：在不同设备和浏览器中测试你的页面
5. **性能优化**：注意图片大小、CSS选择器效率和JavaScript性能

## 评估标准
- ✅ 功能完整性（是否满足所有要求）
- ✅ 代码质量（语义化、可维护性、注释）
- ✅ 用户体验（交互流畅、视觉美观）
- ✅ 响应式设计（多设备适配）
- ✅ 性能表现（加载速度、动画流畅度）

**记住**：个人主页不仅是展示你技术能力的地方，也是展示你设计思维和用户体验理解的机会。用心打造，让它成为你的数字名片！
