# 挑战 2: 实现简化版的 React 状态管理库

⭐⭐⭐⭐ 难度 | 预计完成时间: 20分钟

## 背景
现代 React 应用经常使用状态管理库如 Redux、Zustand 或 Context API。理解这些库的核心原理对于优化应用性能至关重要。

## 任务
实现一个简化版的状态管理库，包含以下功能：
1. **集中式状态存储**：支持嵌套状态对象
2. **订阅机制**：组件可以订阅特定状态路径的变化
3. **批量更新**：支持一次更新多个状态，并只触发一次通知
4. **选择器支持**：允许组件只订阅状态的特定部分

## 要求
- 使用 Python 类实现 `Store` 类
- 支持 `getState(path=None)`、`setState(updates, path=None)`、`subscribe(callback, path=None)` 方法
- 实现路径订阅（如 `"user.profile.name"`）
- 包含完整的中文文档字符串和内联注释
- 提供使用示例，展示如何在复杂应用中使用

## 核心功能细节

### 路径订阅
- `subscribe(callback, "user")` - 订阅 user 对象的所有变化
- `subscribe(callback, "user.profile")` - 只订阅 profile 子对象
- `subscribe(callback)` - 订阅整个状态树

### 批量更新
- `setState({"name": "张三", "age": 25}, "user")` - 更新 user 对象的多个属性
- 所有订阅者应该只收到一次通知，而不是每个属性变化都通知

### 性能考虑
- 避免不必要的通知（只有实际变化才通知）
- 支持深度比较以检测真正的变化
- 内存效率：不要存储重复的状态副本

## 验证
你的实现应该能够处理：
1. 嵌套状态对象的更新和订阅
2. 复杂的路径解析（包括数组索引）
3. 批量更新时的正确通知机制
4. 内存泄漏防护（清理不再需要的订阅）
