# 挑战2：缓存策略设计

## 背景
你正在为一个新闻网站设计缓存策略。该网站的特点是：

- 首页内容每30分钟更新一次
- 文章页面内容相对稳定，但评论会实时更新
- 用户个人资料页面包含个性化内容
- 静态资源（CSS、JS、图片）使用版本化文件名

## 任务
为不同类型的资源设计最优的HTTP缓存策略：

### 步骤1：分析资源类型
将网站资源分为以下几类：
1. HTML页面（首页、文章页、个人页）
2. 静态资源（CSS、JS、图片）
3. API数据（文章列表、评论、用户数据）

### 步骤2：设计缓存头
为每种资源类型设计具体的Cache-Control头：

**示例格式：**
```http
# 静态资源
Cache-Control: public, max-age=31536000, immutable

# 首页HTML
Cache-Control: public, max-age=1800, stale-while-revalidate=600
```

### 步骤3：处理边缘情况
考虑以下场景并提供解决方案：
- 用户发布新评论后，如何确保其他用户能看到？
- 紧急内容更新时，如何快速使缓存失效？
- 移动网络下如何平衡缓存和新鲜度？

### 步骤4：实现验证
使用`example-02-cache-strategy.py`中的模拟器，测试你的缓存策略在不同场景下的命中率。

## 评估标准
- 缓存策略的合理性和技术准确性
- 对用户体验和服务器负载的平衡考虑
- 对实际业务需求的理解和应对

## 提示
参考HTTP RFC 7234规范和现代浏览器的缓存行为，确保策略的兼容性和有效性。
