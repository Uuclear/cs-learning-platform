# 挑战1：性能优化实战

## 背景
你正在开发一个电商网站，用户反映首页加载速度很慢。经过分析，你发现以下问题：

1. 首屏包含大量高分辨率图片（总计5MB）
2. 主JavaScript bundle大小为800KB，包含所有功能代码
3. CSS文件阻塞渲染，大小为200KB
4. 第三方分析脚本同步加载，延迟了页面交互

## 任务
使用你在本课程中学到的知识，提出具体的优化方案：

### 步骤1：图片优化
- 如何优化现有的5MB图片？
- 应该使用什么格式和策略？

### 步骤2：代码分割
- 如何将800KB的bundle进行合理分割？
- 哪些代码应该放在初始bundle中？

### 步骤3：关键资源优化
- 如何处理200KB的CSS文件？
- 第三方脚本应该如何加载？

### 步骤4：缓存策略
- 为不同类型的资源设计合适的缓存策略
- 如何处理版本更新时的缓存失效？

## 评估标准
- 方案的完整性和可行性
- 对Core Web Vitals指标的具体改善预期
- 是否考虑了实际开发中的约束条件

## 提示
参考课程中的`example-01-lighthouse-sim.py`和`example-03-bundle-analyzer.py`来量化你的优化效果。
