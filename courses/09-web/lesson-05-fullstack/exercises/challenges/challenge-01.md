# 挑战1：实现混合渲染策略 ⭐⭐⭐

## 背景
你正在开发一个新闻网站，其中包含以下页面类型：
- 首页（静态内容，每小时更新）
- 文章详情页（动态内容，需要实时评论数）
- 用户个人资料（完全个性化，需要用户登录状态）

## 任务
使用你学到的SSR、SSG和ISR知识，为这个新闻网站设计并实现一个混合渲染策略。

### 具体要求
1. **首页**：使用SSG + ISR，每小时重新生成一次
2. **文章详情页**：使用混合策略
   - 基础文章内容使用ISR（每30分钟更新）
   - 实时评论数使用客户端获取或SSR
3. **用户个人资料**：使用SSR，确保个性化内容

### 实现步骤
1. 创建模拟的数据获取函数（`fetchHomePageData()`, `fetchArticleData()`, `fetchUserData()`）
2. 实现三个不同的数据获取策略函数：
   - `getStaticPropsForHome()` - 带ISR的首页
   - `getHybridPropsForArticle()` - 混合策略的文章页
   - `getServerSidePropsForProfile()` - SSR的个人资料页
3. 创建一个路由匹配器，根据URL路径选择合适的渲染策略
4. 添加性能监控，记录每种策略的执行时间和缓存命中率

### 验证标准
- [ ] 首页在构建时生成，后续请求使用缓存，每小时重新验证
- [ ] 文章页的基础内容使用ISR，实时数据通过其他方式获取
- [ ] 个人资料页每次请求都重新渲染
- [ ] 性能监控显示合理的缓存命中率（>70%）
- [ ] 代码包含完整的中文注释
