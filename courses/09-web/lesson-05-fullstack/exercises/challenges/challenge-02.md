# 挑战2：构建完整的Next.js路由系统 ⭐⭐⭐⭐

## 背景
你需要为一个复杂的电商平台实现一个完整的文件系统路由系统，该系统需要支持：
- 嵌套路由（商品分类 -> 商品列表 -> 商品详情）
- 动态路由参数（用户ID、商品ID、分类ID）
- API路由（RESTful API端点）
- 布局系统（不同区域使用不同布局）
- 中间件（认证、日志、权限检查）
- 错误处理（404、500页面）

## 任务
实现一个高级的文件系统路由器，能够解析复杂的目录结构并正确匹配路由。

### 具体要求
1. **目录结构支持**：
   - `pages/index.js` → `/`
   - `pages/products/[category]/[id].js` → `/products/electronics/123`
   - `pages/api/users/[id].js` → `/api/users/123`
   - `pages/dashboard/layout.js` → 仪表板区域的布局组件

2. **路由匹配功能**：
   - 精确匹配静态路由
   - 支持单个动态参数 `[id]`
   - 支持捕获所有路由 `[...slug]`
   - API路由和页面路由分离处理

3. **高级功能**：
   - 自动查找并应用布局组件（向上查找最近的layout.js）
   - 支持中间件（middleware.js）的自动注册和执行
   - 错误页面的自动回退
   - 路由树的可视化输出

4. **性能要求**：
   - 路由匹配时间 < 1ms
   - 支持1000+路由的大型应用
   - 内存使用优化

### 实现步骤
1. 创建`AdvancedRouter`类，包含扫描、注册、匹配方法
2. 实现递归目录扫描，正确解析各种文件类型
3. 实现智能路由匹配算法，支持动态参数提取
4. 添加布局和中间件的自动发现机制
5. 实现路由树生成和可视化功能
6. 编写完整的测试用例验证所有功能

### 验证标准
- [ ] 正确处理所有路由类型（静态、动态、API、嵌套）
- [ ] 布局和中间件自动发现工作正常
- [ ] 路由匹配性能满足要求
- [ ] 错误处理和回退机制完善
- [ ] 代码结构清晰，有完整的类型注解和中文注释
- [ ] 包含详细的文档字符串和使用示例
