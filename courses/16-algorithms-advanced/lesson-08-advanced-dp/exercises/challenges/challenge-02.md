# 挑战2：动态规划优化实战

## 背景
你正在开发一个高性能的数据处理系统，需要优化一个关键的动态规划算法。原始算法的时间复杂度是 O(n²)，但在实际应用中 n 可能达到 10⁵，这会导致性能问题。

## 问题描述
给定数组 a[1...n] 和常数 C，计算：
```
dp[i] = min{ dp[j] + (sum(a[j+1...i]))² + C } for all j < i
```
其中 dp[0] = 0，sum(a[j+1...i]) 表示从 j+1 到 i 的元素和。

## 要求
1. 实现原始的 O(n²) 算法作为基准
2. 分析状态转移方程，判断是否可以使用斜率优化（Convex Hull Trick）
3. 如果可以，实现 O(n) 或 O(n log n) 的优化版本
4. 比较两种算法在不同输入规模下的性能差异
5. 编写测试用例验证优化算法的正确性

## 输入格式
- 数组 a: 长度为 n 的整数数组
- 常数 C: 整数

## 输出格式
- dp[n] 的值
- 算法执行时间对比

## 示例
输入：
- a = [1, 2, 3, 4, 5]
- C = 10

输出：
- dp[5] = 85
- 性能对比数据

## 提示
- 斜率优化适用于形如 dp[i] = min{ dp[j] + f(i) * g(j) + h(j) } 的转移方程
- 需要维护下凸包或上凸包
- 可以使用双端队列来维护候选决策点
- 注意检查优化条件是否满足（斜率单调性等）
