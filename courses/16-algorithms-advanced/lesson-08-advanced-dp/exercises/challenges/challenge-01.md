# 挑战1：最优二叉搜索树实现

## 背景
给定一组键值 k₁, k₂, ..., kₙ 及其搜索概率 p₁, p₂, ..., pₙ，以及虚拟键（不在树中的键）的搜索概率 q₀, q₁, ..., qₙ，构建一棵最优二叉搜索树，使得期望搜索代价最小。

## 要求
1. 实现 `optimal_bst(p, q)` 函数，返回最小期望搜索代价
2. 实现 `construct_optimal_bst(root, i, j, parent, is_left)` 函数，打印最优二叉搜索树的结构
3. 处理边界情况（空树、单节点等）
4. 时间复杂度应为 O(n³)，可选实现 Knuth 优化将时间复杂度降到 O(n²)

## 输入格式
- `p`: 长度为 n 的列表，p[i] 表示键 kᵢ₊₁ 的搜索概率
- `q`: 长度为 n+1 的列表，q[i] 表示虚拟键 dᵢ 的搜索概率

## 输出格式
- 最小期望搜索代价（浮点数）
- 树的结构（可以是括号表示法或层次遍历）

## 示例
输入：
- p = [0.15, 0.10, 0.05, 0.10]
- q = [0.05, 0.10, 0.05, 0.05, 0.05]

输出：
- 最小期望搜索代价: 2.75
- 树结构: k2(k1(d0, d1), k3(d2, k4(d3, d4)))

## 提示
- 使用动态规划，状态 e[i][j] 表示包含键 kᵢ 到 kⱼ 的子树的期望搜索代价
- 需要辅助数组 w[i][j] 存储概率和
- 根节点选择需要枚举所有可能的根
