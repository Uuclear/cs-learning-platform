# 挑战1: 多商品流问题

**难度**: ★★★★☆ (4/5)

## 问题描述

在标准的网络流问题中，我们只考虑单一类型的商品从源点流向汇点。但在实际应用中，我们经常需要处理多种不同类型的商品同时在网络中流动的情况。

现在给你一个网络，其中有k种不同的商品，每种商品都有自己的源点和汇点。每条边都有一个总容量限制，所有商品通过该边的流量之和不能超过这个容量。

你的任务是判断是否存在一种流量分配方案，使得所有k种商品都能满足其需求（即每种商品都能从其源点向其汇点发送指定数量的流量）。

## 输入输出规格

### 输入格式
- 第一行：n, m, k (顶点数, 边数, 商品种类数)
- 接下来m行：每行三个整数 u, v, c，表示从u到v有一条有向边，总容量为c
- 接下来k行：每行三个整数 s_i, t_i, d_i，表示第i种商品的源点、汇点和需求量

### 输出格式
- 如果存在可行解，输出"YES"
- 否则输出"NO"

### 约束条件
- 2 ≤ n ≤ 50
- 1 ≤ m ≤ 200  
- 1 ≤ k ≤ 10
- 0 ≤ u, v < n
- 1 ≤ c, d_i ≤ 1000

## 示例

### 输入
```
4 5 2
0 1 10
1 2 5
1 3 8
2 3 7
0 3 3
0 3 8
1 2 4
```

### 输出
```
YES
```

### 解释
- 商品1：从0到3，需求8。可以走路径 0->1->3 (流量5) + 0->3 (流量3)
- 商品2：从1到2，需求4。走路径 1->2 (流量4)
- 边(0,1)：流量5 ≤ 10 ✓
- 边(1,2)：流量4 ≤ 5 ✓  
- 边(1,3)：流量5 ≤ 8 ✓
- 边(2,3)：流量0 ≤ 7 ✓
- 边(0,3)：流量3 ≤ 3 ✓

## 提示

1. **建模思路**：这是一个多商品流问题，可以将其转化为单商品流问题来解决
2. **关键观察**：由于k较小（≤10），可以考虑枚举或分层建图的方法
3. **网络构建**：为每种商品创建独立的网络层，然后在对应边上添加容量约束
4. **算法选择**：使用标准的最大流算法，但需要巧妙地构建网络结构

## 参考解决方案思路

将原网络复制k次（每个商品一层），然后：
1. 在每层内部按照原网络结构连接边，容量设为无穷大（或足够大）
2. 对于原网络中的每条边(u,v)，在所有k层的对应位置(u,v)之间添加一个"容量约束边"，容量等于原边容量
3. 添加超级源点连接到各层商品的源点，超级汇点连接各层商品的汇点
4. 计算从超级源点到超级汇点的最大流，如果等于所有商品需求之和，则存在可行解

这种方法的时间复杂度主要取决于最大流算法的复杂度，但由于网络规模扩大了k倍，需要注意效率优化。
