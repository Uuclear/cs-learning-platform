# 挑战2: 带下界的网络流

**难度**: ★★★★★ (5/5)

## 问题描述

在标准网络流问题中，每条边只有上界（容量）约束。但在某些实际场景中，边可能还有下界约束，即通过该边的流量不能少于某个最小值。

给定一个带下界的流网络，其中每条边(u,v)都有下界l(u,v)和上界c(u,v)，要求找到一个可行流（不一定是最大流），使得所有边的流量满足 l(u,v) ≤ f(u,v) ≤ c(u,v)，并且满足流量守恒。

## 输入输出规格

### 输入格式
- 第一行：n, m (顶点数, 边数)
- 接下来m行：每行四个整数 u, v, l, c，表示从u到v有一条有向边，下界为l，上界为c
- 最后一行：s, t (源点和汇点)

### 输出格式
- 如果存在可行流，输出"YES"，然后输出每条边的实际流量
- 否则输出"NO"

### 约束条件
- 2 ≤ n ≤ 100
- 1 ≤ m ≤ 1000
- 0 ≤ u, v < n  
- 0 ≤ l ≤ c ≤ 10000

## 示例

### 输入
```
4 5
0 1 2 5
0 2 1 3
1 2 0 2
1 3 1 4
2 3 2 6
0 3
```

### 输出
```
YES
2
1
1
1
2
```

### 解释
- 边(0,1)：流量2，满足 2≤2≤5
- 边(0,2)：流量1，满足 1≤1≤3  
- 边(1,2)：流量1，满足 0≤1≤2
- 边(1,3)：流量1，满足 1≤1≤4
- 边(2,3)：流量2，满足 2≤2≤6
- 流量守恒：顶点1流入2，流出1+1=2 ✓；顶点2流入1+1=2，流出2 ✓

## 提示

1. **关键转换**：带下界的网络流可以通过添加超级源点和超级汇点转换为标准网络流问题
2. **供需平衡**：对于每个顶点，计算其"需求" = 流入下界之和 - 流出下界之和
3. **网络构建**：
   - 如果顶点需求为正，说明需要额外流入，从超级源点向该顶点连边
   - 如果顶点需求为负，说明需要额外流出，从该顶点向超级汇点连边
   - 原边的容量调整为 c(u,v) - l(u,v)
4. **可行性判断**：如果从超级源点出发的所有边都满流，则存在可行解

## 参考解决方案思路

1. 计算每个顶点的供需差额：D[v] = Σ(l(u,v)) - Σ(l(v,w))
2. 创建新网络：
   - 添加超级源点S'和超级汇点T'
   - 对于每条原边(u,v)，添加边(u,v)容量为c(u,v)-l(u,v)
   - 如果D[v] > 0，添加边(S', v)容量为D[v]
   - 如果D[v] < 0，添加边(v, T')容量为-D[v]
3. 计算从S'到T'的最大流
4. 如果最大流等于所有正D[v]之和，则存在可行解
5. 实际流量 = 下界 + 新网络中的流量

这种方法巧妙地将带约束的问题转换为标准的最大流问题，体现了网络流建模的强大能力。
