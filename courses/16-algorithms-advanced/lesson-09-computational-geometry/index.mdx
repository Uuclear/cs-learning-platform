## 2. 核心概念

### 叉积与方向判断（Cross Product and Orientation）

叉积是计算几何中最基础也最重要的工具。给定三个点 p, q, r，我们可以通过计算向量 pq 和 pr 的叉积来判断它们的相对位置关系：

- 叉积 > 0：r 在 pq 的左侧（逆时针方向）
- 叉积 < 0：r 在 pq 的右侧（顺时针方向）  
- 叉积 = 0：三点共线

叉积的计算公式：(q.x - p.x) * (r.y - p.y) - (q.y - p.y) * (r.x - p.x)

### 点在多边形内测试（Point-in-Polygon Test）

判断一个点是否在多边形内部有两种经典方法：

**射线法（Ray Casting）**：从该点向任意方向发射一条射线，统计与多边形边界的交点数量。奇数个交点表示在内部，偶数个表示在外部。

**绕数法（Winding Number）**：计算该点相对于多边形的绕数。绕数非零表示在内部，绕数为零表示在外部。

### 凸包算法（Convex Hull）

凸包是包含所有点的最小凸多边形。我们主要学习两种算法：

**Graham扫描法**：
1. 找到最下方的点（y坐标最小，x坐标最小）
2. 按极角排序其他点
3. 使用栈维护凸包，通过叉积判断是否需要回溯

时间复杂度：O(n log n)

**Jarvis步进法（Gift Wrapping）**：
1. 从最左点开始
2. 每次选择使得所有其他点都在当前边右侧的点
3. 重复直到回到起点

时间复杂度：O(nh)，其中h是凸包顶点数

### 最近点对（Closest Pair of Points）

在平面上找到距离最近的两个点。暴力解法是O(n²)，但我们可以用分治法优化到O(n log n)：

1. 按x坐标排序所有点
2. 递归求解左右两半的最近距离d
3. 考虑跨越中线的点对，只需要检查中线两侧距离小于d的点
4. 对这些候选点按y坐标排序，每个点最多只需要检查后面6个点

### 线段相交（Line Segment Intersection）

判断两条线段是否相交需要两个条件：
1. 快速排斥实验：两个线段的包围盒相交
2. 跨立实验：每条线段的两个端点分别在另一条线段的两侧

### 扫线算法（Sweep Line Algorithm）

扫线算法是一种处理平面几何问题的重要技术。想象一条垂直线从左到右扫过整个平面，在扫过过程中维护当前状态并处理事件。

典型应用包括：
- 矩形面积并
- 线段相交检测
- Voronoi图构造

## 3. 代码演示

### 叉积计算和方向判断

```python
class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    
    def __repr__(self):
        return f"({self.x}, {self.y})"

def cross_product(o, a, b):
    """计算向量oa和ob的叉积"""
    return (a.x - o.x) * (b.y - o.y) - (a.y - o.y) * (b.x - o.x)

def orientation(p, q, r):
    """判断三点的方向关系"""
    val = cross_product(p, q, r)
    if val == 0:
        return 0  # 共线
    return 1 if val > 0 else 2  # 1表示逆时针，2表示顺时针
```

### Graham扫描法求凸包

```python
import math

def graham_scan(points):
    """Graham扫描法求凸包"""
    if len(points) <= 2:
        return points
    
    # 找到最下方的点
    start = min(points, key=lambda p: (p.y, p.x))
    
    # 按极角排序
    def polar_angle(p):
        return math.atan2(p.y - start.y, p.x - start.x)
    
    sorted_points = sorted(points, key=polar_angle)
    
    # 处理共线点，保留最远的
    def distance_squared(p1, p2):
        return (p1.x - p2.x) ** 2 + (p1.y - p2.y) ** 2
    
    i = 1
    while i < len(sorted_points):
        if cross_product(start, sorted_points[i-1], sorted_points[i]) == 0:
            # 共线，保留距离更远的
            if distance_squared(start, sorted_points[i]) > distance_squared(start, sorted_points[i-1]):
                sorted_points[i-1] = sorted_points[i]
            sorted_points.pop(i)
        else:
            i += 1
    
    # Graham扫描
    hull = [start, sorted_points[1]]
    for i in range(2, len(sorted_points)):
        while len(hull) >= 2 and cross_product(hull[-2], hull[-1], sorted_points[i]) <= 0:
            hull.pop()
        hull.append(sorted_points[i])
    
    return hull
```

### 点在多边形内测试（射线法）

```python
def point_in_polygon(point, polygon):
    """射线法判断点是否在多边形内"""
    x, y = point.x, point.y
    n = len(polygon)
    inside = False
    
    p1x, p1y = polygon[0].x, polygon[0].y
    for i in range(1, n + 1):
        p2x, p2y = polygon[i % n].x, polygon[i % n].y
        if y > min(p1y, p2y):
            if y <= max(p1y, p2y):
                if x <= max(p1x, p2x):
                    if p1y != p2y:
                        xinters = (y - p1y) * (p2x - p1x) / (p2y - p1y) + p1x
                    if p1x == p2x or x <= xinters:
                        inside = not inside
        p1x, p1y = p2x, p2y
    
    return inside
```

## 4. 深入理解

### 为什么Graham扫描法要按极角排序？

极角排序确保了我们按照逆时针方向遍历所有点，这样在扫描过程中可以保证凸包的正确性。如果顺序混乱，可能会错过某些应该包含在凸包中的点。

### 分治法求最近点对的关键洞察

分治法的核心在于第三步：跨越中线的点对。关键洞察是，在宽度为2d的带状区域内，每个点只需要与后面有限个点比较。这是因为如果两个点的距离小于d，它们在y方向上的距离也不能超过d，所以按y排序后，每个点后面最多只有常数个候选点。

### 扫线算法的事件驱动思想

扫线算法将几何问题转化为事件处理问题。每个几何对象（如线段端点）对应一个事件，按x坐标排序。在扫线过程中，我们维护一个活跃对象的数据结构（通常是平衡二叉搜索树），并在遇到事件时更新状态和检测相交。

### 浮点精度问题

计算几何中经常涉及浮点运算，这会带来精度问题。在实际实现中，我们需要考虑使用适当的epsilon值进行比较，或者使用整数运算避免浮点误差。

## 6. 常见误区

### 误区1：忽略边界情况

计算几何问题有很多边界情况需要处理：
- 三点共线
- 点在线段上
- 多边形退化（如自相交）
- 浮点精度导致的错误判断

### 误区2：混淆不同的凸包算法

Graham扫描法和Jarvis步进法适用于不同的场景：
- Graham扫描法适合点数较多但凸包顶点数也较多的情况
- Jarvis步进法适合凸包顶点数很少的情况（输出敏感算法）

### 误区3：不考虑算法复杂度

暴力解法虽然简单，但在大数据量下会超时。例如最近点对问题，O(n²)在n=10⁵时完全不可行，必须使用分治法。

### 误区4：浮点比较错误

直接使用 `==` 比较浮点数会导致错误。应该使用 `abs(a - b) < epsilon` 的方式进行比较。

### 误区5：坐标系理解错误

不同的应用场景可能使用不同的坐标系（如屏幕坐标系y轴向下），需要特别注意方向判断的正确性。
