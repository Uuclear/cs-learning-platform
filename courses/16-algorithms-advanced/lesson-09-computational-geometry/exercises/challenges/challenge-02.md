# 挑战2：扫线算法实现线段相交检测

## 背景
给定一组线段，需要找出所有相交的线段对。暴力解法的时间复杂度是 O(n²)，但对于大量线段来说效率太低。扫线算法可以将时间复杂度优化到 O((n + k) log n)，其中k是相交对的数量。

## 要求
1. 实现事件驱动的扫线算法
2. 正确处理各种事件类型：
   - 线段左端点（插入事件）
   - 线段右端点（删除事件）
   - 线段相交（交点事件）
3. 使用平衡二叉搜索树维护当前活跃的线段
4. 正确处理特殊情况（垂直线段、共点等）
5. 返回所有相交的线段对

## 输入格式
- segments: 线段列表，每条线段由两个Point对象表示

## 输出格式
- intersections: 相交线段对的列表，每个元素是(线段1, 线段2)的元组

## 提示
- 需要定义事件类来表示不同类型的事件
- 活跃线段需要按y坐标排序（在当前扫线位置）
- 当插入或删除线段时，需要检查与相邻线段是否相交
- 当发现新的交点时，需要将其作为新事件加入事件队列
- 注意避免重复报告相同的相交对
