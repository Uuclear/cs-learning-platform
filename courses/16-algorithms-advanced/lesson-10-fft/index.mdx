# 16-10 快速傅里叶变换FFT

## 1. 开场白

大家好！今天我们要学习一个被誉为"二十世纪十大算法之一"的强大工具——快速傅里叶变换（Fast Fourier Transform，简称FFT）。这个算法不仅在信号处理、图像处理、音频分析等领域有着广泛应用，在算法竞赛和密码学中也是解决多项式乘法问题的利器。

想象一下，你需要在一个音乐流媒体平台上实时分析音频频谱，或者在一个大型电商网站上快速计算两个超大整数的乘积，又或者在一个科学计算软件中高效地进行卷积运算。所有这些看似不同的问题，都可以通过FFT得到优雅而高效的解决方案。

FFT的魅力在于它将一个看似复杂的数学变换，通过巧妙的分治策略，将时间复杂度从O(n²)降低到O(n log n)。这不仅仅是常数级别的优化，而是指数级别的飞跃！

## 2. 核心概念

### 多项式表示与乘法（Polynomial Representation and Multiplication）

多项式有两种主要表示方法：
- **系数表示**：P(x) = a₀ + a₁x + a₂x² + ... + aₙ₋₁xⁿ⁻¹
- **点值表示**：{(x₀, y₀), (x₁, y₁), ..., (xₙ₋₁, yₙ₋₁)}，其中 yᵢ = P(xᵢ)

多项式乘法在系数表示下需要O(n²)时间，但在点值表示下只需要O(n)时间（对应点值相乘）。

关键问题：如何在两种表示方法之间高效转换？

### 离散傅里叶变换（Discrete Fourier Transform, DFT）

DFT就是解决上述转换问题的数学工具。给定n个系数a₀, a₁, ..., aₙ₋₁，DFT计算n个点值：

yₖ = Σⱼ₌₀ⁿ⁻¹ aⱼ × ωₙʲᵏ

其中ωₙ = e^(2πi/n)是n次单位根。

### 单位根（Roots of Unity）

n次单位根是满足zⁿ = 1的复数解。它们在复平面上均匀分布在单位圆上：

ωₙᵏ = e^(2πik/n) = cos(2πk/n) + i×sin(2πk/n)

单位根的重要性质：
- **消去引理**：ωₙᵈᵏ = ωₙ/ᵈᵏ
- **折半引理**：(ωₙᵏ)² = ωₙ/₂ᵏ
- **求和引理**：Σₖ₌₀ⁿ⁻¹ (ωₙᵏ)ʲ = 0（当j不是n的倍数时）

### 蝴蝶图（Butterfly Diagram）

蝴蝶图是FFT算法的可视化表示，展示了如何通过递归分解将大问题转化为小问题。每个"蝴蝶"操作对应一次复数乘加运算。

### Cooley-Tukey FFT算法

这是最经典的FFT算法，采用分治策略：

1. 将多项式按奇偶系数分成两部分：
   - A₀(x) = a₀ + a₂x + a₄x² + ...
   - A₁(x) = a₁ + a₃x + a₅x² + ...

2. 递归计算A₀和A₁在n/2次单位根处的值

3. 合并结果：
   - yₖ = A₀(ωₙ/₂ᵏ) + ωₙᵏ × A₁(ωₙ/₂ᵏ)
   - yₖ₊ₙ/₂ = A₀(ωₙ/₂ᵏ) - ωₙᵏ × A₁(ωₙ/₂ᵏ)

### 多项式乘法 via FFT

使用FFT进行多项式乘法的步骤：
1. 将两个多项式的系数补零到长度≥2n-1（通常是2的幂）
2. 对两个多项式分别进行FFT，得到点值表示
3. 对应点值相乘，得到结果多项式的点值表示
4. 对结果进行逆FFT，得到系数表示

### 信号处理直觉

在信号处理中，FFT将时域信号转换为频域信号。每个输出系数代表对应频率分量的强度。这种转换让我们能够：
- 分析信号的频率组成
- 进行滤波（去除不需要的频率）
- 压缩数据（保留主要频率分量）

### 复杂度分析

- DFT直接计算：O(n²)
- FFT递归计算：T(n) = 2T(n/2) + O(n) → T(n) = O(n log n)
- 多项式乘法：O(n log n)而不是O(n²)

## 3. 代码演示

### 复数类和基本运算

```python
import math

class Complex:
    def __init__(self, real=0, imag=0):
        self.real = real
        self.imag = imag
    
    def __add__(self, other):
        return Complex(self.real + other.real, self.imag + other.imag)
    
    def __sub__(self, other):
        return Complex(self.real - other.real, self.imag - other.imag)
    
    def __mul__(self, other):
        return Complex(
            self.real * other.real - self.imag * other.imag,
            self.real * other.imag + self.imag * other.real
        )
    
    def __repr__(self):
        return f"{self.real:.3f}+{self.imag:.3f}i"
```

### 递归FFT实现

```python
def fft_recursive(a, invert=False):
    """
    递归实现FFT算法
    
    参数:
        a: 复数列表，表示多项式系数
        invert: 是否计算逆FFT
    
    返回:
        复数列表，表示DFT结果
    """
    n = len(a)
    if n == 1:
        return a
    
    # 分离奇偶系数
    a0 = [a[i] for i in range(0, n, 2)]
    a1 = [a[i] for i in range(1, n, 2)]
    
    # 递归计算
    y0 = fft_recursive(a0, invert)
    y1 = fft_recursive(a1, invert)
    
    # 合并结果
    y = [Complex() for _ in range(n)]
    angle = 2 * math.pi / n
    if invert:
        angle = -angle
    
    w = Complex(1, 0)  # 初始单位根
    wn = Complex(math.cos(angle), math.sin(angle))
    
    for i in range(n // 2):
        y[i] = y0[i] + w * y1[i]
        y[i + n // 2] = y0[i] - w * y1[i]
        if invert:
            y[i].real /= 2
            y[i].imag /= 2
            y[i + n // 2].real /= 2
            y[i + n // 2].imag /= 2
        w = w * wn
    
    return y
```

### 迭代FFT实现（更高效）

```python
def fft_iterative(a, invert=False):
    """
    迭代实现FFT算法（位逆序置换）
    """
    n = len(a)
    # 位逆序置换
    j = 0
    for i in range(1, n):
        bit = n >> 1
        while j & bit:
            j ^= bit
            bit >>= 1
        j ^= bit
        if i < j:
            a[i], a[j] = a[j], a[i]
    
    # 蝴蝶操作
    length = 2
    while length <= n:
        angle = 2 * math.pi / length
        if invert:
            angle = -angle
        wlen = Complex(math.cos(angle), math.sin(angle))
        
        for i in range(0, n, length):
            w = Complex(1, 0)
            for j in range(length // 2):
                u = a[i + j]
                v = a[i + j + length // 2] * w
                a[i + j] = u + v
                a[i + j + length // 2] = u - v
                w = w * wlen
        
        length <<= 1
    
    if invert:
        for i in range(n):
            a[i].real /= n
            a[i].imag /= n
    
    return a
```

## 4. 深入理解

### 为什么FFT要求n是2的幂？

Cooley-Tukey算法假设n是2的幂，这样可以一直二分到基本情况。对于任意n，可以使用混合基FFT或其他变种，但实现更复杂。在实际应用中，通常将输入补零到最近的2的幂。

### 位逆序置换的意义

在迭代FFT中，位逆序置换确保了数据在蝴蝶操作前处于正确的位置。这是因为递归FFT的调用顺序对应于位逆序的索引。

### 逆FFT的实现

逆FFT与正向FFT非常相似，只是：
- 使用共轭单位根（角度取负）
- 最后除以n进行归一化

### 数值稳定性

FFT涉及大量浮点运算，可能存在数值误差。在实际应用中，通常会对结果进行四舍五入或使用适当的精度控制。

### 应用领域的差异

- **信号处理**：关注频域分析，通常使用实数输入
- **多项式乘法**：关注精确计算，可能需要处理整数系数
- **密码学**：关注大整数运算，可能需要数论变换(NTT)替代FFT

## 6. 常见误区

### 误区1：FFT只能处理2的幂长度

虽然标准Cooley-Tukey算法要求n是2的幂，但存在处理任意长度的FFT算法，如Bluestein算法、Rader算法等。

### 误区2：FFT总是比直接计算快

对于很小的n（比如n<32），直接DFT可能更快，因为FFT的常数因子较大。实际实现中通常设置一个阈值。

### 误区3：忽略数值精度问题

FFT的结果是近似值，对于需要精确结果的应用（如大整数乘法），可能需要使用数论变换(NTT)或多重精度计算。

### 误区4：混淆FFT和DFT

DFT是数学变换，FFT是计算DFT的高效算法。两者概念不同但密切相关。

### 误区5：不理解输入输出的含义

在信号处理中，FFT输出的前半部分对应正频率，后半部分对应负频率。在多项式乘法中，输出就是点值表示。
