## 2. 核心概念

### 字符串匹配问题定义

给定一个长度为n的文本字符串T和一个长度为m的模式字符串P，我们的目标是找到所有位置i（0 ≤ i ≤ n-m），使得T[i:i+m] = P。

### KMP算法（Knuth-Morris-Pratt）

KMP算法的核心思想是**避免回溯**。当发生不匹配时，不是简单地将模式向右移动一位，而是利用已经匹配的信息来决定最优的移动距离。

**失败函数（Failure Function）/ 前缀函数（Prefix Function）**：
对于模式P的每个位置i，计算π[i]表示P[0:i]的最长真前缀（同时也是后缀）的长度。

例如，对于模式"ABABC"：
- π[0] = 0（单个字符没有真前缀）
- π[1] = 0（"AB"的最长公共前后缀长度为0）
- π[2] = 1（"ABA"的最长公共前后缀是"A"，长度为1）
- π[3] = 2（"ABAB"的最长公共前后缀是"AB"，长度为2）
- π[4] = 0（"ABABC"的最长公共前后缀长度为0）

### Rabin-Karp算法

Rabin-Karp算法使用**滚动哈希（Rolling Hash）**技术。它为模式P计算一个哈希值，然后在文本T中滑动窗口，计算每个子串的哈希值进行比较。

关键优化是**滚动计算**：当窗口从位置i移动到i+1时，不需要重新计算整个哈希值，而是通过数学公式快速更新：
```
hash(T[i+1:i+1+m]) = (hash(T[i:i+m]) - T[i] * d^(m-1)) * d + T[i+m]
```
其中d是基数（通常选择大于字符集大小的质数）。

### Boyer-Moore算法

Boyer-Moore算法采用了**从右到左**的匹配策略，并结合两种启发式规则：

1. **坏字符规则（Bad Character Rule）**：当在位置j发生不匹配时，查看文本中的坏字符c = T[i+j]，然后将模式中最右边的c对齐到文本中的c位置。

2. **好后缀规则（Good Suffix Rule）**：利用已经匹配的后缀信息，找到模式中相同后缀的最右出现位置。

Boyer-Moore的实际性能往往优于其他算法，特别是在模式较长且字符集较大的情况下。

### 后缀数组（Suffix Arrays）

后缀数组是文本所有后缀按字典序排列后的起始位置数组。它是处理复杂字符串问题的强大工具，可以用于：
- 最长公共前缀（LCP）计算
- 子串搜索
- 重复子串检测
- 数据压缩

构建后缀数组的经典算法有O(n²log n)的朴素方法和O(n log n)的倍增算法。

## 3. 代码演示

让我们通过具体的代码实现来理解这些算法：

```python
def compute_lps(pattern):
    """
    计算KMP算法的LPS（最长前缀后缀）数组
    
    Args:
        pattern: 模式字符串
        
    Returns:
        list: LPS数组
    """
    m = len(pattern)
    lps = [0] * m
    length = 0  # 当前最长前缀后缀的长度
    i = 1
    
    while i < m:
        if pattern[i] == pattern[length]:
            length += 1
            lps[i] = length
            i += 1
        else:
            if length != 0:
                # 回退到之前计算的位置
                length = lps[length - 1]
            else:
                lps[i] = 0
                i += 1
    
    return lps

def kmp_search(text, pattern):
    """
    KMP字符串搜索算法
    
    Args:
        text: 文本字符串
        pattern: 模式字符串
        
    Returns:
        list: 所有匹配位置的列表
    """
    n, m = len(text), len(pattern)
    if m == 0:
        return []
    
    lps = compute_lps(pattern)
    matches = []
    
    i = j = 0  # i: text索引, j: pattern索引
    
    while i < n:
        if pattern[j] == text[i]:
            i += 1
            j += 1
        
        if j == m:
            matches.append(i - j)
            j = lps[j - 1]
        elif i < n and pattern[j] != text[i]:
            if j != 0:
                j = lps[j - 1]
            else:
                i += 1
    
    return matches

def rabin_karp_search(text, pattern, prime=101):
    """
    Rabin-Karp字符串搜索算法
    
    Args:
        text: 文本字符串
        pattern: 模式字符串
        prime: 用于哈希计算的质数
        
    Returns:
        list: 所有匹配位置的列表
    """
    n, m = len(text), len(pattern)
    if m == 0 or n < m:
        return []
    
    # 计算模式的哈希值
    pattern_hash = 0
    text_hash = 0
    h = 1  # d^(m-1) mod prime
    
    d = 256  # 字符集大小
    
    # 计算h = d^(m-1) mod prime
    for i in range(m - 1):
        h = (h * d) % prime
    
    # 计算模式和文本前m个字符的哈希值
    for i in range(m):
        pattern_hash = (d * pattern_hash + ord(pattern[i])) % prime
        text_hash = (d * text_hash + ord(text[i])) % prime
    
    matches = []
    
    for i in range(n - m + 1):
        if pattern_hash == text_hash:
            # 验证是否真的匹配（避免哈希冲突）
            if text[i:i + m] == pattern:
                matches.append(i)
        
        # 计算下一个窗口的哈希值
        if i < n - m:
            text_hash = (d * (text_hash - ord(text[i]) * h) + ord(text[i + m])) % prime
            # 处理负数
            if text_hash < 0:
                text_hash += prime
    
    return matches

# 示例使用
def demo_string_matching():
    """演示字符串匹配算法"""
    text = "ABABDABACDABABCABCABCABC"
    pattern = "ABABC"
    
    print("=== 字符串匹配算法演示 ===")
    print(f"文本: {text}")
    print(f"模式: {pattern}")
    print()
    
    # KMP算法
    kmp_matches = kmp_search(text, pattern)
    print(f"KMP匹配位置: {kmp_matches}")
    
    # Rabin-Karp算法
    rk_matches = rabin_karp_search(text, pattern)
    print(f"Rabin-Karp匹配位置: {rk_matches}")

if __name__ == "__main__":
    demo_string_matching()
```

## 4. 深入理解

### 算法复杂度对比

| 算法 | 预处理时间 | 搜索时间 | 最坏情况 | 平均情况 |
|------|------------|----------|----------|----------|
| 朴素 | O(1) | O(nm) | O(nm) | O(n+m) |
| KMP | O(m) | O(n) | O(n) | O(n) |
| Rabin-Karp | O(m) | O(nm) | O(nm) | O(n+m) |
| Boyer-Moore | O(m+σ) | O(nm) | O(nm) | O(n/m) |

其中σ是字符集大小。

### KMP的失败函数构建

KMP的失败函数构建过程本身就是一个精妙的动态规划过程。关键洞察是：当我们知道π[0], π[1], ..., π[i-1]时，如何计算π[i]？

如果P[i] == P[π[i-1]]，那么π[i] = π[i-1] + 1。
否则，我们需要回退到π[π[i-1]-1]，继续比较，直到找到匹配或回到0。

这个过程保证了每个字符最多被访问常数次，因此构建LPS数组的时间复杂度是O(m)。

### Rabin-Karp的哈希选择

Rabin-Karp算法的性能很大程度上取决于哈希函数的选择：
- **基数d**：通常选择大于字符集大小的质数
- **模数prime**：选择大质数以减少哈希冲突
- **滚动哈希**：确保可以在O(1)时间内计算下一个窗口的哈希值

在实际应用中，有时会使用双哈希（两个不同的质数）来进一步减少冲突概率。

### Boyer-Moore的预处理

Boyer-Moore需要两个预处理表：
1. **坏字符表**：记录每个字符在模式中最右边出现的位置
2. **好后缀表**：记录每个后缀的匹配信息

虽然预处理时间较长，但在长模式和大字符集的情况下，搜索阶段的跳跃效果非常显著。

## 6. 常见误区

### 误区1：认为Rabin-Karp总是O(n+m)

实际上，Rabin-Karp的最坏情况仍然是O(nm)，当哈希冲突频繁发生时。只有在平均情况下才能达到O(n+m)。

### 误区2：忽略Boyer-Moore的预处理开销

Boyer-Moore虽然搜索很快，但预处理需要O(m+σ)时间。对于短模式或小字符集，预处理开销可能超过收益。

### 误区3：混淆KMP的LPS数组含义

LPS[i]表示的是P[0:i+1]的最长真前缀后缀长度，不是P[0:i]的。注意数组索引的边界。

### 误区4：认为后缀数组只能用于精确匹配

后缀数组配合LCP数组可以解决许多复杂的字符串问题，如最长重复子串、最长公共子串等。

### 练习1：实现Boyer-Moore算法

```python
def build_bad_char_table(pattern):
    """构建坏字符表"""
    table = {}
    m = len(pattern)
    for i in range(m):
        table[pattern[i]] = i
    return table

def boyer_moore_search(text, pattern):
    """Boyer-Moore字符串搜索算法（简化版，只使用坏字符规则）"""
    n, m = len(text), len(pattern)
    if m == 0:
        return []
    
    bad_char = build_bad_char_table(pattern)
    matches = []
    shift = 0
    
    while shift <= n - m:
        j = m - 1
        
        # 从右到左比较
        while j >= 0 and pattern[j] == text[shift + j]:
            j -= 1
        
        if j < 0:
            matches.append(shift)
            # 模式完全匹配
            shift += (m - bad_char.get(text[shift + m], -1)) if shift + m < n else 1
        else:
            # 坏字符规则
            shift += max(1, j - bad_char.get(text[shift + j], -1))
    
    return matches
```
