# 挑战1: 多模式字符串匹配

**难度**: ★★★★☆ (4/5)

## 问题描述

在实际应用中，我们经常需要同时搜索多个模式字符串。例如，网络安全系统需要检测多种恶意软件签名，搜索引擎需要匹配多个关键词。

给定一个文本字符串和k个模式字符串，找出文本中所有模式的出现位置。

## 输入输出规格

### 输入格式
- 第一行：文本字符串T
- 第二行：整数k（模式数量）
- 接下来k行：每个模式字符串P_i

### 输出格式
- 对于每个模式P_i，输出一行包含所有匹配位置（按升序排列），用空格分隔
- 如果没有匹配，输出空行

### 约束条件
- 1 ≤ |T| ≤ 10⁵
- 1 ≤ k ≤ 1000  
- 1 ≤ |P_i| ≤ 100
- 所有字符串只包含小写字母

## 示例

### 输入
```
ababcababa
3
aba
abc
bab
```

### 输出
```
0 6
2
1 7
```

### 解释
- "aba" 在位置0和6出现
- "abc" 在位置2出现  
- "bab" 在位置1和7出现

## 提示

1. **朴素方法**：对每个模式单独运行KMP算法，时间复杂度O(k(n + m))
2. **优化方法**：使用Aho-Corasick自动机，可以一次性处理所有模式
3. **内存考虑**：当k很大时，Aho-Corasick可能占用较多内存
4. **实现选择**：根据约束条件，朴素方法可能已经足够高效

## 参考解决方案思路

由于k ≤ 1000且每个模式长度 ≤ 100，总模式长度不超过10⁵，朴素方法的时间复杂度为O(10⁵ × (10⁵ + 100)) ≈ O(10¹⁰)，这在Python中可能超时。

更好的方法是使用Aho-Corasick自动机：
1. 构建Trie树存储所有模式
2. 添加失败指针（类似KMP的LPS数组）
3. 在文本上运行自动机，一次性找到所有匹配

Aho-Corasick的时间复杂度为O(n + m + z)，其中z是匹配总数，这在大多数情况下更高效。

对于本题的约束，也可以考虑以下优化：
- 如果总模式长度较小，使用朴素方法
- 如果模式很多且较长，使用Aho-Corasick
- 预处理模式，移除重复或包含关系的模式
