# 挑战2: 最长重复子串

**难度**: ★★★★★ (5/5)

## 问题描述

给定一个字符串，找出其中最长的重复子串。重复子串是指在字符串中出现至少两次的子串（出现位置可以重叠）。

例如，在字符串"banana"中，最长重复子串是"ana"（出现在位置1和3）。

## 输入输出规格

### 输入格式
- 一行字符串S

### 输出格式
- 最长重复子串
- 如果有多个相同长度的最长重复子串，输出字典序最小的那个
- 如果没有重复子串，输出空字符串

### 约束条件
- 1 ≤ |S| ≤ 10⁵
- 字符串只包含小写字母

## 示例

### 输入
```
banana
```

### 输出
```
ana
```

### 解释
- "a" 出现3次
- "an" 出现2次  
- "ana" 出现2次（位置1和3）
- "n" 出现2次
- 最长的是"ana"，长度为3

## 提示

1. **朴素方法**：枚举所有子串，使用哈希表记录出现次数，时间复杂度O(n³)
2. **二分答案**：二分重复子串的长度，用滚动哈希检查是否存在重复，时间复杂度O(n log n)
3. **后缀数组**：构建后缀数组和LCP数组，相邻后缀的LCP就是公共前缀长度，时间复杂度O(n log n)
4. **后缀自动机**：构建后缀自动机，找到endpos集合大小≥2的最长状态，时间复杂度O(n)

## 参考解决方案思路

**方法1：二分 + 滚动哈希**
1. 二分答案：重复子串长度范围[0, n-1]
2. 对于每个长度L，计算所有长度为L的子串的哈希值
3. 检查是否有重复的哈希值（注意处理哈希冲突）
4. 找到最大可行的L，然后输出对应的子串

**方法2：后缀数组 + LCP**
1. 构建后缀数组SA
2. 构建LCP数组（最长公共前缀数组）
3. LCP数组的最大值就是最长重复子串的长度
4. 找到对应的位置，提取子串
5. 如果有多个相同长度，选择字典序最小的

对于本题约束（n ≤ 10⁵），两种方法都可以接受。后缀数组方法更优雅但实现复杂，二分+滚动哈希更容易实现且效率不错。

**关键优化**：
- 使用双哈希减少冲突概率
- 在二分过程中记录最优解的具体内容
- 处理字典序要求：当找到相同长度的重复子串时，比较并保留字典序较小的
