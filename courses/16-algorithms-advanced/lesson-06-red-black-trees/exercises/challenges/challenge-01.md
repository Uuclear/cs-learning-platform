# 挑战1：实现红黑树的删除操作

## 背景
红黑树的删除操作比插入操作复杂得多，因为删除黑色节点可能会破坏红黑树的平衡性质。你需要实现完整的删除操作，包括删除后的修复过程。

## 要求
1. 实现 `delete(key)` 方法，能够正确删除指定键值的节点
2. 实现 `_delete_fixup(x)` 方法，在删除后修复红黑树性质
3. 处理所有可能的删除情况（8种不同的修复情况）
4. 确保删除操作后仍然满足红黑树的五个基本性质

## 提示
- 删除操作的第一步是找到要删除的节点，如果该节点有两个子节点，需要找到其后继节点并交换
- 删除黑色节点时，需要用双黑色（double black）的概念来处理
- 修复过程需要考虑兄弟节点的颜色、兄弟节点子节点的颜色等
- 可以参考CLRS算法导论中的红黑树删除算法

## 测试用例
```python
# 创建红黑树并插入数据
tree = RedBlackTree()
keys = [10, 20, 30, 15, 25, 5, 1, 7]
for key in keys:
    tree.insert(key)

# 删除操作
tree.delete(10)  # 删除根节点
tree.delete(5)   # 删除叶子节点
tree.delete(20)  # 删除有两个子节点的节点

# 验证结果仍然满足红黑树性质
assert tree.validate_red_black_properties()
```
