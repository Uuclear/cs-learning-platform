# 编程挑战 2：带心跳机制的聊天应用 ⭐⭐

## 题目描述
扩展基础聊天应用，添加心跳机制和用户在线状态管理功能。

## 具体要求
- 实现WebSocket服务器，支持多用户聊天
- 客户端连接时需要提供用户名
- 服务器定期（每30秒）向所有客户端发送心跳请求（ping）
- 客户端收到ping后必须回复pong
- 如果服务器在45秒内没有收到客户端的任何消息（包括pong），则认为该客户端已离线并断开连接
- 当用户加入或离开时，向所有在线用户广播通知
- 显示当前在线用户列表

## 提示
- 使用asyncio的sleep函数实现定时器
- 维护每个客户端的最后活动时间戳
- 在消息处理循环中检查超时情况
- 考虑使用单独的任务来处理心跳逻辑

## 预期效果
```
用户A加入聊天室 → 所有用户收到"用户A加入了聊天室"
用户B加入聊天室 → 所有用户收到"用户B加入了聊天室"，并更新用户列表
用户A离线（超时）→ 所有用户收到"用户A离开了聊天室"，并更新用户列表
```

## 进阶要求（可选）
- 实现私聊功能（@用户名 消息内容）
- 添加消息历史记录功能
- 支持表情符号和简单格式化
