# 挑战 2: 实现安全合规即代码

## 背景
你的团队正在将安全合规要求（如SOC 2、GDPR）集成到开发流程中。你需要将合规检查自动化，确保每次部署都满足安全基线。

## 任务
编写一个Python脚本 `compliance_checker.py`，模拟安全合规检查流程。该脚本应该：

1. **基础设施合规检查**：验证云资源配置是否符合安全基线（如S3桶是否公开、安全组是否开放过多端口）
2. **数据加密合规**：验证敏感数据是否在传输和静态时都经过加密
3. **访问控制合规**：验证IAM策略是否遵循最小权限原则
4. **日志审计合规**：验证所有关键操作是否有审计日志

## 要求
- 使用Python标准库实现
- 定义合规规则的数据结构（如JSON配置）
- 实现规则引擎，能够评估基础设施状态并输出合规报告
- 报告应包含：合规项、违规项、风险等级、修复建议
- 支持添加自定义合规规则

## 提交内容
- `compliance_checker.py`：完整的合规检查脚本
- `compliance_rules.json`：示例合规规则配置
- 运行结果截图，展示合规报告的输出

## 提示
- 可以使用正则表达式和字符串匹配来模拟基础设施检查
- 风险等级可以分为：critical、high、medium、low
- 修复建议应该是具体可操作的
