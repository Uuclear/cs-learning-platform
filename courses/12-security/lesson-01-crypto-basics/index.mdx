## 2. 核心概念

### 对称加密 (Symmetric Encryption) - AES
对称加密使用**同一个密钥**进行加密和解密。就像你用同一把钥匙锁门和开门。

```python
# 伪代码示例
encrypted = encrypt_AES(plaintext, secret_key)
decrypted = decrypt_AES(encrypted, secret_key)  # 使用相同的密钥
```

**AES (Advanced Encryption Standard)** 是目前最广泛使用的对称加密算法，支持128、192、256位密钥长度。

### 非对称加密 (Asymmetric Encryption) - RSA
非对称加密使用**一对密钥**：公钥（public key）和私钥（private key）。公钥可以公开分享用于加密，只有对应的私钥才能解密。

```python
# 伪代码示例  
encrypted = encrypt_RSA(plaintext, public_key)    # 任何人都可以用公钥加密
decrypted = decrypt_RSA(encrypted, private_key)   # 只有私钥持有者能解密
```

**RSA** 是最著名的非对称加密算法，基于大数分解的数学难题。

### 哈希函数 (Hash Functions) - SHA
哈希函数将任意长度的数据转换为**固定长度**的唯一指纹。重要特性：
- **单向性**：无法从哈希值反推出原文
- **抗碰撞性**：很难找到两个不同的输入产生相同的哈希值

**SHA-256** 是目前推荐的标准哈希算法。

### 数字签名 (Digital Signatures)
数字签名结合了哈希和非对称加密，用于**验证消息的真实性和完整性**：

1. 发送方用私钥对消息哈希值进行加密 → 生成数字签名
2. 接收方用发送方公钥解密签名，得到哈希值
3. 接收方自己计算消息哈希值，与解密结果对比

如果一致，说明消息确实来自发送方且未被篡改。

### 密钥交换 (Key Exchange)
如何在不安全的网络上安全地共享密钥？**Diffie-Hellman 密钥交换**协议解决了这个问题，允许双方在公开信道上协商出一个共享密钥。

## 4. 深入理解

### 加密模式：ECB vs CBC vs GCM
- **ECB (Electronic Codebook)**：每个数据块独立加密，相同明文块产生相同密文块 ❌ 不安全
- **CBC (Cipher Block Chaining)**：每个块与前一个密文块异或后再加密，需要初始化向量(IV) ✅ 更安全
- **GCM (Galois/Counter Mode)**：提供加密和认证，支持并行处理，现代推荐 ✅✅ 最佳选择

### 密钥长度与安全强度
| 算法 | 推荐密钥长度 | 安全强度 |
|------|-------------|----------|
| AES | 256位 | 抗量子计算攻击 |
| RSA | 3072位+ | 当前安全标准 |
| ECC | 256位 | 与RSA 3072位相当 |

记住：**更长的密钥 ≠ 更好的安全性**，关键在于正确的实现和密钥管理。

### 熵与随机数
加密的安全性依赖于**真正的随机性**。计算机生成的"随机数"实际上是伪随机的，密码学应用需要**密码学安全的随机数生成器 (CSPRNG)**。

Python的 `secrets` 模块提供了密码学安全的随机数生成。

## 5. 实际应用

让我们构建一个简单的端到端加密通信系统：

```python
# 使用Python cryptography库
from cryptography.fernet import Fernet
import hashlib
import secrets

# 1. 生成安全密钥
key = Fernet.generate_key()  # 256位AES密钥

# 2. 创建加密器
cipher_suite = Fernet(key)

# 3. 加密消息
message = b"Hello, secure world!"
encrypted_message = cipher_suite.encrypt(message)

# 4. 解密消息  
decrypted_message = cipher_suite.decrypt(encrypted_message)

print(f"Original: {message}")
print(f"Encrypted: {encrypted_message}")
print(f"Decrypted: {decrypted_message}")
```

这个简单的例子展示了现代加密库如何让我们轻松实现安全通信。但在实际应用中，还需要考虑密钥分发、身份验证、前向保密等复杂问题。

## 6. 常见误区

### ❌ 不要自己发明加密算法
> "Anyone can invent a cipher they themselves can't break." — Bruce Schneier

专业的密码学家经过数十年研究才设计出当前的标准算法。自己发明的算法几乎肯定存在漏洞。

### ❌ 随机数不是随机的
使用 `random.randint()` 生成密钥？这是灾难性的！必须使用 `secrets` 或 `os.urandom()`。

### ❌ 密钥管理比算法更重要
再强大的算法，如果密钥存储不当（如硬编码在代码中），整个系统就会崩溃。密钥应该：
- 安全存储（使用密钥管理系统）
- 定期轮换
- 限制访问权限
- 使用强密码保护

### ❌ 忽视侧信道攻击
即使算法本身安全，实现中的时序差异、内存访问模式、功耗变化都可能泄露密钥信息。
