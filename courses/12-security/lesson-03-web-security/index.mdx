## 核心概念 - OWASP Top 10 速览

OWASP Top 10 是 Web 应用安全风险的行业标准清单，帮你按优先级排雷：

### 1. 注入攻击（Injection）
在输入里塞恶意代码（SQL、命令等），操纵后端逻辑。SQL 注入是最经典的「在搜索框里写代码」。

### 2. 跨站脚本（XSS）
把恶意脚本塞进页面，别的用户打开就中招——存储型、反射型、DOM 型，花样不少。

### 3. 跨站请求伪造（CSRF）
用户已登录银行网站，攻击者诱导他点一个链接，**浏览器自动带上 Cookie**，转账就完成了。

### 4. 敏感数据暴露
密码、身份证没加密就存，或用弱算法——等于把日记贴在布告栏。

### 5. 认证和会话管理失效
弱密码策略、会话不过期、没有 MFA，都是给攻击者留门。

### 6. 安全配置错误
默认账号、调试信息外泄、没关的服务端口——「出厂设置」往往是漏洞设置。

### 7. 不安全的反序列化
篡改序列化对象，可能直接 RCE——Java、Python 生态都要当心。

### 8. 使用含有已知漏洞的组件
`npm audit` 和 `pip audit` 不是摆设，过期依赖是定时炸弹。

### 9. 不足的日志记录和监控
被黑了都不知道，等于家里进贼你还在打呼噜。

### 10. 服务端请求伪造（SSRF）
让服务器替你去访问内网 `http://169.254.169.254/`——云环境尤其危险。

## 深入理解 - 纵深防御，别指望「一招鲜」

### 内容安全策略（CSP）
用 HTTP 头限制页面能加载哪些脚本，XSS 的「防火墙」：

```http
Content-Security-Policy: default-src 'self'; script-src 'self'
```

### SameSite Cookie
让 Cookie 不轻易跟着跨站请求走，CSRF 的克星：

```http
Set-Cookie: sessionid=abc123; Path=/; Secure; HttpOnly; SameSite=Lax
```

### 输入验证 + 输出编码
- **入**：白名单校验长度、类型、格式
- **出**：按上下文编码（HTML、URL、JS 各不同）

### HTTPS 只是第一层
传输加密挡不住应用层 XSS/CSRF——就像给信封上了锁，信纸里还是可以写坏话。

## 实际应用 - Python 生态里的「保安队」

| 威胁 | 常用工具 |
|------|----------|
| SQL 注入 | SQLAlchemy / Django ORM（参数化查询） |
| XSS | Bleach、Django 自动转义 |
| CSRF | Flask-WTF、Django CSRF 中间件 |
| 密码存储 | Passlib（bcrypt/argon2） |
| 加密 | `cryptography` 库 |

**原则**：框架自带的防护要用上，别为了「灵活」手写拼接 SQL。

## 常见误区

### ❌ 前端验证 = 安全
前端只是 UX；后端必须再验一遍，攻击者直接调 API 根本不看你的表单。

### ❌ 上了 HTTPS 就万事大吉
HTTPS 保护「路上」；XSS、CSRF、逻辑漏洞在「应用里」。

### ❌ 安全是安全团队的事
每个写 `request.get()` 的人都是防线；Shift Left，越早修越便宜。

### ❌ 黑名单过滤够用
攻击者总比你多一个绕过姿势；**白名单**才是正道。

## 小结与思考

Web 安全没有「毕业那天」——威胁在进化，OWASP Top 10 是地图，不是终点。记住四条：

1. **纵深防御** — 单点失效别拖垮全局  
2. **最小权限** — 能只读就别给写  
3. **安全左移** — 设计、开发、测试都带上安全眼镜  
4. **持续学习** — 关注 CVE、依赖更新、渗透测试报告
