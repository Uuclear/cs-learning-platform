# 实现安全的用户注册登录系统

## 难度：⭐⭐⭐
## 目标时间：45-60分钟

## 背景
用户认证是Web应用中最关键的安全功能之一。一个不安全的认证系统可能导致账户被盗、数据泄露等严重后果。

## 要求

### 功能需求
1. **用户注册**
   - 用户名（3-20字符，字母数字下划线）
   - 邮箱（格式验证）
   - 密码（至少8位，包含大小写字母、数字和特殊字符）
   - 密码确认

2. **用户登录**
   - 用户名/邮箱 + 密码认证
   - 登录失败次数限制（防止暴力破解）
   - 成功登录后生成会话

3. **密码重置**
   - 通过邮箱发送重置链接
   - 重置令牌有有效期（15分钟）

### 安全要求
1. **密码存储**
   - 使用 `bcrypt` 或 `argon2` 进行密码哈希
   - 每个密码使用唯一的盐值

2. **会话管理**
   - 使用加密安全的随机令牌作为会话ID
   - 设置Cookie属性：`HttpOnly`, `Secure`, `SameSite=Lax`
   - 会话有过期时间（30分钟不活动过期）

3. **输入验证**
   - 所有输入进行严格的格式和长度验证
   - 防止用户名枚举（注册和登录时返回相同的错误信息）

4. **输出编码**
   - 在HTML上下文中对所有用户数据进行HTML编码
   - 防止XSS攻击

5. **CSRF防护**
   - 所有敏感操作（登录、注册、密码重置）都需要CSRF令牌

### 技术栈
- Python 3.10+
- 使用内置 `http.server` 模块
- 可使用第三方库：`bcrypt`, `secrets`, `html`
