# 构建Web安全扫描器检测常见漏洞

## 难度：⭐⭐⭐⭐
## 目标时间：90-120分钟

## 背景
自动化安全扫描是发现Web应用漏洞的重要手段。通过构建自己的扫描器，可以深入理解漏洞检测原理和防御机制。

## 要求

### 核心功能
构建一个命令行工具，能够扫描指定的URL并检测以下漏洞：

1. **SQL注入检测**
   - 测试常见的SQL注入向量
   - 检测基于错误的SQL注入
   - 检测基于时间的SQL注入（可选）

2. **XSS检测**
   - 测试反射型XSS向量
   - 分析响应中是否包含未编码的payload
   - 检测HTML上下文中的XSS机会

3. **安全头检测**
   - 检查缺失的安全HTTP头：
     - `Content-Security-Policy`
     - `X-Frame-Options`
     - `X-Content-Type-Options`
     - `Strict-Transport-Security`

4. **Cookie安全检测**
   - 检查Cookie是否缺少安全属性：
     - `HttpOnly`
     - `Secure`
     - `SameSite`

### 技术要求
1. **输入处理**
   - 支持命令行参数指定目标URL
   - 支持扫描多个URL
   - 支持自定义扫描深度（GET/POST参数）

2. **请求处理**
   - 使用 `requests` 库发送HTTP请求
   - 处理重定向和错误状态码
   - 设置合理的超时和重试机制

3. **漏洞检测逻辑**
   - 实现智能的payload测试（避免误报）
   - 对比正常响应和注入响应的差异
   - 使用启发式方法识别潜在漏洞

4. **输出报告**
   - 生成结构化的扫描报告
   - 包含漏洞详情、风险等级和修复建议
   - 支持JSON和文本格式输出

### 安全考虑
- 扫描器本身应该是安全的（不包含漏洞）
- 避免对目标造成拒绝服务
- 尊重robots.txt和速率限制
- 不执行破坏性测试（仅检测，不利用）

### 示例用法
```bash
# 基本扫描
python3 web_scanner.py --url https://example.com

# 扫描多个URL
python3 web_scanner.py --urls urls.txt

# 自定义扫描选项
python3 web_scanner.py --url https://example.com --depth 2 --format json
```

## 评估标准
- ✅ 漏洞检测的准确性和覆盖率
- ✅ 代码架构和模块化设计
- ✅ 性能和资源使用效率
- ✅ 报告的质量和实用性
- ✅ 安全性和可靠性

## 提示
1. 从简单的检测开始，逐步增加复杂性
2. 参考OWASP ZAP或Burp Suite的检测逻辑
3. 使用正则表达式和字符串匹配进行初步检测
4. 考虑使用多线程提高扫描效率（但要控制并发数）
5. 测试你的扫描器对已知漏洞应用的效果
