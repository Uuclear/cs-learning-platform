# 编程挑战 2：装饰器模式实现 🌟🌟

## 背景故事
你正在开发一个咖啡店点单系统。顾客可以点基础饮料（如咖啡、茶），然后添加各种调料（如牛奶、糖、奶油）。每种调料都会增加价格，你需要能够动态地组合这些调料。

## 任务要求
1. 创建一个 `Beverage` 抽象基类，包含 `get_description()` 和 `cost()` 方法
2. 实现至少两种基础饮料类（如 `Coffee`, `Tea`）
3. 创建一个 `CondimentDecorator` 装饰器抽象基类，继承自 `Beverage`
4. 实现至少三种调料装饰器（如 `Milk`, `Sugar`, `WhippedCream`）
5. 编写测试代码，展示如何组合不同的饮料和调料

## 示例输出
```
黑咖啡: ¥15.00
黑咖啡, 牛奶: ¥18.00
黑咖啡, 牛奶, 糖, 奶油: ¥26.00

绿茶: ¥12.00
绿茶, 糖: ¥13.00
```

## 提示
- 装饰器的构造函数需要接收一个 `Beverage` 对象
- 装饰器的 `cost()` 方法应该调用被装饰对象的 `cost()` 并加上自己的价格
- 装饰器的 `get_description()` 方法应该组合被装饰对象的描述和自己的描述

## 难度：⭐⭐（中等）
