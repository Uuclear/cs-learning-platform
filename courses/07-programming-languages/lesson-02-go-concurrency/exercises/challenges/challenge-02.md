# 挑战2: 并发文件处理管道

## 背景
在处理大量文件时，我们需要高效的并发处理机制。Go的channel和goroutine非常适合构建处理管道。

## 任务
实现一个文件处理管道，能够：
1. 读取指定目录下的所有文件
2. 并发计算每个文件的SHA256哈希值
3. 将结果按文件名排序后输出

## 要求
1. 使用管道模式：一个goroutine负责发现文件，多个worker goroutine负责计算哈希，一个goroutine负责收集和排序结果
2. 支持配置worker数量
3. 处理文件读取错误（如权限不足、文件不存在等）
4. 使用channel进行goroutine间通信，避免共享状态
5. 实现优雅关闭，确保所有资源被正确释放

## 提示
- 使用`filepath.Walk`或`os.ReadDir`遍历目录
- 使用`crypto/sha256`计算哈希值
- 考虑使用`select`语句和`context`实现超时控制
- 可以使用结构体封装文件路径和哈希结果

## 验收标准
- 能正确处理包含100+文件的目录
- 在多核CPU上能充分利用并发优势
- 错误处理完善，不会因为单个文件错误导致整个程序崩溃
- 代码结构清晰，符合Go并发最佳实践
