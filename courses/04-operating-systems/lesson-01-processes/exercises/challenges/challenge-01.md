# 编程挑战1: 进程资源监控器

## 背景
系统管理员需要一个工具来监控系统中资源消耗异常的进程。

## 任务要求
编写一个Python程序，能够：
1. 扫描系统中所有正在运行的进程
2. 找出CPU使用率超过80%或内存使用超过1GB的进程
3. 将这些高资源消耗的进程信息保存到日志文件中
4. 提供命令行参数支持自定义阈值

## 具体要求
- 使用`psutil`库获取进程信息
- 支持以下命令行参数：
  - `--cpu-threshold`: CPU使用率阈值（默认80）
  - `--memory-threshold`: 内存阈值MB（默认1024）
  - `--output`: 输出日志文件路径（默认`high_usage_processes.log`）
- 日志格式应包含：时间戳、PID、进程名、CPU使用率、内存使用量
- 处理权限不足和进程终止等异常情况
- 代码要有完整的中文注释

## 示例输出
```
[2026-05-03 10:30:15] PID 12345, chrome, CPU: 85.2%, Memory: 1256.7 MB
[2026-05-03 10:30:15] PID 12346, python3, CPU: 92.1%, Memory: 856.3 MB
```
