# 编程挑战2：多级反馈队列调度算法 ⭐⭐⭐

## 背景
多级反馈队列（Multilevel Feedback Queue, MLFQ）是现代操作系统常用的调度算法，它结合了多种调度策略的优点。

MLFQ使用多个优先级队列：
- 新进程进入最高优先级队列
- 每个队列有不同的时间片大小（高优先级队列时间片短）
- 如果进程在时间片内未完成，则降级到下一级队列
- 最低优先级队列通常使用FCFS或大时间片的轮转调度

## 任务
实现一个MLFQ调度算法，要求：

1. 支持可配置的队列数量（默认3级）
2. 支持可配置的时间片大小（默认[2, 4, 8]）
3. 正确处理进程在不同队列间的迁移
4. 实现老化机制防止饥饿（可选加分项）

## 输入示例
```python
processes = [
    Process(1, 0, 8),
    Process(2, 1, 4),
    Process(3, 2, 9),
    Process(4, 3, 5),
]
```

## 输出要求
- 显示每个时间点的调度事件和各队列状态
- 计算并显示平均周转时间和平均等待时间
- 清晰标识进程在哪个队列中执行

## 提示
- 使用deque列表来表示多个队列
- 在每个时间步长检查新到达的进程并加入最高优先级队列
- 注意处理上下文切换和队列迁移的逻辑
- 参考轮转调度的实现思路

## 扩展挑战（可选）
- 实现老化机制：定期提升长时间等待进程的优先级
- 添加统计信息：每个进程在各级队列中的执行时间
- 支持动态调整时间片大小
