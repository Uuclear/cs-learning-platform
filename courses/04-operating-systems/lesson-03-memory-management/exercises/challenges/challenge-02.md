# 编程挑战 2: 页面置换算法比较 ⭐⭐

## 背景
不同的页面置换算法有不同的性能特征。FIFO（先进先出）简单但可能产生Belady异常，而LRU（最近最少使用）性能更好但实现复杂。

## 任务
创建一个页面置换模拟器，能够：
1. 实现FIFO页面置换算法
2. 实现简单的近似LRU算法（使用时间戳或计数器）
3. 对比两种算法在相同访问序列下的缺页率

## 要求
- 创建 `PageReplacementSimulator` 基类
- 实现 `FIFOPageReplacement` 和 `ApproximateLRU` 子类
- 提供 `access_page(page_id)` 方法处理页面访问
- 提供 `get_statistics()` 方法返回缺页次数和命中率
- 使用至少3个不同的引用字符串测试两种算法

## 测试数据
使用以下引用字符串进行测试：
- `[1, 2, 3, 4, 1, 2, 5, 1, 2, 3, 4, 5]` (帧数=3)
- `[0, 1, 2, 3, 0, 1, 4, 0, 1, 2, 3, 4]` (帧数=4)
- 自定义一个包含局部性特征的序列

## 评估标准
- 正确实现两种算法
- 准确统计缺页次数
- 清晰的代码结构和注释
- 合理的性能对比分析

> 💡 **提示**: 
> - FIFO可以用队列实现
> - 近似LRU可以用字典存储每个页面的最后访问时间
> - Belady异常：增加帧数反而导致更多缺页的现象
