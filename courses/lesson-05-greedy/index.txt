<blockquote>
<p>人生如果能贪心就好了——每次都选看起来最好的选项，最后一定是最幸福的。但算法告诉我们：<strong>有时候行，有时候不行</strong>。</p>

<h2>开场白</h2>
<p>同学们好！今天我们来聊聊算法世界里最"短视"但也最"聪明"的策略——贪心算法。</p>
<p>想象一下，你去自助餐厅吃午饭。面前摆着龙虾、牛排、寿司、甜点……你的胃容量有限（对，就像个背包），你怎么选才能吃得最值？</p>
<ul>
<li><strong>贪心策略</strong>：每次都选单价最高的，先拿龙虾，再拿牛排……</li>
<li><strong>动态规划策略</strong>：先列个表，算算各种组合的总价值，找到最优方案……</li>
<li><strong>你的真实策略</strong>：先转三圈，最后拿了半盘炸鸡和一碗汤</li>
</ul>
<p><strong>那么问题来了：</strong></p>
<ul>
<li>贪心算法为什么叫"贪心"？它到底在"贪"什么？</li>
<li>什么时候"短视"反而能得到全局最优解？</li>
<li>为什么有时候贪心会翻车？</li>
</ul>
<p>别急，今天教授带你看看，这个"只看眼前"的策略到底有什么魔力，又有什么致命缺陷。</p>
<blockquote>
<p>💡 <strong>思考问题</strong>: 如果有一个算法，每一步都选"当前最好的"，它最终一定能得到"全局最好的"结果吗？为什么？</p>
</blockquote>
<hr />
<h2>核心概念</h2>
<h3>什么是贪心算法？</h3>
<p>贪心算法（Greedy Algorithm），顾名思义，就是一种**"鼠目寸光"<strong>的算法策略。它在每一步都做出</strong>当前看起来最好的选择**，希望通过一系列局部最优选择，最终达到全局最优。</p>
<p>你可以把贪心算法想象成一个<strong>从不后悔的人</strong>：</p>
<ul>
<li>做了选择就不回头</li>
<li>不考虑未来的可能性</li>
<li>只关心"现在这个时刻，哪个选项最诱人"</li>
</ul>
<p>听起来很鲁莽对吧？但神奇的是，在某些特定问题中，这种"短视"的策略恰恰能得到<strong>全局最优解</strong>！</p>
<h3>贪心算法的两大支柱</h3>
<p>贪心算法不是对所有问题都有效。要想让贪心算法得到最优解，问题必须具备两个性质：</p>
<h4>1. 贪心选择性质</h4>
<p><strong>定义</strong>：通过一系列局部最优选择，可以得到全局最优解。</p>
<p><strong>通俗理解</strong>：你每次都选"当前最好的"，最后确实能得到"总体最好的"。</p>
<blockquote>
<p>🍕 <strong>类比</strong>：你去比萨店，每次选最大的一块。如果最终你能吃到最多量的比萨，那这个问题就具有贪心选择性质。</p>
</blockquote>
<h4>2. 最优子结构</h4>
<p><strong>定义</strong>：一个问题的最优解包含其子问题的最优解。</p>
<p><strong>通俗理解</strong>：大问题的最优答案，可以从小问题的最优答案拼出来。</p>
<blockquote>
<p>🧩 <strong>类比</strong>：拼图的最终画面，是由每一小块拼图的"最佳位置"组成的。每一步放对了，最后就拼对了。</p>
</blockquote>
<h3>贪心 vs 动态规划</h3>
<p>这是面试必考题！很多同学分不清这两者。让我们用一个表格来对比：</p>
<table>
<thead>
<tr>
<th>特性</th>
<th>贪心算法</th>
<th>动态规划</th>
</tr>
</thead>
<tbody><tr>
<td>选择策略</td>
<td>每步选当前最优，不回头</td>
<td>考虑所有可能，选全局最优</td>
</tr>
<tr>
<td>子问题</td>
<td>不需要求解所有子问题</td>
<td>需要求解所有子问题</td>
</tr>
<tr>
<td>结果保证</td>
<td>不一定得到最优解</td>
<td>保证得到最优解</td>
</tr>
<tr>
<td>时间复杂度</td>
<td>通常较低</td>
<td>通常较高</td>
</tr>
<tr>
<td>空间复杂度</td>
<td>通常较低</td>
<td>通常需要存储子问题解</td>
</tr>
<tr>
<td>适用条件</td>
<td>贪心选择性质 + 最优子结构</td>
<td>重叠子问题 + 最优子结构</td>
</tr>
</tbody></table>
<p><strong>一句话总结</strong>：贪心是"一根筋"，动态规划是"深思熟虑"。能用贪心解决的问题，通常也能用动态规划解决，但贪心更快更简单——前提是你得确认贪心是对的！</p>

<h2>代码演示</h2>
<p>让我们通过三个经典例子来理解贪心算法的精髓。</p>
<h3>示例1：活动选择问题</h3>
<p>这是贪心算法的"Hello World"。</p>
<p><strong>问题描述</strong>：你一天有很多活动可以参加，每个活动有开始时间和结束时间。同一时间只能参加一个活动。如何选择才能参加<strong>最多数量</strong>的活动？</p>
<p><strong>贪心策略</strong>：每次都选择<strong>结束时间最早</strong>的、且与已选活动不冲突的活动。</p>
<p><strong>为什么选结束最早的？</strong> 因为这样给后面的活动留出了最多的时间空间！</p>
<pre><code class="language-python"># 活动选择问题 - 贪心算法
def activity_selection(activities):
    """
    活动选择问题 - 贪心算法
    参数: activities - 列表，每个元素为(活动名, 开始时间, 结束时间)
    返回: 选择的活动列表
    """
    # 按结束时间排序（贪心策略：优先选结束早的）
    sorted_activities = sorted(activities, key=lambda x: x[2])

    selected = []
    last_end_time = 0

    for name, start, end in sorted_activities:
        # 如果当前活动的开始时间 &gt;= 上一个选中活动的结束时间，不冲突
        if start &gt;= last_end_time:
            selected.append((name, start, end))
            last_end_time = end  # 更新最后结束时间

    return selected


# 测试
activities = [
    ("上课", 9, 10), ("打球", 9, 11), ("看书", 10, 12),
    ("写代码", 11, 13), ("看电影", 12, 14), ("开会", 13, 15),
    ("健身", 14, 16), ("约会", 15, 17), ("睡觉", 16, 18),
]

selected = activity_selection(activities)
print(f"最优选择（共 {len(selected)} 个活动）:")
for name, start, end in selected:
    print(f"  {name}: {start}:00 - {end}:00")
</code></pre>
<p><strong>运行结果</strong>：</p>
<pre><code>最优选择（共 5 个活动）:
  上课: 9:00 - 10:00
  看书: 10:00 - 12:00
  看电影: 12:00 - 14:00
  健身: 14:00 - 16:00
  睡觉: 16:00 - 18:00
</code></pre>
<p>看到没？从9个活动中，我们成功选出了5个互不冲突的活动。这就是贪心的力量！</p>
<h3>示例2：找零钱问题——贪心什么时候对，什么时候错</h3>
<p>这是理解贪心局限性的最佳例子。</p>
<p><strong>问题描述</strong>：给定一些面值的硬币，找零时如何用最少数量的硬币？</p>
<pre><code class="language-python"># 找零钱问题 - 贪心 vs 动态规划
def coin_change_greedy(coins, amount):
    """
    找零钱 - 贪心算法
    参数: coins - 硬币面值列表（从大到小排序）
          amount - 需要找零的金额
    返回: 使用的硬币列表
    """
    coins_sorted = sorted(coins, reverse=True)
    result = []
    remaining = amount

    for coin in coins_sorted:
        while remaining &gt;= coin:
            result.append(coin)
            remaining -= coin

    if remaining &gt; 0:
        return None  # 无法完全找零
    return result


def coin_change_dp(coins, amount):
    """
    找零钱 - 动态规划（求最优解）
    返回: 最少硬币数量，如果无法找零返回-1
    """
    dp = [float('inf')] * (amount + 1)
    dp[0] = 0

    for i in range(1, amount + 1):
        for coin in coins:
            if i &gt;= coin:
                dp[i] = min(dp[i], dp[i - coin] + 1)

    return dp[amount] if dp[amount] != float('inf') else -1


# 场景1：美国硬币系统 [25, 10, 5, 1]，找63分
print("=== 美国硬币系统（贪心正确）===")
us_coins = [25, 10, 5, 1]
result1 = coin_change_greedy(us_coins, 63)
dp_result1 = coin_change_dp(us_coins, 63)
print(f"贪心: {result1} (共 {len(result1)} 个)")
print(f"最优: {dp_result1} 个")

# 场景2：特殊硬币系统 [4, 3, 1]，找6分
print("\n=== 特殊硬币系统（贪心失败）===")
special_coins = [4, 3, 1]
result2 = coin_change_greedy(special_coins, 6)
dp_result2 = coin_change_dp(special_coins, 6)
print(f"贪心: {result2} (共 {len(result2)} 个)")
print(f"最优: {dp_result2} 个")
</code></pre>
<p><strong>运行结果</strong>：</p>
<pre><code>=== 美国硬币系统（贪心正确）===
贪心: [25, 25, 10, 1, 1, 1] (共 6 个)
最优: 6 个

=== 特殊硬币系统（贪心失败）===
贪心: [4, 1, 1] (共 3 个)
最优: 2 个
</code></pre>
<p><strong>关键发现</strong>：</p>
<ul>
<li>在美国硬币系统下，贪心是<strong>正确的</strong>！</li>
<li>但在特殊硬币系统下，贪心<strong>翻车了</strong>！贪心选了4+1+1（3个），最优是3+3（2个）</li>
</ul>
<p>这说明：<strong>贪心算法的正确性取决于问题的结构</strong>。</p>
<h3>示例3：分数背包问题</h3>
<p>这是贪心算法大显身手的场景。</p>
<p><strong>问题描述</strong>：你有一个50kg容量的背包，有黄金、白银、铜等物品可以选择。与0-1背包不同，你可以拿走物品的一部分（比如只拿一半黄金）。目标是最大化总价值。</p>
<p><strong>贪心策略</strong>：按<strong>单位价值</strong>（价值/重量）从高到低排序，优先拿"性价比"最高的。</p>
<pre><code class="language-python"># 分数背包问题 - 贪心算法
class Item:
    """物品类"""
    def __init__(self, name, value, weight):
        self.name = name
        self.value = value
        self.weight = weight
        self.ratio = value / weight  # 单位价值


def fractional_knapsack(items, capacity):
    """
    分数背包问题 - 贪心算法
    返回: (总价值, 选择的物品记录)
    """
    # 按单位价值从高到低排序
    items_sorted = sorted(items, key=lambda x: x.ratio, reverse=True)

    total_value = 0
    remaining = capacity

    for item in items_sorted:
        if remaining &lt;= 0:
            break

        if item.weight &lt;= remaining:
            # 全部拿走
            total_value += item.value
            remaining -= item.weight
            print(f"  拿走全部 {item.name} ({item.weight}kg)")
        else:
            # 只拿一部分
            fraction = remaining / item.weight
            total_value += item.value * fraction
            print(f"  拿走 {item.name} 的 {fraction*100:.0f}% ({remaining}kg)")
            remaining = 0

    return total_value


# 测试
items = [
    Item("黄金", 60, 10),   # 单位价值: 6.0
    Item("白银", 100, 20),  # 单位价值: 5.0
    Item("铜",   120, 30),  # 单位价值: 4.0
    Item("铁",   50, 15),   # 单位价值: 3.33
]

capacity = 50  # 背包容量50kg
print(f"背包容量: {capacity}kg")
total = fractional_knapsack(items, capacity)
print(f"总价值: {total}")
</code></pre>
<p><strong>运行结果</strong>：</p>
<pre><code>背包容量: 50kg
  拿走全部 黄金 (10kg)
  拿走全部 白银 (20kg)
  拿走 铜 的 67% (20kg)
总价值: 240.0
</code></pre>
<p>看到没？贪心算法先拿最"值钱"的黄金，再拿白银，最后拿了一部分铜。背包满了，总价值240，这就是最优解！</p>

<h2>深入理解</h2>
<h3>贪心算法的"思考过程"</h3>
<p>当你面对一个问题时，怎么判断能不能用贪心算法？遵循以下步骤：</p>
<p><strong>第一步：确定贪心策略</strong></p>
<p>问自己：<strong>"在当前状态下，哪个选择看起来最诱人？"</strong></p>
<ul>
<li>活动选择 → 选结束最早的</li>
<li>分数背包 → 选单位价值最高的</li>
<li>找零钱 → 选面值最大的</li>
</ul>
<p><strong>第二步：验证贪心选择性质</strong></p>
<p>问自己：<strong>"如果我做了这个选择，剩下的问题是不是和原来一样的子问题？"</strong></p>
<p>如果能，那就可以继续贪心地处理子问题。</p>
<p><strong>第三步：证明最优子结构</strong></p>
<p>问自己：<strong>"问题的最优解，是否包含子问题的最优解？"</strong></p>
<p>如果以上三个问题的答案都是"是"，那恭喜你可以放心用贪心了！</p>

<h2>实际应用</h2>
<p>贪心算法在现实世界中无处不在，只是你可能没注意到。</p>
<h3>应用场景1：霍夫曼编码（数据压缩）</h3>
<p>你每天都在用贪心算法的成果——<strong>ZIP压缩文件</strong>和<strong>JPEG图片</strong>都用到了霍夫曼编码。</p>
<p>霍夫曼编码是一种数据压缩技术，核心思想是：</p>
<ul>
<li>出现频率高的字符，用短编码表示</li>
<li>出现频率低的字符，用长编码表示</li>
</ul>
<p>构建霍夫曼树的过程就是一个贪心过程：</p>
<ol>
<li>统计每个字符出现的频率</li>
<li>每次选频率<strong>最低</strong>的两个节点，合并成一个新节点</li>
<li>重复直到只剩一棵树</li>
</ol>
<p>这就是典型的贪心策略！每次合并最小的，最终得到最优的编码方案。</p>
<h3>应用场景2：Dijkstra最短路径算法</h3>
<p>当你在高德地图或Google地图上查路线时，背后运行的<strong>Dijkstra算法</strong>也是贪心的。</p>
<p>Dijkstra算法的核心：</p>
<ul>
<li>每次从"已知最短距离"的节点中，选距离<strong>最近</strong>的节点</li>
<li>用这个节点去更新邻居的距离</li>
<li>重复直到找到终点</li>
</ul>
<p>这个"每次选最近的"就是贪心选择！当然，Dijkstra的贪心能工作，是因为所有边的权重都是非负的。如果有负权边，就得用Bellman-Ford算法了。</p>
<hr />
<h2>常见误区</h2>
<p>⚠️ <strong>注意这些坑！</strong></p>
<h3>误区1：贪心总是能得到最优解</h3>
<p>这是新手最容易犯的错！贪心算法<strong>只在特定条件下</strong>才得到最优解。</p>
<p>❌ <strong>错误想法</strong>：找零钱问题用贪心一定最优<br />✅ <strong>正确理解</strong>：只有当硬币系统满足特定性质时，贪心才正确。一般情况下应该用动态规划。</p>
<blockquote>
<p>🚨 记住：贪心是"直觉型选手"，不是所有问题都能靠直觉解决。</p>
</blockquote>
<h3>误区2：贪心策略可以随便选</h3>
<p>很多初学者认为，只要选一种"看起来合理"的贪心策略就行。但<strong>不同的贪心策略可能导致完全不同的结果</strong>。</p>
<p>以活动选择为例：</p>
<ul>
<li>按"开始时间最早"贪心 ❌ 不行</li>
<li>按"持续时间最短"贪心 ❌ 不行</li>
<li>按"结束时间最早"贪心 ✅ 才行</li>
</ul>
<p><strong>贪心策略的选择必须经过严格的分析或证明！</strong></p>
<h3>误区3：贪心和动态规划是"二选一"的关系</h3>
<p>其实对于同一个问题，贪心和动态规划可以互补使用：</p>
<ul>
<li>先用贪心的思路<strong>快速估算</strong>一个下界或上界</li>
<li>再用动态规划<strong>精确求解</strong></li>
</ul>
<p>比如在0-1背包问题中，可以先用分数背包的贪心策略算一个上界，用来剪枝，加速动态规划。</p>
<hr />
<h2>动手实践</h2>
<p>现在轮到你动手了！从简单到复杂，三道练习题等着你来挑战。</p>
<h3>练习1：区间合并 ⭐</h3>
<p>给定一组区间，合并所有重叠的区间。</p>
<p>例如：<code>[1, 3]</code> 和 <code>[2, 6]</code> 重叠，合并为 <code>[1, 6]</code>。</p>
<p><strong>输入</strong>：<code>[[1, 3], [2, 6], [8, 10], [15, 18]]</code></p>
<p><strong>输出</strong>：<code>[[1, 6], [8, 10], [15, 18]]</code></p>
<p><strong>要求</strong>：</p>
<ul>
<li>实现一个函数 <code>merge_intervals(intervals)</code></li>
<li>结果按起点排序，区间互不重叠</li>
</ul>

💡 提示（点击展开）<p>先按区间起点排序，然后依次遍历。如果当前区间与上一个区间重叠，就合并（取终点的最大值）；否则开始一个新的区间。</p>
<h3>练习2：任务调度 ⭐⭐</h3>
<p>你有一系列任务，每个任务有截止时间和完成所需时间。一次只能做一个任务。问最多能完成多少个任务？</p>
<p><strong>输入</strong>：<code>[(4, 2), (3, 1), (5, 2), (2, 1), (6, 3)]</code></p>
<p><strong>输出</strong>：4个任务</p>
<p><strong>要求</strong>：</p>
<ul>
<li>实现一个函数 <code>max_tasks(tasks)</code></li>
<li>返回最多能完成的任务数</li>
</ul>

💡 提示（点击展开）<p>按截止时间从早到晚排序，依次尝试完成任务。如果当前时间 + 任务耗时 ≤ 截止时间，就接这个任务。</p>
<h3>练习3：最少站台问题 ⭐⭐⭐</h3>
<p>火车站给出所有火车的到达和离开时间，问最少需要多少个站台？</p>
<p><strong>输入</strong>：</p>
<pre><code>arrivals = [900, 940, 950, 1100, 1500, 1800]
departures = [910, 1200, 1120, 1130, 1900, 2000]
</code></pre>
<p><strong>输出</strong>：3</p>
<p><strong>要求</strong>：</p>
<ul>
<li>实现一个函数 <code>min_platforms(arrivals, departures)</code></li>
<li>返回最少需要的站台数</li>
</ul>

💡 提示（点击展开）<p>把到达和离开事件分别排序，用双指针模拟时间线。遇到到达就+1站台，遇到离开就-1站台。记录过程中的最大值。</p>
<hr />
<h2>小结与思考</h2>
<h3>本节要点</h3>
<ol>
<li><strong>贪心算法的核心</strong>：每一步都做当前最优选择，不回溯</li>
<li><strong>适用条件</strong>：问题必须具备贪心选择性质和最优子结构</li>
<li><strong>贪心不是万能的</strong>：不能保证所有问题都得到最优解</li>
<li><strong>经典应用</strong>：活动选择、分数背包、霍夫曼编码、Dijkstra算法</li>
<li><strong>与动态规划的关系</strong>：贪心是"短视版"的动态规划，更快但不保证最优</li>
</ol>
<h3>思考题</h3>
<ol>
<li><p>找零钱问题中，如果硬币面值是 [1, 5, 10, 25]（美元体系），贪心为什么是正确的？能不能证明？</p>
</li>
<li><p>如果让你在"贪心但可能不是最优"和"动态规划但保证最优"之间做选择，你会考虑哪些因素？</p>
</li>
<li><p>活动选择问题中，如果每个活动还有一个"价值"，目标是最大化总价值（而不是数量），贪心算法还适用吗？为什么？</p>
</li>
</ol>
