# 挑战 2：设计代码审查指标仪表板

## 难度：⭐⭐⭐

### 背景
你的团队想要建立一个代码审查指标仪表板，帮助监控和改进审查流程。你需要设计一个系统来收集、分析和展示关键的审查指标。

### 任务要求
创建一个 `ReviewMetricsDashboard` 类，实现以下功能：

1. **数据收集**：
   - 添加审查会话数据（审查时间、代码行数、评论数、发现的缺陷数、总缺陷数）
   - 支持批量导入历史数据

2. **指标计算**：
   - 平均审查时间（小时）
   - 每行代码的平均评论数
   - 整体缺陷检测率（发现的缺陷 / 总缺陷）
   - 审查效率分数（综合考虑缺陷检测率、评论质量和审查速度）

3. **趋势分析**：
   - 分析最近5次审查的趋势（改善/恶化/稳定）
   - 识别异常值（如特别长的审查时间）

4. **报告生成**：
   - 生成包含所有指标的详细报告
   - 提供针对性的改进建议
   - 支持JSON和文本格式输出

### 输入输出示例
```python
dashboard = ReviewMetricsDashboard()

# 添加审查数据
review_data = {
    "review_time_hours": 2.5,
    "lines_of_code": 300,
    "comments_count": 12,
    "defects_found": 4,
    "total_defects": 6
}
dashboard.add_review(review_data)

# 生成报告
report = dashboard.generate_report()
print(report["efficiency_score"])  # 应输出0-100的分数
print(report["recommendations"])   # 应包含具体的改进建议
```

### 评分标准
- **基础功能**（40分）：正确实现数据收集和基本指标计算
- **高级功能**（30分）：实现趋势分析和异常检测
- **用户体验**（20分）：报告清晰、建议实用
- **代码质量**（10分）：代码结构良好、有适当的错误处理

### 提示
- 使用 `statistics` 模块进行统计计算
- 效率分数可以基于加权公式：缺陷检测率(50%) + 评论质量(30%) + 审查速度(20%)
- 趋势分析可以通过比较连续审查的指标变化来实现
- 考虑边界情况（如空数据、零除法等）

### 扩展思考
- 如何将这个系统集成到现有的CI/CD流程中？
- 哪些指标可能会被误用或导致不良激励？
- 如何确保指标的公平性和准确性？
