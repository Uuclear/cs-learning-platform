# 挑战2：实现完整的API文档自动化系统

## 背景
你正在为一个微服务架构的项目工作，该项目包含多个RESTful API服务。目前每个服务都有基本的代码注释，但缺乏统一的、可交互的API文档。团队希望实现一个完整的文档即代码解决方案，能够自动生成符合OpenAPI 3.0标准的规范，并支持图表可视化。

## 任务
实现一个端到端的API文档自动化系统：

### 步骤1：API规范生成器
- 创建一个Python工具，能够从代码注释和装饰器中提取API信息
- 支持自动生成OpenAPI 3.0规范（JSON和YAML格式）
- 实现完整的请求/响应体定义、参数验证规则和安全方案
- 支持组件复用（schemas、security schemes等）

### 步骤2：集成Mermaid可视化
- 分析API端点之间的关系，自动生成Mermaid流程图
- 创建API调用序列图，展示典型的工作流程
- 将生成的图表嵌入到API文档中，提供可视化上下文

### 步骤3：文档站点生成
- 集成Swagger UI或Redoc，创建交互式API文档站点
- 实现静态站点生成，支持离线访问
- 添加搜索功能和导航结构

### 步骤4：CI/CD自动化
- 创建GitHub Actions工作流，在每次推送时：
  - 自动扫描代码并生成最新的OpenAPI规范
  - 验证规范的有效性
  - 生成并部署交互式文档站点
  - 运行文档质量检查（覆盖率、链接有效性等）

### 步骤5：开发者体验优化
- 创建命令行工具，简化文档生成过程
- 添加文档模板和最佳实践指南
- 实现文档差异检测，当API发生变化时提醒开发者更新文档

## 技术要求
- 使用Python标准库，避免外部依赖（除非必要）
- 所有代码必须包含中文注释
- 支持配置文件自定义生成选项
- 提供完整的错误处理和验证

## 提交要求
- 完整的Python实现代码
- 示例配置文件和输出示例
- GitHub Actions配置文件
- 使用说明文档（README.md）

## 评估标准
- 功能完整性（是否覆盖所有要求）
- 代码质量和可维护性
- 文档生成质量（规范准确性、可视化效果）
- 自动化程度和开发者体验
- 可扩展性和适应性（能否用于其他项目）
