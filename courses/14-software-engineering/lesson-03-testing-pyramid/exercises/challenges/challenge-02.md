# 挑战2：设计微服务测试策略

## 背景
你的团队正在开发一个电商微服务架构，包含以下服务：
- **用户服务**：用户管理、认证授权
- **商品服务**：商品目录、库存管理  
- **订单服务**：订单处理、支付集成
- **通知服务**：邮件、短信通知

各服务通过REST API和消息队列进行通信。系统需要支持高并发和99.9%的可用性。

## 任务
为这个微服务架构设计完整的测试策略，涵盖所有测试层级。

### 具体要求
1. **单元测试策略**：
   - 每个服务的核心业务逻辑单元测试方案
   - 如何处理服务间的依赖（mock策略）
   - 测试数据工厂设计

2. **集成测试策略**：
   - 服务间API契约测试
   - 数据库集成测试
   - 消息队列集成测试
   - 第三方服务（支付网关）集成测试

3. **E2E测试策略**：
   - 关键用户旅程识别（如：浏览商品→下单→支付→收货）
   - 测试环境管理（如何避免测试数据污染）
   - 并行测试和分片策略

4. **特殊考虑**：
   - 如何测试分布式事务的一致性
   - 性能测试和负载测试策略
   - 故障注入测试（Chaos Engineering）

### 交付物
- 测试策略文档（包括各层级的测试范围和工具选择）
- 核心服务的测试代码示例
- CI/CD流水线中的测试执行流程图
- 测试指标监控方案

### 评估标准
- 策略的完整性和覆盖度
- 对微服务特有挑战的解决方案
- 成本效益的平衡
- 可扩展性和维护性
