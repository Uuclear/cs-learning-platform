# 挑战1：实现购物车系统

## 背景
你需要使用TDD的方式实现一个简单的购物车系统，支持添加商品、移除商品、计算总价等功能。

## 需求
购物车系统需要支持以下功能：

1. **添加商品**：可以向购物车中添加商品，每个商品包含名称、价格和数量
2. **移除商品**：可以从购物车中移除指定的商品
3. **清空购物车**：可以清空购物车中的所有商品
4. **计算总价**：可以计算购物车中所有商品的总价
5. **获取商品列表**：可以获取购物车中的所有商品信息
6. **检查商品是否存在**：可以检查指定商品是否在购物车中

## 商品数据结构
每个商品应该包含以下属性：
- `name` (str): 商品名称
- `price` (float): 商品单价
- `quantity` (int): 商品数量

## TDD要求
1. 使用红-绿-重构循环逐步实现功能
2. 为每个功能编写完整的测试用例
3. 考虑边界情况（如负价格、零数量等）
4. 使用适当的测试替身处理可能的外部依赖

## 提示
- 先从最简单的功能开始（如添加商品和计算总价）
- 考虑商品的唯一性（相同名称的商品应该如何处理？）
- 思考如何设计API才能让使用起来最自然

## 验收标准
- 所有测试通过
- 代码具有良好的可读性和可维护性
- 处理了合理的边界情况
- 遵循TDD的开发流程
