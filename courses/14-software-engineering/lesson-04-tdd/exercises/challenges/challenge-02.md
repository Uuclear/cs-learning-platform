# 挑战2：重构现有代码并添加TDD测试

## 背景
你接手了一个现有的简单计算器代码，但没有测试覆盖。你的任务是使用TDD的方式为这个计算器添加完整的测试，并在必要时进行重构。

## 现有代码
```python
class Calculator:
    def add(self, a, b):
        return a + b
    
    def subtract(self, a, b):
        return a - b
    
    def multiply(self, a, b):
        return a * b
    
    def divide(self, a, b):
        return a / b
```

## 问题分析
1. 现有代码缺少错误处理（如除零异常）
2. 没有类型提示和文档字符串
3. 缺少对边界情况的处理
4. 完全没有测试覆盖

## TDD任务
1. **不要直接修改现有代码**，而是先为现有功能编写测试
2. 运行测试，确认它们会因为缺少错误处理而失败
3. 逐步添加错误处理和改进代码，让测试通过
4. 添加更多测试用例覆盖边界情况
5. 在保持测试通过的前提下重构代码，改善设计

## 具体要求
1. 添加完整的类型提示（Type Hints）
2. 为每个方法添加详细的文档字符串
3. 实现适当的错误处理（特别是除零异常）
4. 考虑浮点数精度问题
5. 添加幂运算和平方根功能（使用TDD方式）
6. 确保100%的分支覆盖率

## 验收标准
- 所有测试通过，包括新增的功能测试
- 代码具有完整的类型提示和文档
- 错误处理完善，用户体验良好
- 遵循了TDD的红-绿-重构循环
- 重构后的代码更加清晰和可维护

## 提示
- 使用`unittest`或`pytest`框架
- 考虑使用参数化测试来减少重复代码
- 可以使用`coverage`工具检查测试覆盖率
- 记住：先写测试，再改实现！
