# 数据类型与运算符

## 开场白：不同容器装不同东西

想象一下你的厨房：有大碗、小碗、杯子、勺子，每种容器都有特定的用途和容量。在C语言的世界里，**数据类型**就相当于这些不同的容器，它们决定了你能存储什么类型的数据以及占用多少内存空间。

如果你试图用一个小茶杯装下一整锅汤，会发生什么？溢出！同样，在编程中，如果你试图将一个超出范围的数值存入某个数据类型中，也会出现问题。这就是为什么理解数据类型如此重要——它帮助我们选择合适的"容器"来存储我们的数据。

今天，我们将一起探索C语言中的基本数据类型和各种运算符，让你成为真正的"数据容器管理员"！

## 核心概念

### 基本数据类型

C语言提供了几种基本的数据类型，每种都有其特定的用途：

#### 整数类型 (Integer Types)
- **`char`**: 字符类型，通常占用1字节（8位），可以存储-128到127（有符号）或0到255（无符号）
- **`short`**: 短整型，通常占用2字节，范围-32,768到32,767
- **`int`**: 整型，通常占用4字节，范围-2,147,483,648到2,147,483,647
- **`long`**: 长整型，通常占用4或8字节（取决于系统）
- **`long long`**: 更长的整型，通常占用8字节

#### 浮点类型 (Floating-Point Types)
- **`float`**: 单精度浮点数，占用4字节，约6-7位有效数字
- **`double`**: 双精度浮点数，占用8字节，约15-16位有效数字
- **`long double`**: 扩展精度浮点数，占用10或16字节（取决于系统）

#### 字符类型
- **`char`**: 除了存储小整数，还可以存储单个字符，如 `'A'`, `'1'`, `'$'`

### sizeof 运算符

`sizeof` 是一个特殊的运算符，用于获取数据类型或变量占用的字节数。这对于理解内存布局和进行内存管理非常重要。

```c
sizeof(int)    // 返回 int 类型的字节数
sizeof(variable) // 返回变量占用的字节数
```

### 运算符分类

C语言提供了丰富的运算符：

#### 算术运算符
- `+` 加法
- `-` 减法  
- `*` 乘法
- `/` 除法
- `%` 取模（求余数）

#### 关系运算符
- `==` 等于
- `!=` 不等于
- `<` 小于
- `>` 大于
- `<=` 小于等于
- `>=` 大于等于

#### 逻辑运算符
- `&&` 逻辑与
- `||` 逻辑或
- `!` 逻辑非

## 深入理解

### 整数溢出

当一个整数超出其数据类型的表示范围时，就会发生溢出。对于有符号整数，这会导致未定义行为；对于无符号整数，会发生回绕（wrap around）。

例如，一个8位有符号整数的最大值是127，如果加1变成128，实际上会变成-128（在二进制补码表示中）。

### 浮点精度丢失

浮点数使用有限的位数来表示实数，因此无法精确表示所有实数。例如，0.1在二进制中是一个无限循环小数，所以用`float`或`double`存储时会有微小的误差。

```c
float a = 0.1f;
float b = 0.2f;
float c = a + b; // c 可能不完全等于 0.3
```

### 类型提升规则

在表达式中，C语言会自动进行类型提升：
1. 如果任一操作数是`long double`，另一个也被转换为`long double`
2. 否则，如果任一操作数是`double`，另一个也被转换为`double`
3. 否则，如果任一操作数是`float`，另一个也被转换为`float`
4. 否则，对整数操作数应用整数提升规则

### 有符号 vs 无符号

有符号类型可以表示正数、负数和零，而无符号类型只能表示零和正数。无符号类型的最大值是有符号类型最大值的两倍左右。

```c
unsigned int x = -1; // x 实际上等于 4294967295（在32位系统上）
```

## 实际应用

### 数据大小计算

了解数据类型大小对于内存管理和性能优化至关重要：

```c
// 计算数组总大小
int arr[100];
size_t total_size = sizeof(arr); // 100 * sizeof(int)

// 动态内存分配
int *ptr = malloc(100 * sizeof(int));
```

### 位运算基础

虽然位运算符不在本课主要范围内，但了解它们与数据类型的关系很重要：

```c
unsigned char flags = 0;
flags |= 1 << 3; // 设置第3位
if (flags & (1 << 3)) { // 检查第3位是否设置
    // 执行某些操作
}
```

### 类型转换的实际用途

- **避免整数除法截断**：`(float)a / b` 而不是 `a / b`
- **处理不同精度的计算**：在需要高精度时使用`double`
- **内存效率**：在存储大量数据时，选择合适大小的类型

## 常见误区

### 整数除法截断

在C语言中，两个整数相除的结果仍然是整数，小数部分会被截断：

```c
int a = 5, b = 2;
int result = a / b; // result = 2，不是 2.5
```

解决方法：至少将一个操作数转换为浮点数：

```c
float result = (float)a / b; // result = 2.5
```

### 浮点数比较

由于浮点精度问题，直接比较两个浮点数是否相等通常是不可靠的：

```c
// 错误的做法
if (a == b) { /* ... */ }

// 正确的做法
#define EPSILON 1e-6
if (fabs(a - b) < EPSILON) { /* ... */ }
```

### 无符号整数的陷阱

无符号整数永远不会为负，这可能导致意外行为：

```c
unsigned int i = 0;
i--; // i 现在等于 UINT_MAX（通常是 4294967295）
```

### sizeof 的常见误解

`sizeof` 返回的是字节数，不是元素个数：

```c
int arr[10];
size_t size = sizeof(arr);     // 返回 40（假设 int 是4字节）
size_t count = sizeof(arr) / sizeof(arr[0]); // 返回 10（正确的方法）
```
