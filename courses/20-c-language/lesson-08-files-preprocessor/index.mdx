# 文件操作与预处理器

## 开场白：C语言的"仓库管理员"

想象一下，你的程序就像一个繁忙的办公室，内存是你的办公桌——空间有限但访问超快。而硬盘就像是巨大的仓库，可以存储海量数据但需要专门的管理员来帮你搬运。

在C语言的世界里，**文件操作**就是你的仓库管理员，负责帮你从硬盘这个大仓库中读取数据或者把数据存入仓库。而**预处理器**则像是你的智能秘书，在你正式开始工作前，就已经根据你的要求对文档进行了整理、筛选和格式化。

今天，我们就来学习如何成为一名优秀的"仓库管理员"和"智能秘书"的指挥官！

## 核心概念

### 文件操作基础

在C语言中，文件操作主要通过以下几个标准库函数来完成：

- `fopen()`: 打开文件
- `fclose()`: 关闭文件  
- `fread()`: 读取文件内容
- `fwrite()`: 写入文件内容
- `fgets()`: 读取一行文本
- `fprintf()`: 格式化写入文件

#### 文件指针 FILE*

所有文件操作都围绕着 `FILE*` 这个神秘的指针展开。它实际上是一个结构体指针，包含了文件的各种状态信息：

```c
FILE *fp;  // 声明一个文件指针
```

#### 打开模式

`fopen()` 函数的第二个参数决定了文件的打开模式：

- `"r"`: 只读文本模式
- `"w"`: 只写文本模式（会清空原文件）
- `"a"`: 追加文本模式
- `"rb"`: 只读二进制模式
- `"wb"`: 只写二进制模式
- `"ab"`: 追加二进制模式

### 预处理器魔法

C语言的预处理器在编译之前就对源代码进行处理，主要指令包括：

- `#include`: 包含头文件
- `#define`: 定义宏
- `#ifdef` / `#ifndef` / `#endif`: 条件编译
- `#if` / `#elif` / `#else`: 条件编译的高级形式

## 代码演示

### 示例1：文本文件读写

让我们先看一个简单的文本文件读写例子：

```c
#include <stdio.h>
#include <stdlib.h>

int main() {
    FILE *fp;
    char buffer[256];
    
    // 写入文件
    fp = fopen("example.txt", "w");
    if (fp == NULL) {
        printf("无法创建文件！\n");
        return 1;
    }
    fprintf(fp, "Hello, C语言文件操作！\n");
    fprintf(fp, "这是第二行内容。\n");
    fclose(fp);
    
    // 读取文件
    fp = fopen("example.txt", "r");
    if (fp == NULL) {
        printf("无法打开文件！\n");
        return 1;
    }
    
    while (fgets(buffer, sizeof(buffer), fp) != NULL) {
        printf("读取到: %s", buffer);
    }
    
    fclose(fp);
    return 0;
}
```

### 示例2：二进制文件操作

二进制文件操作更适合处理非文本数据，比如图片、音频或结构体数据：

```c
#include <stdio.h>
#include <stdlib.h>

typedef struct {
    int id;
    char name[50];
    float score;
} Student;

int main() {
    FILE *fp;
    Student students[] = {
        {1, "张三", 95.5},
        {2, "李四", 87.0},
        {3, "王五", 92.3}
    };
    Student read_student;
    int i;
    
    // 写入二进制文件
    fp = fopen("students.dat", "wb");
    if (fp == NULL) {
        perror("无法创建二进制文件");
        return 1;
    }
    
    for (i = 0; i < 3; i++) {
        fwrite(&students[i], sizeof(Student), 1, fp);
    }
    fclose(fp);
    
    // 读取二进制文件
    fp = fopen("students.dat", "rb");
    if (fp == NULL) {
        perror("无法打开二进制文件");
        return 1;
    }
    
    printf("读取的学生信息：\n");
    while (fread(&read_student, sizeof(Student), 1, fp) == 1) {
        printf("ID: %d, 姓名: %s, 分数: %.1f\n", 
               read_student.id, read_student.name, read_student.score);
    }
    
    fclose(fp);
    return 0;
}
```

### 示例3：预处理器宏和条件编译

```c
#include <stdio.h>

// 简单的宏定义
#define PI 3.1415926
#define MAX(a, b) ((a) > (b) ? (a) : (b))
#define SQUARE(x) ((x) * (x))

// 调试宏
#ifdef DEBUG
    #define DEBUG_PRINT(fmt, ...) printf("DEBUG: " fmt "\n", ##__VA_ARGS__)
#else
    #define DEBUG_PRINT(fmt, ...)
#endif

// 平台相关的条件编译
#ifdef _WIN32
    #define CLEAR_SCREEN "cls"
#elif defined(__linux__) || defined(__APPLE__)
    #define CLEAR_SCREEN "clear"
#endif

int main() {
    int x = 5, y = 3;
    
    printf("PI = %.6f\n", PI);
    printf("MAX(%d, %d) = %d\n", x, y, MAX(x, y));
    printf("%d 的平方 = %d\n", x, SQUARE(x));
    
    // 调试输出（只有在定义了DEBUG时才会显示）
    DEBUG_PRINT("程序开始执行，x=%d, y=%d", x, y);
    
    // 清屏命令（根据平台选择）
    system(CLEAR_SCREEN);
    
    return 0;
}
```

## 深入理解

### 宏展开的奥秘

宏不是函数！这是一个非常重要的概念。宏在预处理阶段就被替换了，没有任何类型检查和作用域的概念。

考虑这个危险的例子：
```c
#define SQUARE(x) x * x  // 错误的定义！

int result = SQUARE(3 + 2);  // 展开后变成: 3 + 2 * 3 + 2 = 11!
```

正确的定义应该是：
```c
#define SQUARE(x) ((x) * (x))  // 正确的定义
```

### 条件编译的强大功能

条件编译让你能够编写跨平台的代码，或者根据不同的编译配置启用/禁用特定功能：

```c
// 版本控制
#if VERSION >= 2
    // 新版本特有的功能
    new_feature();
#else
    // 旧版本的兼容实现
    old_feature();
#endif

// 功能开关
#ifdef ENABLE_LOGGING
    log_message("用户登录成功");
#endif
```

### 文件缓冲区

C语言的文件操作默认使用缓冲区来提高性能。这意味着写入的数据可能不会立即写入磁盘，而是先存储在内存缓冲区中。

- `fflush(fp)`: 强制刷新缓冲区
- `setbuf(fp, NULL)`: 禁用缓冲区（不推荐，除非特殊需求）

## 实际应用

### 配置文件解析

让我们创建一个简单的配置文件解析器：

```c
// config.txt 内容:
// DATABASE_HOST=localhost
// DATABASE_PORT=5432
// DEBUG_MODE=true

#include <stdio.h>
#include <string.h>

int parse_config(const char *filename) {
    FILE *fp = fopen(filename, "r");
    if (!fp) return -1;
    
    char line[256];
    while (fgets(line, sizeof(line), fp)) {
        // 跳过注释和空行
        if (line[0] == '#' || line[0] == '\n') continue;
        
        // 查找等号
        char *equal = strchr(line, '=');
        if (!equal) continue;
        
        *equal = '\0';  // 分割键和值
        char *key = line;
        char *value = equal + 1;
        
        // 去除末尾的换行符
        value[strcspn(value, "\n")] = 0;
        
        printf("配置项: %s = %s\n", key, value);
        
        // 在实际应用中，这里会设置相应的全局变量或配置结构体
    }
    
    fclose(fp);
    return 0;
}
```

```c
#ifndef LOGGER_H

#define LOG_LEVEL_DEBUG 0

void log_message(const char *level, const char *file, int line, const char *fmt, ...);

```

## 常见误区

### 误区1：宏就是函数

如前所述，宏只是文本替换，没有函数的安全性和类型检查。避免在宏中使用有副作用的表达式：

```c
#define INC(x) ((x)++)  // 危险！可能导致未定义行为

int a = 5;
int b = INC(a + 1);  // 展开为: ((a + 1)++) — 这是非法的！
```

### 误区2：忘记关闭文件

忘记调用 `fclose()` 会导致资源泄漏，甚至数据丢失（因为缓冲区没有刷新）：

```c
// 错误示例
FILE *fp = fopen("data.txt", "w");
fprintf(fp, "重要数据");
// 忘记 fclose(fp) — 数据可能丢失！
```

### 误区3：不检查文件操作的返回值

文件操作可能失败（权限不足、磁盘满、文件不存在等），必须检查返回值：

```c
// 错误示例
FILE *fp = fopen("important.txt", "r");
char buffer[100];
fgets(buffer, 100, fp);  // 如果fp为NULL，程序会崩溃！
```

### 误区4：混淆文本模式和二进制模式

在Windows系统中，文本模式会自动转换换行符（`\n` ↔ `\r\n`），而二进制模式不会。处理非文本文件时必须使用二进制模式：

```c
// 处理图片文件
FILE *img = fopen("photo.jpg", "rb");  // 必须用 rb！
```

## 小结与思考

今天我们学习了C语言中两个强大的特性：文件操作和预处理器。

**文件操作**让你的程序能够持久化数据，与外部世界交互。记住：
- 始终检查文件操作的返回值
- 不要忘记关闭文件
- 根据数据类型选择正确的打开模式
- 理解缓冲区的工作原理

**预处理器**提供了编译时的代码生成和条件编译能力。记住：
- 宏不是函数，只是文本替换
- 使用括号保护宏参数
- 条件编译是编写跨平台代码的利器
- 谨慎使用复杂的宏，保持代码可读性
