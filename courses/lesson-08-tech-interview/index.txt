<h3>Big-O分析基础</h3>
<p>时间复杂度和空间复杂度是衡量算法效率的核心指标：</p>
<ul>
<li><strong>O(1)</strong>：常数时间 - 最理想的情况</li>
<li><strong>O(log n)</strong>：对数时间 - 高效的分治算法</li>
<li><strong>O(n)</strong>：线性时间 - 单次遍历</li>
<li><strong>O(n log n)</strong>：线性对数时间 - 高效排序算法</li>
<li><strong>O(n²)</strong>：平方时间 - 嵌套循环，需要优化</li>
<li><strong>O(2ⁿ)</strong>：指数时间 - 通常不可接受</li>
</ul>

from typing import List, Optional

def two_sum_sorted(arr: List[int], target: int) -&gt; Optional[tuple]:
    """
    两数之和（已排序数组）- 双指针模式
    时间复杂度: O(n)
    空间复杂度: O(1)
    
    :param arr: 已排序的整数数组
    :param target: 目标和
    :return: 找到的两个数的索引元组，未找到返回None
    """
    left, right = 0, len(arr) - 1
    
    while left &lt; right:
        current_sum = arr[left] + arr[right]
        if current_sum == target:
            return (left, right)
        elif current_sum &lt; target:
            left += 1  # 和太小，移动左指针
        else:
            right -= 1  # 和太大，移动右指针
    
    return None

def max_subarray_sum(arr: List[int]) -&gt; int:
    """
    最大子数组和 - 滑动窗口模式
    时间复杂度: O(n)
    空间复杂度: O(1)
    
    :param arr: 整数数组
    :return: 最大子数组和
    """
    if not arr:
        return 0
    
    max_sum = current_sum = arr[0]
    
    for i in range(1, len(arr)):
        # 要么扩展当前子数组，要么重新开始
        current_sum = max(arr[i], current_sum + arr[i])
        max_sum = max(max_sum, current_sum)
    
    return max_sum

def has_cycle(head) -&gt; bool:
    """
    检测链表环 - 快慢指针模式
    时间复杂度: O(n)
    空间复杂度: O(1)
    
    :param head: 链表头节点
    :return: 是否存在环
    """
    if not head or not head.next:
        return False
    
    slow = fast = head
    
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        
        if slow == fast:
            return True  # 快慢指针相遇，存在环
    
    return False

import time
import random
from typing import List, Callable
import matplotlib.pyplot as plt

def linear_search(arr: List[int], target: int) -&gt; int:
    """线性搜索 - O(n)"""
    for i, val in enumerate(arr):
        if val == target:
            return i
    return -1

def bubble_sort(arr: List[int]) -&gt; List[int]:
    """冒泡排序 - O(n²)"""
    arr = arr.copy()
    n = len(arr)
    for i in range(n):
        for j in range(0, n - i - 1):
            if arr[j] &gt; arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
    return arr

def merge_sort(arr: List[int]) -&gt; List[int]:
    """归并排序 - O(n log n)"""
    if len(arr) &lt;= 1:
        return arr
    
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])
    
    return merge(left, right)

def merge(left: List[int], right: List[int]) -&gt; List[int]:
    """归并两个已排序数组"""
    result = []
    i = j = 0
    
    while i &lt; len(left) and j &lt; len(right):
        if left[i] &lt;= right[j]:
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1
    
    result.extend(left[i:])
    result.extend(right[j:])
    return result

def time_algorithm(algorithm: Callable, data: List[int], *args) -&gt; float:
    """测量算法执行时间"""
    start_time = time.time()
    algorithm(data, *args) if args else algorithm(data)
    end_time = time.time()
    return end_time - start_time

def compare_complexities():
    """比较不同算法的时间复杂度"""
    sizes = [100, 500, 1000, 2000]
    linear_times = []
    quadratic_times = []
    nlogn_times = []
    
    for size in sizes:
        # 生成随机数据
        data = [random.randint(1, 1000) for _ in range(size)]
        target = random.choice(data)
        
        # 测试线性搜索
        linear_time = time_algorithm(linear_search, data, target)
        linear_times.append(linear_time)
        
        # 测试冒泡排序（仅小数据集）
        if size &lt;= 1000:
            quadratic_time = time_algorithm(bubble_sort, data)
            quadratic_times.append(quadratic_time)
        else:
            quadratic_times.append(0)  # 跳过大数组的冒泡排序
        
        # 测试归并排序
        nlogn_time = time_algorithm(merge_sort, data)
        nlogn_times.append(nlogn_time)
    
    print("算法复杂度比较结果:")
    print("数据规模\t线性搜索(O(n))\t冒泡排序(O(n²))\t归并排序(O(n log n))")
    for i, size in enumerate(sizes):
        quad_time = f"{quadratic_times[i]:.4f}" if quadratic_times[i] &gt; 0 else "跳过"
        print(f"{size}\t\t{linear_times[i]:.4f}\t\t{quad_time}\t\t{nlogn_times[i]:.4f}")

if __name__ == "__main__":
    compare_complexities()
    print("\n注意: 实际面试中应优先选择高效算法!")

<h2>深入理解</h2>
<h3>编码面试策略</h3>
<h4>解题流程</h4>
<ol>
<li><strong>理解问题</strong>：重复问题，确认边界条件和输入输出</li>
<li><strong>举例说明</strong>：用具体例子验证理解</li>
<li><strong>暴力解法</strong>：先提出简单但正确的解决方案</li>
<li><strong>优化思路</strong>：讨论时间和空间复杂度的改进空间</li>
<li><strong>实现代码</strong>：编写清晰、有注释的代码</li>
<li><strong>测试验证</strong>：用边界情况测试代码</li>
</ol>
<h4>常见编码模式</h4>
<ul>
<li><strong>双指针</strong>：适用于排序数组、字符串操作</li>
<li><strong>滑动窗口</strong>：适用于子数组/子字符串问题</li>
<li><strong>BFS/DFS</strong>：适用于图和树的遍历</li>
<li><strong>动态规划</strong>：适用于最优子结构问题</li>
<li><strong>哈希表</strong>：适用于快速查找和去重</li>
</ul>
<h3>系统设计框架</h3>
<h4>CAP定理的理解</h4>
<ul>
<li><strong>一致性</strong>（Consistency）：所有节点看到相同数据</li>
<li><strong>可用性</strong>（Availability）：每个请求都能得到响应</li>
<li><strong>分区容忍性</strong>（Partition tolerance）：网络分区时系统继续运行</li>
</ul>
<p>在分布式系统中，只能同时满足其中两个特性。</p>
<h4>数据库选型原则</h4>
<ul>
<li><strong>关系型数据库</strong>（MySQL, PostgreSQL）：强一致性、复杂查询、事务支持</li>
<li><strong>NoSQL数据库</strong>（MongoDB, Cassandra）：高扩展性、灵活模式、最终一致性</li>
<li><strong>键值存储</strong>（Redis, DynamoDB）：简单查询、高性能、缓存场景</li>
</ul>
<h3>行为面试STAR方法</h3>
<p><strong>S</strong>ituation（情境）：描述具体的背景和情况<br /><strong>T</strong>ask（任务）：说明你面临的任务或挑战<br /><strong>A</strong>ction（行动）：详细描述你采取的具体行动<br /><strong>R</strong>esult（结果）：量化结果和学到的经验</p>
<p>例如："在我上一份工作中（S），我们需要在两周内修复一个导致用户流失的关键bug（T）。我组织了紧急会议，分配了调试任务，并实施了自动化测试（A）。最终我们在3天内解决了问题，用户流失率下降了80%（R）。"</p>

<li><strong>积极互动</strong>：主动询问澄清问题，寻求反馈</li>
</ul>

<h2>常见误区</h2>
<h3>❌ 编码面试误区</h3>
<ul>
<li><strong>直接开始编码</strong>：没有充分理解问题就动手</li>
<li><strong>忽视边界条件</strong>：只测试正常情况，忽略空值、极端值</li>
<li><strong>过度优化</strong>：在没有正确解法前就追求最优解</li>
<li><strong>沉默编码</strong>：不与面试官沟通思考过程</li>
</ul>
<h3>❌ 系统设计误区</h3>
<ul>
<li><strong>过早优化</strong>：一开始就设计复杂的分布式系统</li>
<li><strong>忽视非功能需求</strong>：只关注功能，忽略性能、安全、监控</li>
<li><strong>技术堆砌</strong>：盲目使用新技术而不考虑适用性</li>
<li><strong>缺乏权衡分析</strong>：不讨论不同方案的优缺点</li>
</ul>
<h3>❌ 行为面试误区</h3>
<ul>
<li><strong>泛泛而谈</strong>：回答过于笼统，缺乏具体细节</li>
<li><strong>推卸责任</strong>：将失败归咎于他人或外部因素</li>
<li><strong>负面情绪</strong>：抱怨前雇主或同事</li>
<li><strong>准备不足</strong>：对常见问题没有准备答案</li>
</ul>
