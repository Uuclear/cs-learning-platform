队列：先进先出的排队法则

想象一下，你去网红奶茶店排队买奶茶。先到的同学先拿到奶茶，后到的只能在后面苦苦等待。如果你硬要插队，后面所有人都会瞪着你——这就是队列！

开场白

同学们，今天我们聊聊计算机科学中最"公平"的数据结构——队列（Queue）。

先问大家几个问题：

  1. 你有没有遇到过打印机"卡死"的情况？ 明明你发了打印任务，但前面还有10个人的文档在排队，只能慢慢等。为什么打印机不随机选一个打印呢？

  2. 你去银行取号排队，叫号系统是怎么工作的？ 为什么A001号一定比A002号先被叫到？

  3. 你在电影院买票，队伍越来越长，这时突然有人从队伍中间插队会发生什么？ 后面的人会不会暴打他？（开个玩笑）

如果你对这些场景有共鸣，那今天的课就是为你准备的！队列这个数据结构，本质上就是在模拟现实生活中的"排队"。

核心概念

什么是队列？

队列（Queue） 是一种先进先出（FIFO, First In First Out）的线性数据结构。

用一句大白话说：谁先来，谁先走。

就像排队买票：

  • 队尾（Rear/Tail）：新来的人排在最后面
  • 队首（Front/Head）：最先来的人最先离开

队列的四大基本操作

操作 英文名 说明 类比
enqueue 入队 把元素放到队尾 排队站到队伍最后
dequeue 出队 从队首取出元素 窗口办完事离开
front/peek 查看队首 看看队首是谁 看看下一个叫几号
isEmpty 判断空 队列是否为空 看看队伍里有没有人

队列 vs 栈：一对好兄弟

我们上节课学了栈（Stack，后进先出），今天学队列（Queue，先进先出）。这俩就像一对性格完全相反的好兄弟：

特性 栈（Stack） 队列（Queue）
原则 LIFO（后进先出） FIFO（先进先出）
比喻 一摞盘子 排队买票
进出端 同一端进出 一端进，另一端出
典型场景 函数调用、撤销操作 任务调度、消息队列

一句话记住区别：

  • 栈 = "吃火锅时最后放进去的食材最先捞出来"
  • 队列 = "排队做核酸，先来的先做"

示例1: 队列的基本实现

# 队列的基本实现
# 使用Python列表模拟队列（简单但效率不是最优）

class Queue:
    """
    队列类 - 先进先出的数据结构
    就像排队买奶茶，先到的先拿到
    """
    def __init__(self):
        # 用列表存储队列元素
        self.items = []

    def enqueue(self, item):
        """
        入队操作：把元素加到队尾
        就像新来的人站到队伍最后面
        """
        self.items.append(item)
        print(f"  [入队] '{item}' → 队列: {self.items}")

    def dequeue(self):
        """
        出队操作：从队首取出并移除元素
        就像队伍最前面的人办完事离开
        如果队列为空，返回 None
        """
        if self.is_empty():
            print("  [警告] 队列为空，无法出队！")
            return None
        item = self.items.pop(0)  # 从列表开头移除
        print(f"  [出队] '{item}' ← 队列: {self.items}")
        return item

    def front(self):
        """
        查看队首元素（不移除）
        就像看看下一个叫几号
        """
        if self.is_empty():
            return None
        return self.items[0]

    def is_empty(self):
        """判断队列是否为空"""
        return len(self.items) == 0

    def size(self):
        """返回队列中的元素个数"""
        return len(self.items)

    def __str__(self):
        """打印队列状态"""
        if self.is_empty():
            return "队列: [空]"
        return f"队列: {self.items} (共{self.size()}人)"


def demonstrate_basic_queue():
    """演示队列的基本操作"""
    print("=== 队列基本操作演示 ===\n")

    # 创建一个队列（模拟银行叫号）
    queue = Queue()
    print(f"初始状态: {queue}\n")

    # 顾客来排队
    print("--- 顾客陆续到来 ---")
    queue.enqueue("顾客A")
    queue.enqueue("顾客B")
    queue.enqueue("顾客C")
    print(f"当前状态: {queue}\n")

    # 查看下一个
    print("--- 查看下一个 ---")
    print(f"  队首是: {queue.front()}\n")

    # 顾客办理业务（出队）
    print("--- 顾客办理业务 ---")
    queue.dequeue()
    queue.dequeue()
    print(f"当前状态: {queue}\n")

    # 又来了新顾客
    print("--- 新顾客到来 ---")
    queue.enqueue("顾客D")
    queue.enqueue("顾客E")
    print(f"最终状态: {queue}")


# 运行演示
if __name__ == "__main__":
    demonstrate_basic_queue()

# 预期输出:
# === 队列基本操作演示 ===
#
# 初始状态: 队列: [空]
#
# --- 顾客陆续到来 ---
#   [入队] '顾客A' → 队列: ['顾客A']
#   [入队] '顾客B' → 队列: ['顾客A', '顾客B']
#   [入队] '顾客C' → 队列: ['顾客A', '顾客B', '顾客C']
# 当前状态: 队列: ['顾客A', '顾客B', '顾客C'] (共3人)
#
# --- 查看下一个 ---
#   队首是: 顾客A
#
# --- 顾客办理业务 ---
#   [出队] '顾客A' ← 队列: ['顾客B', '顾客C']
#   [出队] '顾客B' ← 队列: ['顾客C']
# 当前状态: 队列: ['顾客C'] (共1人)
#
# --- 新顾客到来 ---
#   [入队] '顾客D' → 队列: ['顾客C', '顾客D']
#   [入队] '顾客E' → 队列: ['顾客C', '顾客D', '顾客E']
# 最终状态: 队列: ['顾客C', '顾客D', '顾客E'] (共3人)

代码解释：

  • enqueue 用 append() 把元素加到队尾，时间复杂度 O(1)
  • dequeue 用 pop(0) 从队首移除元素，时间复杂度 O(n)（因为列表后面的元素要往前挪）
  • 这就是为什么用普通列表实现的队列不是最优的——出队操作太慢了！

示例2: 循环队列的实现

# 循环队列的实现
# 使用固定大小数组 + 首尾指针，解决普通队列的空间浪费问题

class CircularQueue:
    """
    循环队列 - 像旋转木马一样循环利用空间
    核心思想：数组首尾相连，形成一个"环"
    """
    def __init__(self, capacity):
        # 固定大小的数组
        self.capacity = capacity
        self.queue = [None] * capacity
        # 队首指针和队尾指针
        self.front = 0    # 指向队首元素
        self.rear = 0     # 指向下一个可插入的位置
        self.size = 0     # 当前元素个数

    def enqueue(self, item):
        """入队：把元素放到 rear 位置"""
        if self.is_full():
            print(f"  [警告] 队列已满（容量{self.capacity}），无法入队！")
            return False

        self.queue[self.rear] = item
        # rear 指针向前移动，到末尾后回到开头（取模运算）
        self.rear = (self.rear + 1) % self.capacity
        self.size += 1
        print(f"  [入队] '{item}' → {self}")
        return True

    def dequeue(self):
        """出队：取出 front 位置的元素"""
        if self.is_empty():
            print("  [警告] 队列为空，无法出队！")
            return None

        item = self.queue[self.front]
        self.queue[self.front] = None  # 清空
        # front 指针向前移动，到末尾后回到开头
        self.front = (self.front + 1) % self.capacity
        self.size -= 1
        print(f"  [出队] '{item}' ← {self}")
        return item

    def is_full(self):
        """队列是否已满"""
        return self.size == self.capacity

    def is_empty(self):
        """队列是否为空"""
        return self.size == 0

    def front_item(self):
        """查看队首元素"""
        if self.is_empty():
            return None
        return self.queue[self.front]

    def __str__(self):
        return f"循环队列: {self.queue}, 大小: {self.size}/{self.capacity}"


def demonstrate_circular_queue():
    """演示循环队列的工作原理"""
    print("=== 循环队列演示 ===\n")

    # 创建一个容量为4的循环队列（模拟停车场）
    parking = CircularQueue(capacity=4)
    print(f"初始状态: {parking}\n")

    # 车辆陆续进入
    print("--- 车辆进入停车场 ---")
    parking.enqueue("宝马")
    parking.enqueue("奔驰")
    parking.enqueue("特斯拉")
    parking.enqueue("比亚迪")
    print(f"\n停车场满了: {parking}\n")

    # 尝试再停一辆
    print("--- 试图再停车 ---")
    parking.enqueue("五菱宏光")  # 满了！
    print()

    # 车辆离开
    print("--- 车辆离开 ---")
    parking.dequeue()  # 宝马走了
    parking.dequeue()  # 奔驰走了
    print(f"\n空出两个车位: {parking}\n")

    # 新车进来 - 注意看 rear 指针回到了开头！
    print("--- 新车停入（复用空位） ---")
    parking.enqueue("小鹏")
    parking.enqueue("蔚来")
    print(f"\n最终状态: {parking}")
    print("  ↑ 注意：小鹏和蔚来停在了数组开头的空位，空间被复用了！")


# 运行演示
if __name__ == "__main__":
    demonstrate_circular_queue()

#
# 空出两个车位: 循环队列: [None, None, '特斯拉', '比亚迪'], 大小: 2/4
#
# --- 新车停入（复用空位） ---
#   [入队] '小鹏' → 循环队列: ['小鹏', None, '特斯拉', '比亚迪'], 大小: 3/4
#   [入队] '蔚来' → 循环队列: ['小鹏', '蔚来', '特斯拉', '比亚迪'], 大小: 4/4
#
# 最终状态: 循环队列: ['小鹏', '蔚来', '特斯拉', '比亚迪'], 大小: 4/4
#   ↑ 注意：小鹏和蔚来停在了数组开头的空位，空间被复用了！

代码解释：

  • 循环队列用固定大小数组实现，通过 (rear + 1) % capacity 让指针"转圈"
  • 当 rear 到达数组末尾时，取模运算让它回到索引 0，就像旋转木马一样循环利用空间
  • 这样就避免了普通队列 pop(0) 的 O(n) 问题——入队和出队都是 O(1)！

示例3: BFS算法演示（队列的经典应用）

# BFS（广度优先搜索）演示
# 队列的经典应用：像水波纹一样逐层扩散

from collections import deque


def bfs(graph, start):
    """
    广度优先搜索（Breadth-First Search）
    就像往池塘扔石头，水波纹一圈一圈往外扩散

    参数:
        graph: 邻接表表示的图 {节点: [相邻节点列表]}
        start: 起始节点

    返回:
        访问顺序列表
    """
    # 记录已访问的节点
    visited = set()
    # 结果列表（访问顺序）
    result = []
    # 用双端队列作为 BFS 的队列
    queue = deque([start])
    visited.add(start)

    print(f"  初始: 队列 = {list(queue)}, 已访问 = {visited}")

    while queue:
        # 从队首取出一个节点
        node = queue.popleft()
        result.append(node)
        print(f"  处理节点 '{node}' → 结果: {result}")

        # 把这个节点的所有未访问邻居加入队列
        for neighbor in graph.get(node, []):
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)
                print(f"    邻居 '{neighbor}' 入队 → 队列: {list(queue)}")

        print(f"  本轮结束: 队列 = {list(queue)}\n")

    return result


def demonstrate_bfs():
    """演示BFS算法"""
    print("=== BFS（广度优先搜索）演示 ===\n")

    # 构建一个社交网络图（谁认识谁）
    # 用邻接表表示
    social_network = {
        "张三": ["李四", "王五", "赵六"],
        "李四": ["张三", "王五", "孙七"],
        "王五": ["张三", "李四", "周八"],
        "赵六": ["张三", "周八"],
        "孙七": ["李四", "郑九"],
        "周八": ["王五", "赵六", "郑九"],
        "郑九": ["孙七", "周八"],
    }

    print("社交网络关系:")
    for person, friends in social_network.items():
        print(f"  {person} 认识: {', '.join(friends)}")
    print()

    # 从"张三"开始BFS（看看他的朋友圈有多大）
    print("--- 从张三开始扩散 ---\n")
    visit_order = bfs(social_network, "张三")

    print("=" * 50)
    print(f"BFS访问顺序: {' → '.join(visit_order)}")
    print(f"总共访问了 {len(visit_order)} 个人")
    print("这就是'六度分隔理论'的基础——通过队列，我们逐层探索社交网络！")

代码解释：

  • BFS 用队列实现：从起点开始，逐层访问邻居
  • 每次处理一个节点时，把它的所有未访问邻居加入队列
  • 因为是队列，先加入的邻居先被处理，保证了"逐层"扩散
  • deque 是 Python 标准库中的双端队列，popleft() 是 O(1) 操作

深入理解

为什么普通队列的 dequeue 是 O(n)？

回到示例1的代码，dequeue 用了 self.items.pop(0)。这有什么问题？

想象一个班级50个人排队：

  • 最前面的同学走了（出队）
  • 剩下的49个人都要往前挪一步！

这就是 pop(0) 的代价——每出队一次，后面的 n-1 个元素都要移动。时间复杂度 O(n)。

解决方案有三种：

  1. 循环队列（示例2）：用取模运算让指针转圈，O(1) 出队
  2. 双端队列 deque（示例3）：Python 标准库，底层是双向链表，O(1) 出队
  3. 链表实现：用头尾两个指针，直接从头节点断开，O(1) 出队

循环队列的巧妙之处

循环队列的关键是 (rear + 1) % capacity 这行代码。

初始:  [_, _, _, _, _]   front=0, rear=0
入队A: [A, _, _, _, _]   front=0, rear=1
入队B: [A, B, _, _, _]   front=0, rear=2
出队:  [_, B, _, _, _]   front=1, rear=2
出队:  [_, _, _, _, _]   front=2, rear=2
入队C: [_, _, C, _, _]   front=2, rear=3
入队D: [_, _, C, D, _]   front=2, rear=4
入队E: [_, _, C, D, E]   front=2, rear=5→0（取模！）
入队F: [F, _, C, D, E]   front=2, rear=1

看！rear 指针到了末尾后，通过取模运算回到了开头，空间被循环利用。

队列的时间复杂度

实现方式 enqueue dequeue front 空间
列表（pop(0)） O(1) O(n) O(1) O(n)
循环队列 O(1) O(1) O(1) O(n) 固定容量
链表 O(1) O(1) O(1) O(n)
deque O(1) O(1) O(1) O(n)

队列的变种

1. 双端队列（Deque, Double-Ended Queue）

两端都可以进出，既可以从前面进，也可以从后面进。

 ← 入队 | A | B | C | D | 出队 →
        ↑               ↑
      前端(front)     后端(rear)

Deque 就像一个两头都能通的管道，灵活性极高。Python 的 collections.deque 就是它。

2. 优先队列（Priority Queue）

不按 FIFO 出队，而是按"优先级"出队——优先级最高的先出。

入队: 急诊病人(优先级10) → 感冒病人(优先级3) → 骨折病人(优先级8)
出队: 急诊病人(10) → 骨折病人(8) → 感冒病人(3)

医院急诊室就是优先队列！不是谁先来谁先看，而是谁病得重谁先看。

优先队列通常用**堆（Heap）**来实现，这是我们后面要讲的内容。

实际应用

应用场景1: 操作系统任务调度

操作系统用队列管理各种任务：

CPU调度队列: [进程A] → [进程B] → [进程C] → ...
               ↓         ↓         ↓
          时间片用完  时间片用完  执行中...
          回到队尾    回到队尾

每个进程获得一个时间片（比如10ms），用完后回到队列尾部等待下一次调度。这就是时间片轮转调度算法，保证每个进程都有机会执行。

应用场景2: 消息队列（中间件）

分布式系统中广泛使用的消息队列：

生产者(Producer) → [消息队列] → 消费者(Consumer)

比如 RabbitMQ、Kafka、Redis 的 List 结构都可以作为消息队列。订单系统产生订单消息，放到队列里；库存系统从队列中消费消息，一个一个处理。队列起到了解耦和削峰的作用。

常见误区

⚠️ 注意这些坑！

误区1: "用列表的 append + pop(0) 就是好的队列"

错误理解：

Python 的 list.append() 和 list.pop(0) 不就是入队和出队吗？完美！

真相：
pop(0) 的时间复杂度是 O(n)！因为每次出队后，后面所有元素都要往前挪。如果队列有1000个元素，每次出队都要移动999个元素。

✅ 正确做法：
用 collections.deque，它的 append() 和 popleft() 都是 O(1)：

from collections import deque
q = deque()
q.append("A")       # 入队 O(1)
q.popleft()         # 出队 O(1) ← 注意是 popleft() 不是 pop()

误区2: "循环队列一定能存 capacity 个元素"

错误理解：

循环队列容量为5，就能存5个元素！

真相：
循环队列通常需要"浪费"一个位置来区分"满"和"空"的状态。当 rear 的下一个是 front 时就认为是满了，实际只能存 capacity - 1 个元素。

✅ 正确做法：
如果想存 n 个元素，创建循环队列时容量要设为 n + 1。或者用一个额外的 size 变量记录元素个数（示例2的写法）。

误区3: "队列只能在队尾入队，队首出队"

错误理解：

队列就是 FIFO，只能一头进一头出。

真相：
这只是"标准队列"的规则。实际应用中有很多变种：

  • 双端队列（Deque）：两端都可以进出
  • 优先队列：按优先级出队，不按顺序
  • 阻塞队列：为空时等待，满了也等待（常用于多线程）

✅ 正确做法：
根据实际需求选择合适的数据结构。如果需要两端操作，用 Deque；如果需要优先级，用优先队列。

练习1: 实现链表队列 ⭐

描述：
用链表实现一个队列，确保 enqueue 和 dequeue 都是 O(1) 时间复杂度。

要求：

  • 实现 enqueue(item) 方法
  • 实现 dequeue() 方法
  • 实现 front() 方法
  • 实现 is_empty() 和 size() 方法
  • 所有操作必须是 O(1) 时间复杂度

提示：
用一个 head 指针指向队首，一个 tail 指针指向队尾。入队时更新 tail，出队时更新 head。

💡 提示（点击展开）
  • 创建 Node 类，每个节点有 val 和 next 指针
  • head 指向第一个节点，tail 指向最后一个节点
  • 入队：在 tail 后面加新节点，更新 tail
  • 出队：移除 head 指向的节点，head 移到下一个节点
  • 注意处理队列从空变非空、从非空变空的边界情况

练习2: 最近请求次数 ⭐⭐

描述：
实现一个 RecentCounter 类，计算过去 3000 毫秒内的请求数。

要求：

  • 实现 ping(timestamp) 方法：添加一个时间戳为 timestamp 的请求
  • 返回过去 3000 毫秒内（包括当前请求）的请求总数
  • 保证每次调用的 timestamp 严格递增

示例：

counter = RecentCounter()
counter.ping(1)     # 返回 1（请求: [1]）
counter.ping(100)   # 返回 2（请求: [1, 100]）
counter.ping(3001)  # 返回 3（请求: [1, 100, 3001]）
counter.ping(3002)  # 返回 3（请求: [100, 3001, 3002]，1已过期）

提示：
用队列存储请求时间戳，每次 ping 时移除超过 3000ms 的旧请求。

💡 提示（点击展开）
  • 队列中存储所有请求的时间戳
  • 每次新请求进来，先入队
  • 然后检查队首：如果队首时间戳 < 当前时间戳 - 3000，就出队
  • 循环直到队首在有效范围内
  • 队列的长度就是有效请求数

练习3: 用两个栈实现队列 ⭐⭐⭐

描述：
用两个栈实现一个队列，支持 enqueue 和 dequeue 操作。

要求：

  • 只能用栈（后进先出）的操作：push、pop、peek、isEmpty
  • 实现 enqueue(item) 和 dequeue() 方法
  • 均摊时间复杂度：dequeue 应该是 O(1)

示例：

q = MyQueue()
q.enqueue(1)
q.enqueue(2)
q.dequeue()  # 返回 1
q.enqueue(3)
q.dequeue()  # 返回 2
q.dequeue()  # 返回 3

提示：
用两个栈 in_stack 和 out_stack。入队时 push 到 in_stack。出队时如果 out_stack 为空，把 in_stack 的所有元素倒过来 push 到 out_stack。

💡 提示（点击展开）
  • in_stack 负责接收所有入队操作
  • out_stack 负责提供出队元素
  • 当 out_stack 为空时，将 in_stack 的元素全部弹出并 push 到 out_stack
  • 这样 out_stack 的栈顶就是最早的元素（FIFO）
  • 每个元素最多被 push 和 pop 各两次（一次进 in_stack，一次转移到 out_stack），所以均摊 O(1)
  • 这是一个经典的面试问题！

小结与思考

本节要点

  1. 队列 = 先进先出（FIFO） —— 就像排队，先来的人先走

  2. 四大基本操作 —— enqueue（入队）、dequeue（出队）、front（查看队首）、isEmpty（判断空）

  3. 实现方式 —— 列表简单但出队慢，循环队列和 deque 更高效

  4. 经典应用 —— BFS 搜索、任务调度、消息队列、打印机队列

  5. 变种结构 —— 双端队列（Deque）和优先队列（Priority Queue）扩展了队列的能力
