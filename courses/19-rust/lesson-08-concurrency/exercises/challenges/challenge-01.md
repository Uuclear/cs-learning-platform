# 挑战 1: 并行文件处理器

## 背景
你正在开发一个文件处理系统，需要同时处理多个大文件以提高效率。

## 要求
创建一个程序，能够：
1. 接收一个文件路径列表作为输入
2. 为每个文件创建一个独立的工作线程
3. 每个线程计算文件的行数、单词数和字符数（类似 `wc` 命令）
4. 使用通道将结果发送回主线程
5. 主线程收集所有结果并显示汇总统计

## 提示
- 使用 `std::fs::read_to_string` 读取文件内容
- 使用 `mpsc` 通道在线程间传递结果
- 考虑错误处理：如果某个文件不存在或无法读取怎么办？
- 确保程序在所有线程完成后正确退出

## 扩展挑战（可选）
- 添加文件大小限制，超过限制的文件跳过处理
- 实现并发限制，最多同时处理 N 个文件（使用信号量或线程池概念）

## 验证
你的程序应该能够正确处理以下情况：
- 空文件
- 不存在的文件
- 包含中文字符的文件
- 多个大文件同时处理
