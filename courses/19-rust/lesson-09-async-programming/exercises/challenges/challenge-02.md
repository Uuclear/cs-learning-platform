# 挑战 2：异步文件处理管道

## 目标
构建一个异步文件处理管道，能够并发读取多个文件，进行数据处理，并将结果写入输出文件。

## 要求
1. 实现 `process_files` 函数，接受输入文件路径列表和输出目录
2. 并发读取所有输入文件（使用 `tokio::fs::read_to_string`）
3. 对每个文件内容进行简单的文本处理（例如：统计行数、单词数、字符数）
4. 将处理结果以 JSON 格式写入对应的输出文件
5. 实现优雅的错误处理和资源清理

## 具体实现步骤
1. 创建数据结构来表示文件统计信息
2. 实现文件读取和处理的异步函数
3. 使用 `tokio::spawn` 或 `futures::future::join_all` 来并发处理
4. 实现结果写入函数
5. 在主函数中组合所有组件

## 示例数据结构
```rust
#[derive(Serialize)]
struct FileStats {
    filename: String,
    lines: usize,
    words: usize,
    characters: usize,
    processing_time_ms: u64,
}
```

## 提示
- 使用 `serde` 进行 JSON 序列化
- 考虑使用 `tokio::time::Instant` 测量处理时间
- 注意文件路径的正确处理
- 考虑大文件的内存使用情况

## 扩展挑战（可选）
- 添加文件类型检测，对不同类型的文件应用不同的处理逻辑
- 实现流式处理，避免一次性将大文件全部加载到内存
- 添加进度报告功能，显示处理进度

## 验证标准
- 能够正确处理各种文件（空文件、大文件、不存在的文件等）
- 并发处理确实提高了性能
- 输出格式正确，包含所有要求的信息
- 错误处理完善，程序不会因为单个文件错误而崩溃
