# 挑战 1：并发 HTTP 客户端

## 目标
实现一个并发的 HTTP 客户端，能够同时向多个 URL 发起请求，并处理各种可能的错误情况。

## 要求
1. 创建一个 `fetch_urls` 函数，接受 URL 列表作为参数
2. 使用 `tokio::spawn` 为每个 URL 创建独立的任务
3. 实现适当的错误处理，区分网络错误、HTTP 状态码错误等
4. 添加超时控制，单个请求不应超过 5 秒
5. 返回包含成功结果和失败信息的结构化数据

## 提示
- 使用 `reqwest` 库进行 HTTP 请求
- 考虑使用 `tokio::time::timeout` 实现超时
- 使用 `Result` 类型来处理成功和失败的情况
- 考虑如何优雅地处理部分成功、部分失败的情况

## 扩展挑战（可选）
- 添加重试机制，对临时性错误自动重试
- 实现速率限制，避免对同一域名发起过多并发请求
- 添加请求日志记录功能

## 验证标准
- 能够正确处理正常响应和各种错误情况
- 并发执行确实比顺序执行更快
- 超时机制工作正常
- 代码结构清晰，错误处理完善
