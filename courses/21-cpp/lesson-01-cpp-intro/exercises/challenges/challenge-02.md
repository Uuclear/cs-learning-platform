# 挑战 2：C 与 C++ 风格对比分析

## 目标
深入理解 C 和 C++ 在实际编程中的差异，通过实现相同功能的两种风格代码来体验各自的优缺点。

## 任务
1. **选择一个实用功能**（如文件操作、字符串处理、动态数组等）
2. **分别用 C 风格和 C++ 风格实现**
3. **编写对比分析报告**

## 具体要求

### C 风格实现要求：
- 使用标准 C 库函数（如 stdio.h, string.h, stdlib.h 等）
- 手动管理内存（malloc/free）
- 使用指针进行操作
- 处理错误返回值

### C++ 风格实现要求：
- 使用 C++ 标准库（如 iostream, string, vector 等）
- 利用 RAII 自动管理资源
- 使用引用或智能指针
- 考虑使用异常处理

### 对比分析报告内容：
1. **代码长度对比**：哪种风格的代码更简洁？
2. **安全性对比**：哪种风格更容易避免常见错误（如缓冲区溢出、内存泄漏）？
3. **性能对比**：在你的实现中，哪种风格可能更快？
4. **可读性对比**：哪种风格的代码更容易理解和维护？
5. **适用场景**：在什么情况下你会选择其中一种风格？

## 示例功能选择（任选其一）：
- **动态字符串拼接**：实现一个函数，能够将多个字符串连接成一个
- **文件内容统计**：统计文本文件中的行数、单词数和字符数
- **简单数据结构**：实现一个动态数组，支持添加、删除和访问元素

## 提交内容
- C 风格实现代码（.c 文件）
- C++ 风格实现代码（.cpp 文件）
- 对比分析报告（README.md 或注释形式）

## 评分标准
- 功能正确性（40%）
- 代码质量（30%）
- 分析深度（30%）
