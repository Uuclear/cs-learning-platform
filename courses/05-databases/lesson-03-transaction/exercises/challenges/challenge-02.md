# 构建银行转账系统

**难度**: ⭐⭐⭐⭐

## 描述
构建一个支持高并发转账的银行系统，处理多个账户之间的资金转移，确保在任何情况下都能保持数据一致性，并正确处理各种异常情况。

## 要求
1. **数据库设计**:
   - 账户表（账户ID、账户名、余额、版本号、创建时间）
   - 交易记录表（交易ID、转出账户ID、转入账户ID、金额、状态、创建时间、完成时间）

2. **核心功能**:
   - 实现转账功能，支持任意两个账户间的转账
   - 确保转账的原子性：要么全部成功，要么全部失败
   - 处理并发转账场景，避免死锁和数据不一致

3. **高级特性**:
   - 实现两种转账策略：乐观锁和悲观锁
   - 支持批量转账（一个事务中处理多个转账）
   - 实现转账超时机制（长时间未完成的转账自动回滚）

4. **错误处理与恢复**:
   - 处理账户不存在、余额不足等业务异常
   - 实现死锁检测和自动重试
   - 支持事务回滚后的状态恢复

5. **监控与审计**:
   - 记录所有转账操作的详细日志
   - 提供账户余额查询和交易历史查询功能
   - 实现资产守恒验证（所有账户余额总和不变）

## 提示
- 使用BEGIN/COMMIT/ROLLBACK控制事务边界
- 对于悲观锁，总是按账户ID升序锁定资源以避免死锁
- 对于乐观锁，使用版本号字段并在UPDATE时验证版本
- 考虑使用保存点（SAVEPOINT）实现部分回滚
- SQLite的timeout参数可以帮助处理锁等待

## 测试场景
- 创建4个账户，初始余额分别为5000、3000、7000、4000元
- 同时执行10个并发转账操作（不同账户对之间）
- 验证最终所有账户余额总和仍为19000元
- 验证没有出现负余额或数据不一致

## 扩展挑战（可选）
