# 实现秒杀系统的库存管理

**难度**: ⭐⭐⭐

## 描述
秒杀系统是电商网站中的常见场景，需要在极短时间内处理大量并发请求，同时确保库存不会超卖。你的任务是实现一个高并发的秒杀库存管理系统。

## 要求
1. **数据库设计**:
   - 创建商品表（包含商品ID、名称、总库存、已售数量）
   - 创建订单表（包含订单ID、商品ID、用户ID、购买数量、状态、创建时间）

2. **核心功能**:
   - 实现秒杀下单功能，确保不会超卖
   - 支持高并发访问（至少100个并发线程）
   - 使用适当的事务隔离级别和锁策略

3. **性能要求**:
   - 在100个并发请求下，系统应该能在5秒内完成所有请求
   - 不允许出现负库存或超卖情况

4. **错误处理**:
   - 处理库存不足的情况
   - 实现死锁检测和重试机制
   - 记录详细的日志信息

## 提示
- 考虑使用乐观锁或悲观锁策略
- 可以使用版本号字段实现乐观锁
- 对于高并发场景，考虑先扣减库存再创建订单的策略
- 使用指数退避算法处理重试
- SQLite的WAL模式可以提高并发性能

## 测试场景
- 商品A有100件库存
- 同时有150个用户尝试购买商品A（每人购买1件）
- 验证最终只有100个订单成功，库存为0

## 扩展挑战（可选）
- 实现分布式秒杀系统（使用多个数据库实例）
- 添加限流功能，防止恶意刷单
- 实现库存预热和缓存机制
