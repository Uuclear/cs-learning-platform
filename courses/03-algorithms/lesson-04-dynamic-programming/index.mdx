# 动态规划：聪明的"抄近路"

> 如果递归是个死磕到底的愣头青，那动态规划就是个精打细算的老油条。

## 开场白

同学们好！今天我们聊聊算法世界中最让人又爱又恨的东西——**动态规划**（Dynamic Programming，简称DP）。

说实话，这名字起得特别烂。"动态"？它一点都不动态。"规划"？它跟规划也没啥关系。这个名字的发明者Richard Bellman后来自己都说，当时只是为了拿经费才起了个"花里胡哨"的名字。

但名字烂归烂，动态规划可是面试的**常客**，大厂的敲门砖。你打开LeetCode，随便翻几页就能看到DP家族——什么"打家劫舍"、"股票买卖"、"编辑距离"，花样百出，把多少候选人按在地上摩擦。

**那么问题来了：**

- 什么是动态规划？它和我们学过的递归有什么关系？
- 为什么有些问题用DP做，速度能从"等到天荒地老"变成"嗖的一下"？
- DP看起来难，到底有没有套路可循？

别怕，今天教授带你从0到1搞懂动态规划。我们不说废话，不搞学术黑话，就用最接地气的例子，让你从"DP是啥"到"DP我会了"。

> 💡 **思考问题**: 你从家到学校，每天走同一条路。有一天你发现，如果先绕到朋友家拿本书，再去学校，总路程反而更短了。为什么？——因为有些路你走了一遍又一遍，如果能记住之前算过的结果，就能省掉重复劳动！

## 核心概念

### 什么是动态规划？

一句话概括：**动态规划 = 递归 + 记忆化 = 别犯傻，算过的事别再算。**

想象你在玩一个迷宫游戏：

- 走到一个岔路口，你选左边，发现是死胡同
- 回到岔路口，你又选了左边... 又是死胡同
- 第三次你还选左边...

你是不是觉得这人脑子有问题？对，这就是**朴素递归**在做的事——**反复计算已经算过的子问题**。

动态规划的做法很简单：**拿个小本本记下来**。第一次走进死胡同时，在岔路口贴个便签："左边 = 死路，别走！"下次再来，直接看便签，不用重走一遍。

### 动态规划的三大特征

不是所有问题都能用DP解决的。一个问题能用DP，必须满足两个条件（也有人说是三大特征，我习惯算两个核心+一个加分项）：

#### 1. 重叠子问题（最重要的特征）

还是刚才的迷宫例子。如果你每步都走到一个全新区域，之前走过的路对现在毫无帮助，那就没法"记忆"。

**斐波那契数列**是最经典的重叠子问题：

```
计算 F(5):
F(5) = F(4) + F(3)
     = (F(3) + F(2)) + F(3)
     = ((F(2) + F(1)) + F(2)) + F(3)
     ...
```

看到了吗？F(3)被算了2次，F(2)被算了3次！问题规模一大，重复计算就像滚雪球一样爆炸增长。

#### 2. 最优子结构

这个问题的**最优解可以从子问题的最优解推导出来**。

比如背包问题：前5个物品在容量为10时的最优解 = max(不装第5个物品的最优解, 装第5个物品的最优解)。大问题的最优解由小问题的最优解拼出来。

> 生活类比：你要找从北京到上海的最快路线。如果最快路线经过南京，那么北京→南京这段也必须是最快的。这就是最优子结构。

#### 3. 无后效性（加分项）

子问题的解一旦确定，就不会因为后面的决策而改变。今天算出来的F(5)=5，明天还是5，不会因为你明天心情好就变成了6。

**关键思路**：

- **状态定义**：dp[i][w] = 考虑前i个物品，背包容量为w时的最大价值

| 1 | 0 | 0 | 3 | 4 | 4 | 7 | 7 | 7 | 7 | 7 | 7 |
| 2 | 0 | 0 | 3 | 4 | 5 | 7 | 8 | 9 | 9 | 12 | 12 |


- **边界条件**：dp[0][j] = dp[i][0] = 0，空串没有公共子序列

**实际用途**：DNA序列比对、git diff算法、拼写检查器！没错，你每次用`git diff`看代码改动，底层就是LCS。

## 深入理解

### 动态规划的两种写法

很多人学DP最困惑的就是：**记忆化搜索和迭代法到底有什么区别？**

#### 方法一：自顶向下（记忆化搜索）

```python
def fib_memo(n, memo=None):
    if memo is None:
        memo = {}
    if n in memo:
        return memo[n]
    if n <= 1:
        return n
    memo[n] = fib_memo(n - 1, memo) + fib_memo(n - 2, memo)
    return memo[n]
```

**特点**：
- 从大问题出发，递归地分解成小问题
- 用一个字典（哈希表）或数组记录已计算的结果
- **只计算真正需要的子问题**，不需要的不会浪费计算
- 写法上更像原始递归，容易从递归思路转换过来

**就像**：你要去图书馆找一本特定的书，只翻你需要的几页，而不是整本书从头读到尾。

#### 方法二：自底向上（迭代法）

```python
def fib_iterative(n):
    if n <= 1:
        return n
    prev2, prev1 = 0, 1
    for i in range(2, n + 1):
        current = prev1 + prev2
        prev2, prev1 = prev1, current
    return prev1
```

**特点**：
- 从小问题出发，逐步推到大问题
- 通常可以用循环替代递归，**没有调用栈溢出风险**
- 可以做**状态压缩**，节省空间
- 写起来需要想清楚"从小到大的顺序"

**就像**：你从字母A开始学外语，一个一个字母、一个一个单词地学上去，最终能读懂整篇文章。

#### 两者对比

| 特性 | 自顶向下（记忆化） | 自底向上（迭代） |
|------|-------------------|-----------------|
| 思路 | 大问题→小问题 | 小问题→大问题 |
| 代码 | 递归+记忆表 | 循环+DP表 |
| 计算量 | 只算需要的 | 可能算不需要的 |
| 栈溢出 | 有递归深度限制 | 不会溢出 |
| 空间优化 | 较难 | 容易（状态压缩） |
| 调试 | 容易跟踪 | 需要理解填表顺序 |

**教授建议**：初学者先写记忆化搜索（思路更自然），熟练后再写迭代法（性能更好，能做空间优化）。

### 动态规划解题四步曲

遇到DP问题，按这四步来，不会走偏：

**第1步：定义状态**

问自己：**"我要在DP表里存什么？"**

通常是"dp[i] = 前i个元素的某个最优值"或"dp[i][j] = 前i个和前j个的某个最优值"。

> 状态定义得好，解题就成功了一半。就像数学题设对了变量，后面的推导就顺了。

**第2步：写出状态转移方程**

问自己：**"当前状态能从哪些之前的状态推出来？"**

这是DP的灵魂。找到"大问题和小问题之间的关系"。

> 背包问题的状态转移方程：dp[i][w] = max(不装, 装)
> 最长公共子序列：字符相同就+1，不同就取max

**第3步：确定边界条件**

问自己：**"最小的子问题是什么？它的值是多少？"**

边界条件是DP的基石。没有正确的边界，整个DP表都会歪掉。

> 斐波那契：F(0)=0, F(1)=1
> 背包问题：dp[0][w]=0（0个物品）
> LCS：dp[0][j]=dp[i][0]=0（空串）

**第4步：编写代码**

有了前面三步，写代码就是照葫芦画瓢。

> 填DP表就像填Excel表格。先定表格结构（状态定义），再定每个格子怎么算（状态转移），先填第一行/第一列（边界条件），然后按顺序填满。

### 时间/空间复杂度分析

| 问题 | 时间复杂度 | 空间复杂度 | 备注 |
|------|-----------|-----------|------|
| 斐波那契（记忆化） | O(n) | O(n) | 可以压缩到O(1) |
| 0-1背包 | O(n×W) | O(n×W) | W是背包容量，可以压缩到O(W) |
| 最长公共子序列 | O(m×n) | O(m×n) | m,n是字符串长度 |

**关键洞察**：DP的时间复杂度 ≈ **状态数量 × 每个状态的计算时间**。状态数越多，计算量越大。

## 实际应用

### 应用场景1: 拼写检查和自动纠错

当你打字打错，比如把"dynamic"打成"dynmic"，手机会自动提示正确拼写。这个功能的核心算法就是**编辑距离**（Edit Distance），它就是一个经典的DP问题！

编辑距离问的是：**把一个字符串变成另一个，最少需要几次操作（插入、删除、替换字符）？**

```
"dynmic" → "dynamic" 需要 2 次操作：
1. 在 'n' 后面插入 'a' → "dyn amic"
2. 完成！(实际上需要插入'a'和'm'，但这是简化说明)
```

微信的拼写检查、Word的自动更正、甚至搜索引擎的"你是不是要找..."，底层都是编辑距离的DP算法。

### 应用场景2: DNA序列比对

生物学家拿到两段DNA序列后，需要找出它们的相似区域。这正是我们课堂上讲的**最长公共子序列**问题！

```
DNA1: AGCTAGCTA
DNA2: AGTACTGA
LCS:  AGTACTA (相似度 77.8%)
```

相似度越高，说明这两段DNA可能来自相似的物种或者有相似的基因功能。这就是**生物信息学**的基础算法之一。

### 应用场景3: 投资最优策略

假设你有10万元可以投资3个项目，每个项目投入不同金额会有不同收益。如何分配资金让总收益最大？

这就是**背包问题的变体**：资金 = 背包容量，项目 = 物品，投入金额 = 重量，收益 = 价值。

## 常见误区

⚠️ **注意这些坑！**

### 误区1: 所有递归问题都能用DP优化

**错误认知**："既然DP = 递归 + 记忆化，那我把所有递归都加上记忆化不就变快了？"

**真相**：只有存在**重叠子问题**的递归才值得加记忆化。如果每个子问题只算一次（比如树的遍历），记忆化就是脱裤子放屁——白费劲。

> **例子**：遍历二叉树，每个节点只访问一次，没有重叠子问题，加记忆化没用。

✅ **正确做法**：先判断问题是否有重叠子问题，再决定是否用DP。

### 误区2: DP表一定是二维的

**错误认知**："DP就是一个二维数组。"

**真相**：DP表的维度取决于你的状态定义。一维、二维、三维甚至更高维都有可能。

> **一维DP**：斐波那契、爬楼梯
> **二维DP**：背包问题、LCS
> **三维DP**：有些区间DP问题需要三维状态

✅ **正确做法**：根据问题的需要定义状态，状态有几个变量，DP就有几维。

### 误区3: 记忆化搜索和迭代法随便选一个就行

**错误认知**："两种写法等价，随便写一个。"

**真相**：在某些场景下，记忆化搜索可能遇到**递归深度溢出**的问题（Python默认递归深度1000层）。而迭代法没有这个问题。

另外，迭代法更容易做**状态压缩**（比如斐波那契只用两个变量），空间复杂度可以大幅降低。

✅ **正确做法**：简单问题用记忆化搜索（好写），复杂问题或数据量大时用迭代法（稳健）。

<details>
<summary>💡 提示（点击展开）</summary>

爬到第n阶，只能从第n-1阶（爬1步）或第n-2阶（爬2步）来。
所以：dp[n] = dp[n-1] + dp[n-2]，这和斐波那契一样！

边界：dp[1] = 1, dp[2] = 2

### 练习2: 不同路径 ⭐⭐

**描述**：一个机器人位于m×n网格的左上角，每次只能向右或向下移动。问有多少条不同的路径可以到达右下角？

**要求**：
1. 定义DP状态
2. 写出状态转移方程
3. 实现并测试3×7的网格

<details>
<summary>💡 提示（点击展开）</summary>

dp[i][j] = 到达位置(i,j)的路径数
dp[i][j] = dp[i-1][j] + dp[i][j-1]（只能从上面或左边来）
边界：第一行和第一列都是1（只有一条路能到）

</details>

### 练习3: 零钱兑换 ⭐⭐⭐

**描述**：给定不同面额的硬币 coins = [1, 5, 10]，凑出总金额 amount = 15 最少需要几枚硬币？

**要求**：
1. 定义DP状态：dp[i] = 凑出金额i需要的最少硬币数
2. 写出状态转移方程
3. 实现并输出结果

<details>
<summary>💡 提示（点击展开）</summary>

dp[i] = min(dp[i - coin] + 1) for coin in coins
意思是：凑出金额i，可以从金额(i-coin)的状态加一枚coin硬币得到
取所有可能选择中的最小值

边界：dp[0] = 0（凑0元不需要硬币）
如果凑不出某个金额，dp[i] = ∞

对于coins=[1,2,5], amount=15，答案应该是3枚（5+5+5）

</details>

## 小结与思考

### 本节要点

今天我们学习了动态规划的核心知识：

1. **动态规划的本质**：递归 + 记忆化 = 别犯傻，算过的事别再算

2. **两个核心特征**：
   - 重叠子问题：同样的小问题会被反复算
   - 最优子结构：大问题的最优解由小问题的最优解拼出来

3. **两种写法**：
   - 自顶向下（记忆化搜索）：从大问题递归分解，只算需要的子问题
   - 自底向上（迭代法）：从最小的子问题推到大问题，能做空间压缩

4. **解题四步曲**：
   - 定义状态 → 写出状态转移 → 确定边界 → 编写代码

5. **三个经典案例**：
   - 斐波那契（入门练手）
   - 0-1背包（二维DP表的典型）
   - 最长公共子序列（字符串DP，实际应用广泛）

### 思考题

1. **动态规划和贪心算法有什么区别？** 贪心算法也是做"最优选择"，为什么它不能解决背包问题？

2. **如果背包问题的物品可以分割（比如可以只装半个物品），贪心算法能解决吗？** 这和0-1背包有什么本质区别？

3. **你能想到生活中哪些场景可以用动态规划的思路来优化决策？** 比如时间管理、旅行规划等。
