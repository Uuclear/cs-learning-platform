## 挑战1: 爬楼梯

### 难度
⭐⭐

### 描述
你正在爬一段有 n 阶的楼梯。每次你可以选择爬 1 阶或 2 阶。请问有多少种不同的方法可以爬到楼顶？

### 输入
一个正整数 n（1 ≤ n ≤ 45）

### 输出
不同的爬楼梯方法总数

### 示例

**示例 1:**
```
输入: n = 3
输出: 3
解释: 有三种方法：1+1+1, 1+2, 2+1
```

**示例 2:**
```
输入: n = 5
输出: 8
解释: 八种方法：11111, 1112, 1121, 1211, 2111, 122, 212, 221
```

### 约束条件
- 1 ≤ n ≤ 45
- 保证结果不会溢出32位整数范围

### 提示
- 爬到第n阶，只能从第n-1阶（爬1步）或第n-2阶（爬2步）到达
- 这和我们学过的哪个数列非常相似？
- 试着写出前几项：n=1, 2, 3, 4时各有多少种方法

### 进阶思考
- 如果每次可以爬1阶、2阶或3阶，问题如何变化？
- 如果楼梯上有几个台阶坏了不能踩（给出不能踩的台阶编号列表），如何求解？

### 思路
爬到第n阶的方法数 = 爬到第n-1阶的方法数 + 爬到第n-2阶的方法数。
这正是斐波那契数列！dp[n] = dp[n-1] + dp[n-2]。

边界条件：dp[1] = 1（爬1阶只有1种方法），dp[2] = 2（1+1或2）。

### 代码
```python
def climb_stairs(n):
    """
    爬楼梯问题
    dp[i] = dp[i-1] + dp[i-2]
    """
    # 边界情况
    if n <= 2:
        return n

    # 状态压缩：只需要保存前两个值
    prev2 = 1  # dp[1]
    prev1 = 2  # dp[2]

    for i in range(3, n + 1):
        current = prev1 + prev2
        prev2 = prev1
        prev1 = current

    return prev1


if __name__ == "__main__":
    # 测试
    for n in [1, 2, 3, 5, 10]:
        print(f"n = {n}: {climb_stairs(n)} 种方法")

# 输出:
# n = 1: 1 种方法
# n = 2: 2 种方法
# n = 3: 3 种方法
# n = 5: 8 种方法
# n = 10: 89 种方法
```

### 复杂度分析
- 时间复杂度: O(n)，循环n次
- 空间复杂度: O(1)，状态压缩只用了两个变量
