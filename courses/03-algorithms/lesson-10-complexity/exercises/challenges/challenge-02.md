# 编程挑战 2：性能优化实战

## 题目描述

你发现团队中的一个关键函数运行非常缓慢。这个函数用于在一个大型数据集中查找所有满足特定条件的元素对。

原始代码如下：

```python
def find_pairs_slow(data, target_sum):
    """
    在数据中查找所有和为target_sum的元素对
    当前实现的时间复杂度很高！
    """
    pairs = []
    n = len(data)
    
    # 检查所有可能的元素对
    for i in range(n):
        for j in range(i + 1, n):
            if data[i] + data[j] == target_sum:
                pairs.append((data[i], data[j]))
    
    return pairs
```

## 问题分析

1. 分析当前实现的时间复杂度
2. 当数据集很大时（比如100万个元素），这个函数会有多慢？
3. 提出至少两种不同的优化方案

## 要求

1. **实现优化版本**：编写至少一个优化后的函数，显著降低时间复杂度
2. **性能对比**：编写测试代码，对比原始版本和优化版本在不同数据规模下的运行时间
3. **复杂度分析**：详细说明你的优化方案的时间和空间复杂度
4. **边界情况处理**：考虑重复元素、空数组等边界情况

## 示例

输入：`data = [1, 2, 3, 4, 5], target_sum = 5`
输出：`[(1, 4), (2, 3)]`
