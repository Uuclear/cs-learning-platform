# 编程挑战1：网络延迟优化

## 背景
你是一家互联网公司的网络工程师，负责优化公司内部网络的通信延迟。网络可以表示为一个有向图，其中顶点代表服务器，边代表网络连接，权重代表通信延迟（毫秒）。

## 任务
实现一个函数 `optimize_network_delay(graph: Dict[str, Dict[str, int]], source: str) -> Dict[str, int]`，该函数使用Dijkstra算法计算从源服务器到所有其他服务器的最小通信延迟。

## 要求
- 输入：`graph` 是邻接表表示的有向图，`source` 是源服务器名称
- 输出：字典，键为服务器名称，值为从源服务器到该服务器的最小延迟
- 如果某个服务器无法到达，对应的值应为 -1
- 图中不会包含负权边

## 示例
```python
graph = {
    'A': {'B': 10, 'C': 3},
    'B': {'C': 1, 'D': 2},
    'C': {'B': 4, 'D': 8, 'E': 2},
    'D': {'E': 7},
    'E': {'D': 9}
}

result = optimize_network_delay(graph, 'A')
# result 应该是 {'A': 0, 'B': 7, 'C': 3, 'D': 9, 'E': 5}
```

## 提示
- 你可以直接使用标准库中的 `heapq` 模块
- 注意处理无法到达的服务器情况
- 确保正确初始化距离字典
