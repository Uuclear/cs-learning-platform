# 挑战1：区间合并

## 难度
⭐⭐

## 描述
给定一组时间区间，合并所有重叠的区间。

例如，区间 `[1, 3]` 和 `[2, 6]` 有重叠（因为 2 ≤ 3），合并后变为 `[1, 6]`。

## 输入
一个区间列表，每个区间是一个二元组 `[start, end]`，其中 `start ≤ end`。

## 输出
合并后的区间列表，按起点排序，且区间之间互不重叠。

## 示例

**示例 1:**
```
输入: [[1, 3], [2, 6], [8, 10], [15, 18]]
输出: [[1, 6], [8, 10], [15, 18]]
解释: [1, 3] 和 [2, 6] 重叠，合并为 [1, 6]
```

**示例 2:**
```
输入: [[1, 4], [4, 5]]
输出: [[1, 5]]
解释: [1, 4] 和 [4, 5] 在端点处相接，视为重叠
```

## 约束条件
- 区间数量: 0 ≤ n ≤ 1000
- 区间范围: 0 ≤ start ≤ end ≤ 10000

## 提示
- 想一想按什么顺序处理区间最方便判断重叠
- 维护一个"当前合并区间"，遇到新区间时判断是否需要合并

## 进阶思考
- 如果区间不相邻但间隔很小（比如间隔≤1），是否也应该合并？

### 思路
贪心策略：按区间起点排序，然后依次遍历。维护一个结果列表，每次遇到新区间时：
- 如果与结果列表中最后一个区间重叠，合并（更新终点为两者的最大值）
- 如果不重叠，直接添加到结果列表

### 代码
```python
def merge_intervals(intervals):
    if not intervals:
        return []

    # 按区间起点排序
    sorted_intervals = sorted(intervals, key=lambda x: x[0])
    merged = [sorted_intervals[0]]

    for current in sorted_intervals[1:]:
        last = merged[-1]
        # 判断是否重叠
        if current[0] <= last[1]:
            last[1] = max(last[1], current[1])
        else:
            merged.append(current)

    return merged
```

### 复杂度分析
- 时间复杂度: O(n log n)，主要是排序时间
- 空间复杂度: O(n)，存储结果
