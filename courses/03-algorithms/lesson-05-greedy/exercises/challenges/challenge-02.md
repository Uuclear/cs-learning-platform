# 挑战2：任务调度

## 难度
⭐⭐⭐

## 描述
你有一系列任务，每个任务有一个截止时间（deadline）和完成该任务所需的时间。
你一次只能做一个任务。问最多能完成多少个任务？

## 输入
一个任务列表，每个任务是一个二元组 `(deadline, duration)`。
- `deadline`: 任务必须在此时间点之前完成
- `duration`: 完成该任务所需的时间

## 输出
最多能完成的任务数，以及完成的是哪些任务。

## 示例

**示例 1:**
```
输入: [(4, 2), (3, 1), (5, 2), (2, 1), (6, 3)]
输出: 4个任务
解释: 按顺序完成 (2,1)→(3,1)→(4,2)→(6,3)，时间点分别为 0-1, 1-2, 2-4, 4-7
     其中 (4,2) 在时间点4完成，但截止时间也是4，满足条件
     但 (6,3) 需要在时间点7完成，超过截止时间6，所以不能选
     最优选择: (2,1), (3,1), (4,2), (5,2) 共4个
```

**示例 2:**
```
输入: [(3, 2), (3, 2)]
输出: 1个任务
解释: 两个任务截止时间都是3，都需要2小时，只能选一个
```

## 约束条件
- 任务数量: 1 ≤ n ≤ 1000
- 截止时间: 1 ≤ deadline ≤ 10000
- 持续时间: 1 ≤ duration ≤ 1000

## 提示
- 按什么顺序考虑任务？截止时间早的还是晚的？
- 如果一个任务在当前时间无法在截止时间前完成，该怎么办？

## 进阶思考
- 如果想最大化完成任务的总价值（每个任务有不同价值），贪心策略还适用吗？

### 思路
贪心策略：按截止时间从早到晚排序，依次尝试完成任务。
用一个变量记录当前时间，如果当前时间 + 任务耗时 ≤ 截止时间，就接这个任务。

### 代码
```python
def max_tasks(tasks):
    if not tasks:
        return 0, []

    # 按截止时间排序
    sorted_tasks = sorted(tasks, key=lambda x: x[0])

    completed = []
    current_time = 0

    for deadline, duration in sorted_tasks:
        if current_time + duration <= deadline:
            completed.append((deadline, duration))
            current_time += duration

    return len(completed), completed
```

### 复杂度分析
- 时间复杂度: O(n log n)，排序时间
- 空间复杂度: O(n)，存储结果
