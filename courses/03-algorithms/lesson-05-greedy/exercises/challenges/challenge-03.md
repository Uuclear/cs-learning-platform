# 挑战3：最少站台问题

## 难度
⭐⭐⭐⭐

## 描述
一个火车站给出所有火车的到达时间和离开时间。
每辆火车到达时需要占用一个站台，离开后站台释放。
问这个火车站最少需要多少个站台？

## 输入
- `arrivals`: 到达时间列表
- `departures`: 离开时间列表

时间格式为整数 `HHMM`，例如 `900` 表示 9:00，`1530` 表示 15:30。

## 输出
最少需要的站台数量。

## 示例

**示例 1:**
```
输入: arrivals = [900, 940, 950, 1100, 1500, 1800]
     departures = [910, 1200, 1120, 1130, 1900, 2000]
输出: 3
解释:
  9:00 到达，需要1个站台
  9:10 离开，释放1个站台（当前0个）
  9:40 到达，需要1个站台
  9:50 到达，需要2个站台
  11:00 到达，此时9:40和9:50的火车还在，需要3个站台
  11:20 离开，释放1个站台
  11:30 离开，释放1个站台
  15:00 到达，需要1个站台
  18:00 到达，需要2个站台
  19:00 离开，释放1个站台
  20:00 离开，释放1个站台
  最大同时需要3个站台
```

**示例 2:**
```
输入: arrivals = [900, 1000]
     departures = [930, 1030]
输出: 1
解释: 两辆火车不会同时在站，1个站台就够
```

## 约束条件
- 火车数量: 1 ≤ n ≤ 10000
- 时间范围: 0 ≤ time ≤ 2359
- 到达时间严格小于对应离开时间

## 提示
- 把到达和离开事件分开，各自排序
- 用两个指针模拟"时间线"，遇到到达就+1站台，遇到离开就-1站台
- 记录过程中的最大值

## 进阶思考

### 思路
贪心策略：将到达和离开事件分别排序，用双指针模拟时间线。
- 当前事件是到达：需要一个站台，站台数+1
- 当前事件是离开：释放一个站台，站台数-1
- 记录过程中站台数的最大值

### 代码
```python
def min_platforms(arrivals, departures):
    arrivals.sort()
    departures.sort()

    platforms = 0
    max_platforms = 0
    i = 0  # 到达指针
    j = 0  # 离开指针

    while i < len(arrivals):
        if arrivals[i] < departures[j]:
            platforms += 1
            max_platforms = max(max_platforms, platforms)
            i += 1
        else:
            platforms -= 1
            j += 1

    return max_platforms
```

### 复杂度分析
- 时间复杂度: O(n log n)，主要是排序时间
- 空间复杂度: O(1)，除了输入数组外只用常数额外空间
