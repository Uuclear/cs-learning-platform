# 贪心算法：每一步都选最好的

> 人生如果能贪心就好了——每次都选看起来最好的选项，最后一定是最幸福的。但算法告诉我们：**有时候行，有时候不行**。

---

## 开场白

同学们好！今天我们来聊聊算法世界里最"短视"但也最"聪明"的策略——贪心算法。

想象一下，你去自助餐厅吃午饭。面前摆着龙虾、牛排、寿司、甜点……你的胃容量有限（对，就像个背包），你怎么选才能吃得最值？

- **贪心策略**：每次都选单价最高的，先拿龙虾，再拿牛排……
- **动态规划策略**：先列个表，算算各种组合的总价值，找到最优方案……
- **你的真实策略**：先转三圈，最后拿了半盘炸鸡和一碗汤

**那么问题来了：**
- 贪心算法为什么叫"贪心"？它到底在"贪"什么？
- 什么时候"短视"反而能得到全局最优解？
- 为什么有时候贪心会翻车？

别急，今天教授带你看看，这个"只看眼前"的策略到底有什么魔力，又有什么致命缺陷。

## 核心概念

### 什么是贪心算法？

贪心算法（Greedy Algorithm），顾名思义，就是一种**"鼠目寸光"**的算法策略。它在每一步都做出**当前看起来最好的选择**，希望通过一系列局部最优选择，最终达到全局最优。

你可以把贪心算法想象成一个**从不后悔的人**：
- 做了选择就不回头
- 不考虑未来的可能性
- 只关心"现在这个时刻，哪个选项最诱人"

听起来很鲁莽对吧？但神奇的是，在某些特定问题中，这种"短视"的策略恰恰能得到**全局最优解**！

### 贪心算法的两大支柱

贪心算法不是对所有问题都有效。要想让贪心算法得到最优解，问题必须具备两个性质：

#### 1. 贪心选择性质

**定义**：通过一系列局部最优选择，可以得到全局最优解。

**通俗理解**：你每次都选"当前最好的"，最后确实能得到"总体最好的"。

> 🍕 **类比**：你去比萨店，每次选最大的一块。如果最终你能吃到最多量的比萨，那这个问题就具有贪心选择性质。

#### 2. 最优子结构

**定义**：一个问题的最优解包含其子问题的最优解。

**通俗理解**：大问题的最优答案，可以从小问题的最优答案拼出来。

> 🧩 **类比**：拼图的最终画面，是由每一小块拼图的"最佳位置"组成的。每一步放对了，最后就拼对了。

### 贪心 vs 动态规划

这是面试必考题！很多同学分不清这两者。让我们用一个表格来对比：

| 特性 | 贪心算法 | 动态规划 |
|------|----------|----------|
| 选择策略 | 每步选当前最优，不回头 | 考虑所有可能，选全局最优 |
| 子问题 | 不需要求解所有子问题 | 需要求解所有子问题 |
| 结果保证 | 不一定得到最优解 | 保证得到最优解 |
| 时间复杂度 | 通常较低 | 通常较高 |
| 空间复杂度 | 通常较低 | 通常需要存储子问题解 |
| 适用条件 | 贪心选择性质 + 最优子结构 | 重叠子问题 + 最优子结构 |

**一句话总结**：贪心是"一根筋"，动态规划是"深思熟虑"。能用贪心解决的问题，通常也能用动态规划解决，但贪心更快更简单——前提是你得确认贪心是对的！

## 代码演示

让我们通过三个经典例子来理解贪心算法的精髓。

### 示例1：活动选择问题

这是贪心算法的"Hello World"。

**问题描述**：你一天有很多活动可以参加，每个活动有开始时间和结束时间。同一时间只能参加一个活动。如何选择才能参加**最多数量**的活动？

**贪心策略**：每次都选择**结束时间最早**的、且与已选活动不冲突的活动。

**为什么选结束最早的？** 因为这样给后面的活动留出了最多的时间空间！

```python
# 活动选择问题 - 贪心算法
def activity_selection(activities):
    """
    活动选择问题 - 贪心算法
    参数: activities - 列表，每个元素为(活动名, 开始时间, 结束时间)
    返回: 选择的活动列表
    """
    # 按结束时间排序（贪心策略：优先选结束早的）
    sorted_activities = sorted(activities, key=lambda x: x[2])

    selected = []
    last_end_time = 0

    for name, start, end in sorted_activities:
        # 如果当前活动的开始时间 >= 上一个选中活动的结束时间，不冲突
        if start >= last_end_time:
            selected.append((name, start, end))
            last_end_time = end  # 更新最后结束时间

    return selected


# 测试
activities = [
    ("上课", 9, 10), ("打球", 9, 11), ("看书", 10, 12),
    ("写代码", 11, 13), ("看电影", 12, 14), ("开会", 13, 15),
    ("健身", 14, 16), ("约会", 15, 17), ("睡觉", 16, 18),
]

selected = activity_selection(activities)
print(f"最优选择（共 {len(selected)} 个活动）:")
for name, start, end in selected:
    print(f"  {name}: {start}:00 - {end}:00")
```

**运行结果**：
```
最优选择（共 5 个活动）:
  上课: 9:00 - 10:00
  看书: 10:00 - 12:00
  看电影: 12:00 - 14:00
  健身: 14:00 - 16:00
  睡觉: 16:00 - 18:00
```

看到没？从9个活动中，我们成功选出了5个互不冲突的活动。这就是贪心的力量！

### 示例2：找零钱问题——贪心什么时候对，什么时候错

这是理解贪心局限性的最佳例子。

**问题描述**：给定一些面值的硬币，找零时如何用最少数量的硬币？

```python
# 找零钱问题 - 贪心 vs 动态规划
def coin_change_greedy(coins, amount):
    """
    找零钱 - 贪心算法
    参数: coins - 硬币面值列表（从大到小排序）
          amount - 需要找零的金额
    返回: 使用的硬币列表
    """
    coins_sorted = sorted(coins, reverse=True)
    result = []
    remaining = amount

    for coin in coins_sorted:
        while remaining >= coin:
            result.append(coin)
            remaining -= coin

    if remaining > 0:
        return None  # 无法完全找零
    return result


def coin_change_dp(coins, amount):
    """
    找零钱 - 动态规划（求最优解）
    返回: 最少硬币数量，如果无法找零返回-1
    """
    dp = [float('inf')] * (amount + 1)
    dp[0] = 0

    for i in range(1, amount + 1):
        for coin in coins:
            if i >= coin:
                dp[i] = min(dp[i], dp[i - coin] + 1)

    return dp[amount] if dp[amount] != float('inf') else -1


# 场景1：美国硬币系统 [25, 10, 5, 1]，找63分
print("=== 美国硬币系统（贪心正确）===")
us_coins = [25, 10, 5, 1]
result1 = coin_change_greedy(us_coins, 63)
dp_result1 = coin_change_dp(us_coins, 63)
print(f"贪心: {result1} (共 {len(result1)} 个)")
print(f"最优: {dp_result1} 个")

# 场景2：特殊硬币系统 [4, 3, 1]，找6分
print("\n=== 特殊硬币系统（贪心失败）===")
special_coins = [4, 3, 1]
result2 = coin_change_greedy(special_coins, 6)
dp_result2 = coin_change_dp(special_coins, 6)
print(f"贪心: {result2} (共 {len(result2)} 个)")
print(f"最优: {dp_result2} 个")
```

**运行结果**：
```
=== 美国硬币系统（贪心正确）===
贪心: [25, 25, 10, 1, 1, 1] (共 6 个)
最优: 6 个

=== 特殊硬币系统（贪心失败）===
贪心: [4, 1, 1] (共 3 个)
最优: 2 个
```

**关键发现**：
- 在美国硬币系统下，贪心是**正确的**！
- 但在特殊硬币系统下，贪心**翻车了**！贪心选了4+1+1（3个），最优是3+3（2个）

这说明：**贪心算法的正确性取决于问题的结构**。

### 示例3：分数背包问题

这是贪心算法大显身手的场景。

**问题描述**：你有一个50kg容量的背包，有黄金、白银、铜等物品可以选择。与0-1背包不同，你可以拿走物品的一部分（比如只拿一半黄金）。目标是最大化总价值。

**贪心策略**：按**单位价值**（价值/重量）从高到低排序，优先拿"性价比"最高的。

```python
# 分数背包问题 - 贪心算法
class Item:
    """物品类"""
    def __init__(self, name, value, weight):
        self.name = name
        self.value = value
        self.weight = weight
        self.ratio = value / weight  # 单位价值


def fractional_knapsack(items, capacity):
    """
    分数背包问题 - 贪心算法
    返回: (总价值, 选择的物品记录)
    """
    # 按单位价值从高到低排序
    items_sorted = sorted(items, key=lambda x: x.ratio, reverse=True)

    total_value = 0
    remaining = capacity

    for item in items_sorted:
        if remaining <= 0:
            break

        if item.weight <= remaining:
            # 全部拿走
            total_value += item.value
            remaining -= item.weight
            print(f"  拿走全部 {item.name} ({item.weight}kg)")
        else:
            # 只拿一部分
            fraction = remaining / item.weight
            total_value += item.value * fraction
            print(f"  拿走 {item.name} 的 {fraction*100:.0f}% ({remaining}kg)")
            remaining = 0

    return total_value


# 测试
items = [
    Item("黄金", 60, 10),   # 单位价值: 6.0
    Item("白银", 100, 20),  # 单位价值: 5.0
    Item("铜",   120, 30),  # 单位价值: 4.0
    Item("铁",   50, 15),   # 单位价值: 3.33
]

capacity = 50  # 背包容量50kg
print(f"背包容量: {capacity}kg")
total = fractional_knapsack(items, capacity)
print(f"总价值: {total}")
```

**运行结果**：
```
背包容量: 50kg
  拿走全部 黄金 (10kg)
  拿走全部 白银 (20kg)
  拿走 铜 的 67% (20kg)
总价值: 240.0
```

看到没？贪心算法先拿最"值钱"的黄金，再拿白银，最后拿了一部分铜。背包满了，总价值240，这就是最优解！

## 深入理解

### 贪心算法的"思考过程"

当你面对一个问题时，怎么判断能不能用贪心算法？遵循以下步骤：

**第一步：确定贪心策略**

问自己：**"在当前状态下，哪个选择看起来最诱人？"**

- 活动选择 → 选结束最早的
- 分数背包 → 选单位价值最高的
- 找零钱 → 选面值最大的

**第二步：验证贪心选择性质**

问自己：**"如果我做了这个选择，剩下的问题是不是和原来一样的子问题？"**

如果能，那就可以继续贪心地处理子问题。

**第三步：证明最优子结构**

问自己：**"问题的最优解，是否包含子问题的最优解？"**

如果以上三个问题的答案都是"是"，那恭喜你可以放心用贪心了！

### 复杂度分析

贪心算法的时间复杂度主要由**排序**决定：

| 问题 | 排序复杂度 | 贪心选择复杂度 | 总复杂度 |
|------|-----------|---------------|----------|
| 活动选择 | O(n log n) | O(n) | O(n log n) |
| 分数背包 | O(n log n) | O(n) | O(n log n) |
| 找零钱(贪心) | O(n log n) | O(amount/min_coin) | O(n log n + amount) |

空间复杂度通常是 O(n) 或 O(1)，取决于是否需要额外存储。

### 贪心算法 vs 其他算法全家福

| 算法 | 思想 | 保证最优 | 典型问题 |
|------|------|----------|----------|
| 贪心 | 每一步选当前最好的 | 不一定 | 活动选择、分数背包 |
| 动态规划 | 算完所有子问题再选 | 保证 | 0-1背包、最长公共子序列 |
| 回溯 | 试错+回退 | 保证 | N皇后、数独 |
| 分支限界 | 剪枝+搜索 | 保证 | 旅行商问题 |

## 实际应用

贪心算法在现实世界中无处不在，只是你可能没注意到。

### 应用场景1：霍夫曼编码（数据压缩）

你每天都在用贪心算法的成果——**ZIP压缩文件**和**JPEG图片**都用到了霍夫曼编码。

霍夫曼编码是一种数据压缩技术，核心思想是：
- 出现频率高的字符，用短编码表示
- 出现频率低的字符，用长编码表示

构建霍夫曼树的过程就是一个贪心过程：
1. 统计每个字符出现的频率
2. 每次选频率**最低**的两个节点，合并成一个新节点
3. 重复直到只剩一棵树

这就是典型的贪心策略！每次合并最小的，最终得到最优的编码方案。

### 应用场景2：Dijkstra最短路径算法

当你在高德地图或Google地图上查路线时，背后运行的**Dijkstra算法**也是贪心的。

Dijkstra算法的核心：
- 每次从"已知最短距离"的节点中，选距离**最近**的节点
- 用这个节点去更新邻居的距离
- 重复直到找到终点

这个"每次选最近的"就是贪心选择！当然，Dijkstra的贪心能工作，是因为所有边的权重都是非负的。如果有负权边，就得用Bellman-Ford算法了。

## 常见误区

⚠️ **注意这些坑！**

### 误区1：贪心总是能得到最优解

这是新手最容易犯的错！贪心算法**只在特定条件下**才得到最优解。

❌ **错误想法**：找零钱问题用贪心一定最优
✅ **正确理解**：只有当硬币系统满足特定性质时，贪心才正确。一般情况下应该用动态规划。

> 🚨 记住：贪心是"直觉型选手"，不是所有问题都能靠直觉解决。

### 误区2：贪心策略可以随便选

很多初学者认为，只要选一种"看起来合理"的贪心策略就行。但**不同的贪心策略可能导致完全不同的结果**。

以活动选择为例：
- 按"开始时间最早"贪心 ❌ 不行
- 按"持续时间最短"贪心 ❌ 不行
- 按"结束时间最早"贪心 ✅ 才行

**贪心策略的选择必须经过严格的分析或证明！**

### 误区3：贪心和动态规划是"二选一"的关系

其实对于同一个问题，贪心和动态规划可以互补使用：

- 先用贪心的思路**快速估算**一个下界或上界
- 再用动态规划**精确求解**

比如在0-1背包问题中，可以先用分数背包的贪心策略算一个上界，用来剪枝，加速动态规划。

### 练习1：区间合并 ⭐

给定一组区间，合并所有重叠的区间。

例如：`[1, 3]` 和 `[2, 6]` 重叠，合并为 `[1, 6]`。

**输入**：`[[1, 3], [2, 6], [8, 10], [15, 18]]`

**输出**：`[[1, 6], [8, 10], [15, 18]]`

**要求**：
- 实现一个函数 `merge_intervals(intervals)`
- 结果按起点排序，区间互不重叠

<details>
<summary>💡 提示（点击展开）</summary>

### 练习2：任务调度 ⭐⭐

你有一系列任务，每个任务有截止时间和完成所需时间。一次只能做一个任务。问最多能完成多少个任务？

**输入**：`[(4, 2), (3, 1), (5, 2), (2, 1), (6, 3)]`

**输出**：4个任务

**要求**：
- 实现一个函数 `max_tasks(tasks)`
- 返回最多能完成的任务数

<details>
<summary>💡 提示（点击展开）</summary>

按截止时间从早到晚排序，依次尝试完成任务。如果当前时间 + 任务耗时 ≤ 截止时间，就接这个任务。

</details>

**输出**：3

**要求**：
- 实现一个函数 `min_platforms(arrivals, departures)`
- 返回最少需要的站台数

## 小结与思考

### 本节要点

1. **贪心算法的核心**：每一步都做当前最优选择，不回溯
2. **适用条件**：问题必须具备贪心选择性质和最优子结构
3. **贪心不是万能的**：不能保证所有问题都得到最优解
4. **经典应用**：活动选择、分数背包、霍夫曼编码、Dijkstra算法
5. **与动态规划的关系**：贪心是"短视版"的动态规划，更快但不保证最优

3. 活动选择问题中，如果每个活动还有一个"价值"，目标是最大化总价值（而不是数量），贪心算法还适用吗？为什么？
