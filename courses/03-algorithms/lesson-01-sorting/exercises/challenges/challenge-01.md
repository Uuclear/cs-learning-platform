# 挑战1：实现选择排序

## 难度
⭐

## 描述
请实现选择排序算法。选择排序的核心思路是：每一轮从剩余未排序的元素中找到最小的，放到已排序部分的末尾。

## 输入
一个无序的整数列表

## 输出
排序后的整数列表（从小到大）

## 示例

**示例 1:**
```
输入: [29, 10, 14, 37, 13]
输出: [10, 13, 14, 29, 37]
```

**示例 2:**
```
输入: [5, 2, 9, 1, 5, 6]
输出: [1, 2, 5, 5, 6, 9]
```

## 约束条件
- 数组长度：0 <= n <= 1000
- 元素值范围：-10000 <= 元素 <= 10000

## 提示
- 使用双重循环：外层控制已排序部分的边界，内层寻找未排序部分的最小值
- 记得记录最小值的索引，而不是值本身
- 交换只在找到新的最小值之后进行

## 进阶思考

- 选择排序是稳定排序吗？为什么？

### 思路
对于每个位置 i（从 0 开始），在 i 到末尾的范围内找到最小元素的索引 min_idx，然后将 arr[i] 和 arr[min_idx] 交换。重复这个过程直到整个数组排好序。

### 代码
```python
def selection_sort(arr):
    """
    选择排序
    参数: arr - 待排序的列表
    返回: 排序后的列表
    """
    n = len(arr)
    for i in range(n):
        # 假设当前位置是最小值
        min_idx = i
        # 在剩余部分中寻找真正的最小值
        for j in range(i + 1, n):
            if arr[j] < arr[min_idx]:
                min_idx = j
        # 交换当前位置和最小值位置
        if min_idx != i:
            arr[i], arr[min_idx] = arr[min_idx], arr[i]
    return arr

# 测试
if __name__ == "__main__":
    test = [29, 10, 14, 37, 13]
    print(f"输入: {test}")
    selection_sort(test)
    print(f"输出: {test}")

# 输出:
# 输入: [29, 10, 14, 37, 13]
# 输出: [10, 13, 14, 29, 37]
```

### 复杂度分析
- 时间复杂度: O(n²) —— 无论最好还是最坏情况，都需要两层遍历
- 空间复杂度: O(1) —— 原地排序，只需要常数额外空间
