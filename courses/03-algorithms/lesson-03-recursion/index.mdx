# 递归：函数的自我修养

> 想象一下，你站在两面镜子中间，往左看，镜子里有你，镜子里的你又在往左看……如此循环往复，永无止境。这就是递归的精髓——函数调用了自己，自己又调用了自己，层层嵌套，直到某个时刻喊停。

## 核心概念

### 什么是递归？

**递归**（Recursion），说白了就是**函数调用自己**。

听起来很玄乎？其实你每天都在用递归的思维：

- **数学归纳法**：要证明 n 成立，先假设 n-1 成立，再去证明 n
- **家族族谱**：你的祖先，是你父母的祖先，一直追溯到最早的那位
- **剥洋葱**：剥一层，还剩一个更小的洋葱；再剥，直到剥完

用程序员的话讲：**把一个大问题，拆解成一个更小的同类问题，一直拆到不能再拆为止。**

### 递归的三要素

一个正确的递归函数，**缺一不可**地需要以下三个要素：

#### 1. 基准情况（Base Case）——"什么时候该停下来"

这是递归的**刹车**。没有刹车的车，会一直往前开直到撞上悬崖。

> 就像俄罗斯套娃，最小的那颗珠子就是基准情况——它里面没有东西了，不用开了。

```
基准情况 = 最小子问题，答案直接已知，不需要再递归
```

#### 2. 递归情况（Recursive Case）——"怎么把问题变小"

这是递归的**发动机**。你需要回答：**"当前问题怎么用更小的同类问题来表达？"**

```
递归情况 = 把大问题分解为更小的同类问题 + 调用自身
```

#### 3. 向基准情况收敛——"保证能停下来"

每一次递归调用，问题规模必须**减小**，而且必须朝着基准情况的方向走。否则……

```python
def infinite_recursion():
    return infinite_recursion()  # 没有变小！无限递归，直到栈溢出

# 运行结果: RecursionError: maximum recursion depth exceeded
# 翻译：朋友，你的栈爆了
```

### 递归 vs 迭代：双胞胎兄弟

递归和循环（迭代）是解决问题的两种思路，它们经常可以互换：

| 特性 | 递归 | 迭代（循环） |
|------|------|-------------|
| 思维方式 | 从上到下，先拆后合 | 从下到上，一步步累加 |
| 代码简洁度 | 通常更优雅 | 可能啰嗦 |
| 空间复杂度 | 需要调用栈，O(n) | 通常 O(1) |
| 理解难度 | 初学较难 | 直观好懂 |
| 适用场景 | 树、图、分治、回溯 | 简单累加、遍历 |

> **教授金句**：能用循环解决的问题，没必要非得用递归；但有些问题，用递归写出来就像诗一样优雅。

## 代码演示

让我们通过代码来感受递归的魅力：

### 示例1: 斐波那契数列——从朴素到记忆化

斐波那契数列是最经典的递归例子：`F(n) = F(n-1) + F(n-2)`，第 n 项等于前两项之和。

```python
# 斐波那契数列：朴素递归 vs 记忆化递归
# 展示递归的力量和陷阱

def fibonacci_naive(n):
    """
    朴素递归实现斐波那契数列
    公式: F(n) = F(n-1) + F(n-2)
    基准: F(0)=0, F(1)=1
    
    时间复杂度: O(2^n) —— 指数级爆炸！
    """
    # 基准情况：前两项直接返回
    if n <= 0:
        return 0
    if n == 1:
        return 1
    
    # 递归情况：前两项之和
    return fibonacci_naive(n - 1) + fibonacci_naive(n - 2)


def fibonacci_memo(n, memo=None):
    """
    记忆化递归实现斐波那契数列
    用字典缓存已经算过的结果，避免重复计算
    
    时间复杂度: O(n) —— 从指数级降到线性！
    """
    # 初始化缓存字典
    if memo is None:
        memo = {}
    
    # 先查缓存：算过的直接拿来用
    if n in memo:
        return memo[n]
    
    # 基准情况
    if n <= 0:
        return 0
    if n == 1:
        return 1
    
    # 递归计算，并保存到缓存
    result = fibonacci_memo(n - 1, memo) + fibonacci_memo(n - 2, memo)
    memo[n] = result  # 记住这个结果
    
    return result


if __name__ == "__main__":
    print("=== 斐波那契数列 ===\n")
    
    # 朴素递归（小心！n太大会很慢）
    print("--- 朴素递归 O(2^n) ---")
    for i in range(10):
        print(f"  F({i}) = {fibonacci_naive(i)}")
    # 输出:
    #   F(0) = 0
    #   F(1) = 1
    #   F(2) = 1
    #   F(3) = 2
    #   F(4) = 3
    #   F(5) = 5
    #   F(6) = 8
    #   F(7) = 13
    #   F(8) = 21
    #   F(9) = 34
    
    print()
    
    # 记忆化递归（快如闪电）
    print("--- 记忆化递归 O(n) ---")
    for i in range(15):
        print(f"  F({i}) = {fibonacci_memo(i)}")
    # 输出:
    #   F(0) = 0
    #   F(1) = 1
    #   ... F(14) = 377
    
    print()
    
    # 性能对比
    import time
    
    print("--- 性能对比：计算 F(35) ---")
    start = time.time()
    result1 = fibonacci_naive(35)
    t1 = time.time() - start
    print(f"  朴素递归: F(35) = {result1}, 耗时 {t1:.3f}s")
    
    start = time.time()
    result2 = fibonacci_memo(35)
    t2 = time.time() - start
    print(f"  记忆化递归: F(35) = {result2}, 耗时 {t2:.6f}s")
    print(f"  速度提升: {t1/t2:.0f} 倍！")
    
# 预期输出（F(35) 性能对比）:
# --- 性能对比：计算 F(35) ---
#   朴素递归: F(35) = 9227465, 耗时 0.852s
#   记忆化递归: F(35) = 9227465, 耗时 0.000012s
#   速度提升: 71000 倍！
```

**教授点评**：看到了吗？朴素递归会做大量**重复工作**。计算 `F(5)` 时，`F(3)` 被算了 2 次，`F(2)` 被算了 3 次！记忆化的核心思想就是**"算过的记下来，下次直接查"**——这就是用空间换时间。

### 示例2: 汉诺塔问题——递归的经典之作

汉诺塔是一个古老的益智游戏：把 n 个盘子从 A 柱移到 C 柱，每次只能移动一个，且大盘不能压在小盘上面。

```python
# 汉诺塔问题：递归的优雅典范
# n个盘子从源柱移到目标柱，借助辅助柱

move_count = 0  # 全局计数器


def hanoi(n, source, target, auxiliary):
    """
    汉诺塔递归解法
    参数:
        n: 盘子数量
        source: 源柱子（起始位置）
        target: 目标柱子（最终位置）
        auxiliary: 辅助柱子（中间借用）
    
    核心思路（三步走）:
        1. 把 n-1 个盘子从 source 移到 auxiliary（借助 target）
        2. 把最底下的最大盘子从 source 移到 target
        3. 把 n-1 个盘子从 auxiliary 移到 target（借助 source）
    """
    global move_count
    
    # 基准情况：只有一个盘子，直接移
    if n == 1:
        move_count += 1
        print(f"  第{move_count}步: {source} -> {target}  (移动盘子1)")
        return
    
    # 递归情况：三步走
    
    # 第一步：把上面 n-1 个盘子移到辅助柱
    hanoi(n - 1, source, auxiliary, target)
    
    # 第二步：把最底下的盘子移到目标柱
    move_count += 1
    print(f"  第{move_count}步: {source} -> {target}  (移动盘子{n})")
    
    # 第三步：把 n-1 个盘子从辅助柱移到目标柱
    hanoi(n - 1, auxiliary, target, source)


if __name__ == "__main__":
    print("=== 汉诺塔问题 ===\n")
    
    print("--- 3个盘子 ---")
    move_count = 0
    hanoi(3, 'A', 'C', 'B')
    print(f"  总共移动 {move_count} 步 (2^3 - 1 = 7)")
    # 输出:
    #   第1步: A -> C  (移动盘子1)
    #   第2步: A -> B  (移动盘子2)
    #   第3步: C -> B  (移动盘子1)
    #   第4步: A -> C  (移动盘子3)
    #   第5步: B -> A  (移动盘子1)
    #   第6步: B -> C  (移动盘子2)
    #   第7步: A -> C  (移动盘子1)
    #   总共移动 7 步 (2^3 - 1 = 7)
    
    print()
    
    print("--- 4个盘子 ---")
    move_count = 0
    hanoi(4, 'A', 'C', 'B')
    print(f"  总共移动 {move_count} 步 (2^4 - 1 = 15)")
    
    print()
    
    # 步数规律
    print("--- 步数规律 ---")
    for n in range(1, 7):
        moves = 2 ** n - 1
        print(f"  {n}个盘子需要 {moves} 步")
    # 输出:
    #   1个盘子需要 1 步
    #   2个盘子需要 3 步
    #   3个盘子需要 7 步
    #   4个盘子需要 15 步
    #   5个盘子需要 31 步
    #   6个盘子需要 63 步
```

**教授点评**：汉诺塔之所以用递归如此优雅，是因为问题本身就是递归定义的——"移 n 个盘子" = "移 n-1 个 + 移1个 + 移 n-1 个"。迭代写法？……你还是别想了，那代码会像一团意大利面条。

### 示例3: 递归调用栈可视化

让我们看看递归在底层是如何工作的——调用栈是如何增长和收缩的。

```python
# 递归调用栈可视化
# 展示递归过程中调用栈的增长和收缩

def visualize_factorial(n, depth=0):
    """
    可视化阶乘递归的调用栈过程
    depth 参数用来缩进，展示递归深度
    """
    indent = "  " * depth  # 根据深度缩进
    
    # 打印进入信息
    print(f"{indent}>>> 进入 factorial({n})")
    
    # 基准情况
    if n <= 1:
        print(f"{indent}    基准情况: 返回 1")
        print(f"{indent}<<< 退出 factorial({n}), 返回 1")
        return 1
    
    # 递归调用
    print(f"{indent}    递归调用: {n} * factorial({n-1})")
    result = n * visualize_factorial(n - 1, depth + 1)
    
    # 打印返回信息
    print(f"{indent}    计算: {n} * {result // n} = {result}")
    print(f"{indent}<<< 退出 factorial({n}), 返回 {result}")
    
    return result


def factorial_simple(n):
    """
    简洁版阶乘递归（对比用）
    """
    if n <= 1:
        return 1
    return n * factorial_simple(n - 1)


if __name__ == "__main__":
    print("=== 递归调用栈可视化 ===\n")
    print("--- 计算 4! 的调用过程 ---\n")
    
    result = visualize_factorial(4)
    
    print(f"\n最终结果: 4! = {result}")
    # 输出:
    # === 递归调用栈可视化 ===
    #
    # --- 计算 4! 的调用过程 ---
    #
    # >>> 进入 factorial(4)
    #     递归调用: 4 * factorial(3)
    #   >>> 进入 factorial(3)
    #       递归调用: 3 * factorial(2)
    #     >>> 进入 factorial(2)
    #         递归调用: 2 * factorial(1)
    #       >>> 进入 factorial(1)
    #           基准情况: 返回 1
    #       <<< 退出 factorial(1), 返回 1
    #         计算: 2 * 1 = 2
    #       <<< 退出 factorial(2), 返回 2
    #       计算: 3 * 2 = 6
    #     <<< 退出 factorial(3), 返回 6
    #     计算: 4 * 6 = 24
    #   <<< 退出 factorial(4), 返回 24
    #
    # 最终结果: 4! = 24
    
    print()
    print("--- 递归深度限制演示 ---")
    try:
        # Python 默认递归深度限制大约 1000 层
        factorial_simple(2000)
    except RecursionError as e:
        print(f"  错误: {e}")
        print("  提示: Python 默认有递归深度限制，防止栈溢出")
    
# 预期输出:
#   错误: maximum recursion depth exceeded
#   提示: Python 默认有递归深度限制，防止栈溢出
```

**教授点评**：看可视化输出，是不是清楚看到递归的**两个阶段**？第一阶段叫"递"（一路深入到底），第二阶段叫"归"（一路返回并计算）。这就是为什么叫"递归"——**先递出去，再归回来**。

## 深入理解

### 工作原理：调用栈的秘密

每次函数调用，Python（以及几乎所有编程语言）都会在内存的**调用栈**上创建一个**栈帧**（Stack Frame）。栈帧里存放：

- **函数参数**：这次调用传入了什么值
- **局部变量**：函数内部定义的变量
- **返回地址**：函数结束后回到哪里继续执行

递归函数调用自己时，就在栈上又压入一个新栈帧。层层叠加，直到基准情况触发，然后逐层弹出。

```
调用栈的增长和收缩（以 factorial(3) 为例）:

压栈阶段（"递"）:          弹出阶段（"归"）:
[ factorial(3) ]            ← 计算 3*2=6, 弹出
[ factorial(2) ]            ← 计算 2*1=2, 弹出
[ factorial(1) ]            ← 返回 1, 弹出
[     main     ]            [     main     ]
```

### 递归深度与栈溢出

栈的空间是**有限的**。Python 默认递归深度限制约 1000 层。如果递归太深，就会栈溢出：

```python
import sys
print(f"Python 最大递归深度: {sys.getrecursionlimit()}")
# 输出: Python 最大递归深度: 1000
```

> **注意**：你可以用 `sys.setrecursionlimit()` 调高限制，但这是饮鸩止渴。栈的物理空间有限，强行突破会导致真正的段错误（Segmentation Fault）。

### 时间复杂度 vs 空间复杂度

| 算法 | 时间复杂度 | 空间复杂度（调用栈） |
|------|-----------|-------------------|
| 朴素斐波那契 | O(2^n) | O(n) |
| 记忆化斐波那契 | O(n) | O(n) |
| 阶乘递归 | O(n) | O(n) |
| 汉诺塔 | O(2^n) | O(n) |
| 快速排序（平均） | O(n log n) | O(log n) |

### 递归 vs 迭代的哲学对比

递归是**声明式**思维："我要什么"；迭代是**命令式**思维："我怎么一步步做到"。

```python
# 声明式（递归）：我要的是 n * (n-1)!
def factorial_recursive(n):
    if n <= 1:
        return 1
    return n * factorial_recursive(n - 1)

# 命令式（迭代）：我从1乘到n
def factorial_iterative(n):
    result = 1
    for i in range(2, n + 1):
        result *= i
    return result
```

哪种更好？取决于场景。阶乘用迭代就够了；但汉诺塔，你试试用迭代写……祝你好运。

## 实际应用

### 应用场景1: 文件系统遍历

你用过 `find` 命令或者在文件管理器里搜索文件吗？文件系统本质是一棵树，遍历树最自然的方式就是递归。

```python
import os

def list_files(directory, indent=""):
    """递归列出目录下所有文件"""
    print(f"{indent}📁 {os.path.basename(directory)}")
    for item in os.listdir(directory):
        path = os.path.join(directory, item)
        if os.path.isdir(path):
            list_files(path, indent + "  ")  # 递归进入子目录
        else:
            print(f"{indent}  📄 {item}")
```

### 应用场景2: JSON/XML 解析

结构化数据（JSON、XML、HTML DOM）都是嵌套的树形结构。递归遍历和解析这些数据结构是最自然的方式。

```python
def count_json_depth(obj):
    """递归计算 JSON 对象的最大嵌套深度"""
    if isinstance(obj, dict):
        if not obj:
            return 1
        return 1 + max(count_json_depth(v) for v in obj.values())
    elif isinstance(obj, list):
        if not obj:
            return 1
        return 1 + max(count_json_depth(v) for v in obj)
    return 0  # 叶子节点
```

### 应用场景3: 分形图形绘制

自然界中很多图形都是自相似的（分形）：树枝分叉、雪花晶体、海岸线。用递归可以简洁地描述这种自相似性。

## 常见误区

⚠️ **注意这些坑！**

### 误区1: "递归就是炫技，没用"

**错误理解**：
> 递归代码又慢又难懂，循环不香吗？

**真相**：
递归不是炫技，而是一种**思维方式**。有些问题天然就是递归定义的（树遍历、分治、回溯），用递归写不仅更简洁，而且**更符合人类直觉**。用迭代反而像在"逆着思维方向走"。

✅ **正确做法**：
- 能用循环简单解决的问题，用循环
- 树/图遍历、分治算法、回溯搜索，优先考虑递归

### 误区2: "忘了写基准情况"

**错误理解**：
> 我写了递归调用，但忘记处理最小的情况了

```python
def sum_to_n(n):
    """错误示范：没有基准情况"""
    return n + sum_to_n(n - 1)  # 永远停不下来！
    
# 运行结果: RecursionError
```

**真相**：
没有基准情况的递归，就是无限循环的优雅说法。

✅ **正确做法**：
写递归时，**第一步永远是写基准情况**。先想"什么时候停"，再想"怎么递"。

### 误区3: "递归没有返回值"

**错误理解**：
> 递归调用的结果我忘了接收，或者忘了 return

```python
def find_max(lst):
    """错误示范：递归调用了但没用返回值"""
    if len(lst) == 1:
        return lst[0]
    find_max(lst[1:])  # 计算了但没有 return！
    # 隐式返回 None
```

✅ **正确做法**：
确保每个分支都有明确的返回值：

```python
def find_max(lst):
    if len(lst) == 1:
        return lst[0]
    return max(lst[0], find_max(lst[1:]))  # 正确：接收并使用返回值
```

### 练习3: 递归生成全排列 ⭐⭐⭐

**描述**：

- 递归情况：固定第一个元素，递归生成剩余元素的排列，然后拼接
- 返回列表的列表

**示例**：

<details>
<summary>💡 提示（点击展开）</summary>

- 遍历每个元素，把它作为"第一个元素"

- 把"第一个元素"插入到每个子排列的最前面
- n 个元素的排列数是 n!，所以时间复杂度至少是 O(n!)

## 小结与思考

### 本节要点

1. **递归 = 函数调用自己** —— 但必须有基准情况（刹车），否则会栈溢出
2. **递归三要素** —— 基准情况（什么时候停）、递归情况（怎么变小）、向基准收敛（保证能停）
3. **记忆化优化** —— "算过的记下来"，用空间换时间，O(2^n) 可以降到 O(n)
4. **调用栈** —— 每次调用都在栈上创建新帧，"递"是压栈，"归"是弹栈
5. **递归不是万能药** —— 能用循环解决就别递归，但树/分治/回溯用递归最优雅

### 思考题

1. **递归和循环可以互相转换吗？** 理论上是的，那么编译器能不能自动把尾递归优化成循环？为什么 Python 不支持尾递归优化？

2. **汉诺塔问题中，如果 n=64（传说中的黄金盘子数量），需要多少步？** 假设每秒移动一次，需要多少年才能移完？这和我们宇宙的年龄相比如何？

3. **递归的本质是把问题"变小"，但变小的方式有很多种。** 你能想到除了"减1"之外，还有哪些问题规模缩小的方式吗？（提示：二分查找、快速排序是怎么缩小问题的？）
