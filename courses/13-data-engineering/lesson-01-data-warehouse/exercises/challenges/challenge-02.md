# 实现缓慢变化维Type 2 ⭐⭐⭐⭐

## 背景
在数据仓库中，维度表的数据会随时间发生变化。对于重要的业务维度（如产品信息、客户信息），我们需要追踪这些变化的历史记录，以便进行准确的历史分析。SCD Type 2是处理这种情况的标准方法。

## 要求

### 1. 表结构设计
实现一个支持SCD Type 2的产品维度表，包含以下字段：
- **代理键 (Surrogate Key)**: 整数主键，用于事实表关联
- **业务键 (Natural Key)**: 产品ID，来自源系统的唯一标识
- **产品属性**: 产品名称、类别、品牌、价格等
- **版本控制字段**: 
  - 生效日期 (effective_date)
  - 失效日期 (expiry_date)  
  - 是否当前版本 (is_current)
  - 版本号 (version)

### 2. 核心功能实现
实现以下Python函数：

#### `insert_initial_version(conn, product_data)`
- 插入产品的初始版本
- 设置生效日期为指定日期，失效日期为'9999-12-31'
- is_current设置为True，version设置为1

#### `update_product_version(conn, product_id, new_attributes, effective_date)`
- 当产品属性发生变化时调用
- 将当前版本的is_current设为False，expiry_date设为effective_date
- 插入新版本，设置相应的属性和版本控制字段

#### `get_current_products(conn)`
- 查询所有当前有效的产品信息
- 只返回is_current=True的记录

#### `get_products_as_of_date(conn, as_of_date)`
- 查询指定日期有效的产品信息
- 返回在as_of_date处于生效期间的记录
