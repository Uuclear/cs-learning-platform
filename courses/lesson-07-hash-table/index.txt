<h2>核心概念</h2>
<h3>什么是哈希表？</h3>
<p>哈希表（Hash Table），也叫散列表，是一种通过**键值对（Key-Value Pair）**来存储和访问数据的数据结构。</p>
<p>核心思想很简单：</p>
<ol>
<li>你给数据起个"名字"（Key）</li>
<li>通过一个"翻译规则"（Hash Function）把名字变成一个数字</li>
<li>用这个数字作为位置，把数据放进去或取出来</li>
</ol>
<p>用生活化的比喻：</p>
<ul>
<li>🏪 <strong>超市存包柜</strong>：你存包时拿到一个号码牌（Key），号码决定了你的包放在哪个柜子（Value）</li>
<li>📱 <strong>手机通讯录</strong>：输入"张三"（Key），瞬间弹出他的电话（Value）</li>
<li>🗂️ <strong>公司花名册</strong>：按工号（Key）查员工信息（Value）</li>
</ul>
<h3>哈希函数：神奇的翻译官</h3>
<p>哈希函数（Hash Function）是哈希表的心脏。它的作用是：<strong>把任意长度的输入，映射成一个固定范围的整数</strong>。</p>
<pre><code>哈希函数(key) → 索引位置
</code></pre>
<p>一个<strong>好的</strong>哈希函数需要具备以下特征：</p>
<ol>
<li><strong>确定性</strong>：相同的输入永远产生相同的输出。不能今天映射到5号柜，明天变成12号柜</li>
<li><strong>均匀分布</strong>：尽量把数据均匀分散到各个位置，不要都挤在一起</li>
<li><strong>计算快速</strong>：映射过程要快，不然哈希表就没优势了</li>
<li><strong>输入敏感</strong>：稍有差别的输入产生不同的输出（理想情况）</li>
</ol>
<p>举个简单的例子：</p>
<pre><code class="language-python"># 一个简单的哈希函数：把字符串每个字符的ASCII值加起来，然后取余
def simple_hash(key, table_size):
    total = 0
    for char in key:
        total += ord(char)  # 获取字符的ASCII值
    return total % table_size

# 测试
print(simple_hash("apple", 10))   # 输出: 4
print(simple_hash("banana", 10))  # 输出: 1
print(simple_hash("cherry", 10))  # 输出: 9
</code></pre>
<blockquote>
<p>这个哈希函数很粗糙，但原理一目了然：先算出一个数字，再用取余（%）操作把数字限制在表的范围内。</p>
</blockquote>
<h3>哈希碰撞：当两个钥匙开同一把锁</h3>
<p>现实是残酷的——再好的哈希函数也<strong>无法完全避免碰撞（Collision）</strong>。</p>
<p>碰撞：<strong>两个不同的Key，经过哈希函数计算后，得到了相同的索引位置</strong>。</p>
<pre><code>table_size = 10

哈希("apple") → 4
哈希("eppla") → 4  ← 碰撞！两个不同的词都指向位置4
</code></pre>
<p>这就像超市存包柜：你拿着3号柜的牌子去存包，发现里面已经有一个包了。咋办？</p>
<p>好消息是，计算机科学家们想出了聪明的解决方案。我们马上来看！</p>

<h3>示例2: 手写哈希表 + 链地址法</h3>
<pre><code class="language-python"># ============================================
# 示例2：手写哈希表 - 链地址法解决碰撞
# 理解哈希表的底层实现原理
# ============================================

class HashTable:
    """
    手写哈希表，使用链地址法解决碰撞
    核心思路：每个位置是一个链表，碰撞了就往链尾挂
    """

    def __init__(self, size=10):
        # 初始化桶数组，每个桶是一个空列表（链表）
        self.size = size
        self.buckets = [[] for _ in range(size)]
        self.count = 0  # 当前存储的键值对数量
        print(f"🎉 创建了大小为 {size} 的哈希表")

    def _hash(self, key):
        """
        哈希函数：将键映射到桶的索引
        Python内置的hash()已经很好了，这里用它
        """
        return hash(key) % self.size

    def put(self, key, value):
        """
        插入或更新键值对
        如果键已存在，就更新值；否则新建
        """
        idx = self._hash(key)
        bucket = self.buckets[idx]

        # 检查这个键是否已经存在
        for i, (k, v) in enumerate(bucket):
            if k == key:
                bucket[i] = (key, value)  # 更新值
                print(f"  更新 [{key}] → {value} (桶{idx})")
                return

        # 键不存在，添加到链表尾部
        bucket.append((key, value))
        self.count += 1
        print(f"  插入 [{key}] → {value} (桶{idx})")

        # 检查是否需要扩容
        if self.count &gt; self.size * 0.7:
            self._resize()

    def get(self, key, default=None):
        """
        查找键对应的值
        找不到就返回默认值
        """
        idx = self._hash(key)
        bucket = self.buckets[idx]

        for k, v in bucket:
            if k == key:
                return v

        return default

    def remove(self, key):
        """
        删除键值对
        """
        idx = self._hash(key)
        bucket = self.buckets[idx]

        for i, (k, v) in enumerate(bucket):
            if k == key:
                del bucket[i]
                self.count -= 1
                print(f"  删除 [{key}] (桶{idx})")
                return True

        print(f"  警告: [{key}] 不存在！")
        return False

    def _resize(self):
        """
        扩容操作：桶数量翻倍，重新哈希所有元素
        """
        old_size = self.size
        old_buckets = self.buckets

        self.size *= 2
        self.buckets = [[] for _ in range(self.size)]
        self.count = 0

        print(f"  🔧 扩容: {old_size} → {self.size}，重新哈希...")

        # 把所有旧数据重新哈希到新表
        for bucket in old_buckets:
            for key, value in bucket:
                self.put(key, value)

    def display(self):
        """
        可视化展示哈希表的状态
        """
        print("\n=== 哈希表状态 ===")
        for i, bucket in enumerate(self.buckets):
            if bucket:
                pairs = " → ".join(f"[{k}:{v}]" for k, v in bucket)
                print(f"  桶{i}: {pairs}")
            else:
                print(f"  桶{i}: (空)")
        print(f"  总计: {self.count} 个键值对, 大小: {self.size}")
        print("=" * 30)


# 测试手写哈希表
print("=== 测试手写哈希表（链地址法）===")
ht = HashTable(size=5)

# 插入一些键值对
ht.put("name", "小明")
ht.put("age", 18)
ht.put("city", "北京")
ht.put("hobby", "编程")
ht.put("language", "Python")

# 查找
print(f"\n查找 'name': {ht.get('name')}")
print(f"查找 'city': {ht.get('city')}")
print(f"查找 '不存在的键': {ht.get('nope', '找不到！')}")

# 更新
ht.put("name", "小红")
print(f"\n更新后 'name': {ht.get('name')}")

# 删除
ht.remove("age")

# 展示最终状态
ht.display()
</code></pre>

def word_frequency(text):
    """
    统计文本中每个单词出现的频率
    哈希表天然适合这种"键→计数"的场景
    """
    # 清理文本：转小写，按空格拆分
    words = text.lower().split()

    freq = {}
    for word in words:
        # 去掉标点符号
        word = word.strip(".,!?;:\"'()[]{}")
        if word:
            freq[word] = freq.get(word, 0) + 1

    return freq


def print_top_words(freq, top_n=5):
    """
    打印出现频率最高的N个单词

    # 按频率排序
    sorted_words = sorted(freq.items(), key=lambda x: x[1], reverse=True)

    print(f"词频统计 TOP {top_n}:")
    print("-" * 30)
    for i, (word, count) in enumerate(sorted_words[:top_n], 1):
        bar = "█" * count
        print(f"  {i:2d}. {word:15s} ({count:3d}次) {bar}")

<h2>深入理解</h2>
<h3>哈希表的灵魂：为什么这么快？</h3>
<p>哈希表能在**O(1)**时间内完成插入、查找、删除，这个速度在所有常见数据结构中是最快的（之一）！</p>
<p>秘密就在于它<strong>跳过了比较</strong>，直接定位。</p>
<p>想象你要在人群中找张三：</p>
<ul>
<li><strong>数组/链表</strong>：从第一个人开始问"你是张三吗？"——最坏要问n次，<strong>O(n)</strong></li>
<li><strong>排序数组+二分查找</strong>：每次排除一半——log n次，<strong>O(log n)</strong></li>
<li><strong>哈希表</strong>：直接算出张三站在哪个位置，走过去——<strong>O(1)</strong></li>
</ul>
<pre><code>找"张三":
  数组: [问]→[问]→[问]→[问]→[问]→[问]→[找到!]  ← 要问6次
  哈希: [计算哈希]→直接走过去→[找到!]  ← 只算1次
</code></pre>
<h3>碰撞的三种解决方案</h3>
<h4>方案1：链地址法（Separate Chaining）⭐ 最常用</h4>
<p>每个桶里存一个<strong>链表</strong>。碰撞时，把新元素挂在链表后面。</p>
<pre><code>桶4: [apple:"红色"] → [eppla:"一种乐器"] → [pplea:"一种水果"]
</code></pre>
<ul>
<li><strong>优点</strong>：简单直观，链表长度可以无限增长</li>
<li><strong>缺点</strong>：链表太长时，查找变成O(n)</li>
<li><strong>最佳实践</strong>：当链平均长度超过某个阈值（如0.7），就扩容</li>
</ul>
<h4>方案2：开放地址法（Open Addressing）</h4>
<p>不引入链表，而是在哈希表中<strong>找下一个空桶</strong>来存放碰撞的元素。</p>
<p>常见的探测策略：</p>
<ul>
<li><strong>线性探测</strong>：位置满了就看下一个（+1, +2, +3...）</li>
<li><strong>二次探测</strong>：位置满了就按平方跳跃（+1, +4, +9...）</li>
<li><strong>双重哈希</strong>：用第二个哈希函数计算步长</li>
</ul>
<pre><code>线性探测:
  桶4: [apple:"红色"]
  桶4的下一个: [eppla:"一种乐器"]  ← 本来也该放桶4，但满了所以放桶5
  桶5的下一个: [pplea:"一种水果"]  ← 桶4和5都满了，所以放桶6
</code></pre>
<ul>
<li><strong>优点</strong>：不需要额外数据结构（链表），缓存友好</li>
<li><strong>缺点</strong>：负载因子高时性能急剧下降，删除操作复杂</li>
<li><strong>注意</strong>：装得太满时要及时扩容！</li>
</ul>
<h4>方案3：再哈希法（Double Hashing）</h4>
<p>碰撞时用<strong>第二个不同的哈希函数</strong>重新计算位置。</p>
<pre><code>第一次: hash1("eppla") → 4 (冲突!)
第二次: hash2("eppla") → 7 (放到桶7)
</code></pre>
<ul>
<li><strong>优点</strong>：减少"聚集"现象（多个元素挤在一起）</li>
<li><strong>缺点</strong>：计算成本高，需要设计第二个哈希函数</li>
</ul>
<h3>负载因子（Load Factor）：何时扩容？</h3>
<p>负载因子 = 元素数量 / 表大小</p>
<p>它决定了哈希表有多"满"。负载因子越高，碰撞概率越大。</p>
<pre><code>表大小=10，已有7个元素:
  负载因子 = 7/10 = 0.7  ← 通常在这个阈值扩容

表大小=10，已有9个元素:
  负载因子 = 9/10 = 0.9  ← 已经很挤了！碰撞频繁！
</code></pre>
<p><strong>经验法则</strong>：负载因子超过0.7时就扩容（通常是翻倍）。</p>

<h2>实际应用</h2>
<p>哈希表无处不在！看看它在你每天用的软件里扮演什么角色：</p>
<h3>应用场景1: Python的dict —— 你每天都在用</h3>
<p>Python的<code>dict</code>就是哈希表的官方实现！每次你写<code>d[key] = value</code>时，Python底层就在做哈希映射。</p>
<pre><code class="language-python"># 这就是在操作哈希表
config = {
    "database": "localhost",
    "port": 5432,
    "username": "admin"
}

print(config["database"])  # 哈希表查找
config["password"] = "xxx"  # 哈希表插入
</code></pre>
<p>Python 3.7+的dict还<strong>保持了插入顺序</strong>，这是额外加了一个双向链表来实现的。</p>
<h3>应用场景2: 数据库索引</h3>
<p>数据库的<strong>哈希索引</strong>是最基础的索引类型。当你用<code>WHERE id = 123</code>查询时，如果id字段有哈希索引，数据库可以直接定位到数据行，而不是全表扫描。</p>
<pre><code class="language-sql">-- 有哈希索引：瞬间定位
SELECT * FROM users WHERE email = 'user@example.com';
-- 内部过程: hash("user@example.com") → 直接跳到那一行
</code></pre>
<p>当然，数据库更常用的是B+树索引（支持范围查询），但哈希索引在等值查询中更快。</p>
<h3>应用场景3: 缓存系统</h3>
<p>缓存（Cache）的本质就是一个<strong>大哈希表</strong>：Key是请求URL或数据ID，Value是缓存的结果。</p>
<pre><code class="language-python"># 简易缓存实现
cache = {}

def get_user_info(user_id):
    # 先看缓存有没有
    if user_id in cache:
        return cache[user_id]  # 缓存命中！

    # 没有就查数据库
    data = query_database(user_id)
    cache[user_id] = data  # 写入缓存
    return data
</code></pre>
<p>Redis、Memcached这些专业缓存系统，底层都是大规模哈希表。</p>
<h3>应用场景4: 密码存储</h3>
<p>你注册网站时，网站<strong>不应该存你的明文密码</strong>。正确的做法是：存密码的<strong>哈希值</strong>。</p>
<pre><code class="language-python">import hashlib

# 用户注册：存密码的哈希值
password = "my_secret_password"
stored_hash = hashlib.sha256(password.encode()).hexdigest()
print(f"存储的哈希: {stored_hash}")
# 存储的哈希: 652c...（固定长度的字符串）

# 用户登录：验证密码
def verify_password(input_pwd, stored_hash):
    input_hash = hashlib.sha256(input_pwd.encode()).hexdigest()
    return input_hash == stored_hash  # 哈希值相同就登录成功
</code></pre>
<p>哈希函数是<strong>单向的</strong>：你能从密码算出哈希值，但不能从哈希值反推密码。这就是密码安全的基础。</p>
<h3>应用场景5: 集合去重</h3>
<p>Python的<code>set</code>底层也是哈希表！利用哈希表的"键唯一"特性，可以轻松去重。</p>
<pre><code class="language-python"># 利用集合去重
numbers = [1, 2, 2, 3, 3, 3, 4, 4, 4, 4]
unique = set(numbers)
print(f"去重前: {numbers}")
print(f"去重后: {unique}")
# 去重后: {1, 2, 3, 4}
</code></pre>

<h2>常见误区</h2>
<p>⚠️ <strong>注意这些坑！</strong></p>
<h3>误区1: 哈希表的查找永远是O(1)</h3>
<p><strong>错误理解</strong>：<br />"哈希表查找是O(1)，所以比数组快！"</p>
<p><strong>真相</strong>：<br />哈希表的O(1)是<strong>平均情况</strong>。最坏情况下（所有键都碰撞到同一个桶），查找退化成<strong>O(n)</strong>——和链表一样慢！</p>
<pre><code>极端碰撞情况:
桶3: [key1] → [key2] → [key3] → [key4] → ... → [key1000]
找key1000需要遍历整个链 → O(n)！
</code></pre>
<p>✅ <strong>正确做法</strong>：</p>
<ul>
<li>选择好的哈希函数，减少碰撞</li>
<li>控制负载因子在0.7以下，及时扩容</li>
<li>对于关键场景，要有性能监控和回退方案</li>
</ul>
<h3>误区2: 哈希表是有序的</h3>
<p><strong>错误理解</strong>：</p>
<pre><code class="language-python">d = {"c": 3, "a": 1, "b": 2}
for key in d:
    print(key)  # 以为会按a, b, c的顺序输出？
</code></pre>
<p><strong>真相</strong>：<br />传统哈希表<strong>不保证插入顺序</strong>。你存进去的顺序和遍历时看到的顺序可能完全不同！</p>
<blockquote>
<p>注：Python 3.7+的dict确实保持了插入顺序，但这是语言特性，不代表哈希表本身有序。Java的HashMap也是无序的（需要用LinkedHashMap才有序）。</p>
</blockquote>
<p>✅ <strong>正确做法</strong>：</p>
<ul>
<li>需要有序遍历时，用排序后的键列表：<code>for key in sorted(d.keys())</code></li>
<li>或者使用专门的有序数据结构（如TreeMap、BTreeMap）</li>
</ul>
<h3>误区3: 可以用可变对象做哈希表的Key</h3>
<p><strong>错误代码</strong>：</p>
<pre><code class="language-python"># ❌ 危险操作！用列表做字典的key
d = {}
key = [1, 2, 3]
d[key] = "value"  # TypeError: unhashable type: 'list'

# 就算某些语言允许了，后续修改key会更灾难：
key.append(4)  # key变了，但哈希值没重新计算！
print(d[key])  # 找不到原来的值了！
</code></pre>
<p><strong>为什么会这样</strong>：<br />哈希表依赖key的哈希值来定位。如果key可变，你改了key之后，原来的哈希值就"过期"了，数据找不回来了。</p>
<p>✅ <strong>正确做法</strong>：</p>
<pre><code class="language-python"># ✅ 用不可变对象做key
d = {}
d[(1, 2, 3)] = "value"  # 元组是不可变的，安全！
d["name"] = "小明"       # 字符串也是不可变的

# 如果必须用列表，先转成元组
key_list = [1, 2, 3]
d[tuple(key_list)] = "value"
</code></pre>

<h2>动手实践</h2>
<p>现在轮到你动手了！</p>
<h3>练习1: 两数之和 ⭐</h3>
<p><strong>描述</strong>：<br />给定一个整数数组和一个目标值，找出数组中两个数，使它们的和等于目标值。返回这两个数的索引。</p>
<p><strong>要求</strong>：</p>
<ul>
<li>输入: <code>nums = [2, 7, 11, 15]</code>, <code>target = 9</code></li>
<li>输出: <code>[0, 1]</code>（因为nums[0] + nums[1] = 2 + 7 = 9）</li>
<li>假设每组输入恰好有一个答案</li>
<li>不能使用同一个元素两次</li>
<li><strong>提示</strong>：用哈希表可以在O(n)时间内解决！</li>
</ul>

💡 提示（点击展开）<ol>
<li>遍历数组，对于每个数<code>num</code>，检查<code>target - num</code>是否已经在哈希表中</li>
<li>如果在，说明找到了答案</li>
<li>如果不在，把当前数和它的索引存入哈希表</li>
<li>这样只需要遍历一次数组！</li>
</ol>
<h3>练习2: 有效的字母异位词 ⭐⭐</h3>
<p><strong>描述</strong>：<br />给定两个字符串s和t，判断t是否是s的字母异位词（即：两个字符串包含相同的字符，只是排列不同）。</p>
<p><strong>要求</strong>：</p>
<ul>
<li>输入: <code>s = "anagram"</code>, <code>t = "nagaram"</code></li>
<li>输出: <code>True</code>（因为两个字符串都包含a:3, n:1, g:1, r:1, m:1）</li>
<li>输入: <code>s = "rat"</code>, <code>t = "car"</code></li>
<li>输出: <code>False</code></li>
<li>使用哈希表统计字符频率来比较</li>
</ul>

💡 提示（点击展开）<ol>
<li>如果两个字符串长度不同，直接返回False</li>
<li>用哈希表统计s中每个字符的出现次数</li>
<li>再遍历t，每遇到一个字符就在哈希表中减1</li>
<li>如果某个字符不存在或计数为0，返回False</li>
<li>或者更简单：统计两个字符串的字符频率，直接比较两个哈希表是否相等</li>
</ol>
<h3>练习3: 设计LRU缓存 ⭐⭐⭐</h3>
<p><strong>描述</strong>：<br />实现一个LRU（Least Recently Used，最近最少使用）缓存。它有固定容量，当容量满了时，淘汰最久没有访问过的元素。</p>
<p><strong>要求</strong>：</p>
<ul>
<li><code>get(key)</code>：如果key存在，返回值并标记为"最近使用"；不存在返回-1</li>
<li><code>put(key, value)</code>：插入或更新键值对。如果满了，淘汰最久未使用的元素</li>
<li><code>get</code>和<code>put</code>操作都必须是**O(1)**时间复杂度</li>
<li>容量: 2</li>
</ul>
<p>示例操作序列：</p>
<pre><code>put(1, 1)    → 缓存: {1: 1}
put(2, 2)    → 缓存: {1: 1, 2: 2}
get(1)       → 返回 1，缓存: {2: 2, 1: 1}（1被标记为最近使用）
put(3, 3)    → 淘汰2（最久未用），缓存: {1: 1, 3: 3}
get(2)       → 返回 -1（2已被淘汰）
put(4, 4)    → 淘汰1，缓存: {3: 3, 4: 4}
get(1)       → 返回 -1
get(3)       → 返回 3
get(4)       → 返回 4
</code></pre>

💡 提示（点击展开）<ol>
<li>需要两个数据结构配合：<strong>哈希表 + 双向链表</strong></li>
<li>哈希表负责O(1)查找，双向链表负责O(1)移动节点（标记最近使用）</li>
<li>链表头部放"最近使用"的节点，尾部放"最久未用"的节点</li>
<li><code>get</code>操作：从哈希表找到节点，移到链表头部</li>
<li><code>put</code>操作：如果key已存在，更新值并移到头部；如果是新key，插入头部；如果满了，删除尾部节点</li>
</ol>
<blockquote>
<p>这是LeetCode 146题，经典的中等难度题！Python可以用<code>collections.OrderedDict</code>来简化实现。</p>
</blockquote>
