8:T466a,<h1>树：倒着长的家族谱</h1>
<p>上一节课我们学了链表——小朋友们手拉手排成一队。今天我们要学的数据结构可就厉害了，它让小朋友不再排成一条线，而是<strong>分叉、分叉、再分叉</strong>，像一棵树一样伸展开来。</p>
<hr />
<h2>开场白</h2>
<p>想象一下你打开家谱。你的上面是父母，父母的上面是爷爷奶奶、外公外婆……再往上是祖宗十八代。你会发现：<strong>一个人可能有多个后代，但只有一个父母</strong>（生物学意义上的，别跟我杠）。</p>
<p>再想想你们公司的组织架构图：CEO在最上面，下面有几个VP，每个VP下面有总监，总监下面有经理，经理下面有你。</p>
<p>这两种结构有个共同点：<strong>从上到下，不断分叉，而且没有环</strong>。在CS的世界里，我们把这种结构叫做——<strong>树（Tree）</strong>。</p>
<blockquote>
<p>💡 <strong>思考问题1</strong>：链表是"一条线"，树是"分叉线"。如果让你设计一个能存家谱的数据结构，你会怎么做？</p>
</blockquote>
<blockquote>
<p>💡 <strong>思考问题2</strong>：为什么公司的组织架构图不能出现环？如果A管B，B管C，C管A，会发生什么？</p>
</blockquote>
<hr />
<h2>核心概念</h2>
<h3>什么是树？</h3>
<p>树是一种<strong>非线性</strong>的数据结构，它由<strong>节点</strong>（node）和<strong>边</strong>（edge）组成。</p>
<p>简单说：<strong>树就是一堆点，点之间用线连起来，而且没有环</strong>。</p>
<p>更正式一点：</p>
<ul>
<li>树要么为空（没有节点）</li>
<li>要么有一个<strong>根节点</strong>（root），以及若干互不相交的<strong>子树</strong>（subtree）</li>
</ul>
<p>这个定义本身就很"递归"——"子树本身也是一棵树"。套娃文学在CS中屡见不鲜。</p>

<p>一个容易混淆的点：<strong>树的"根"在上面，"叶子"在下面——这跟现实中的树是倒过来的！</strong> 所以CS的树是"倒着长的"。这是历史原因，早期数学家画树就是这么画的，大家就跟着这么叫了。</p>
<h3>二叉树</h3>
<p><strong>二叉树</strong>是每个节点最多有两个子节点的树——一个左孩子，一个右孩子。</p>
<pre><code>       A
     /   \
    B     C
   / \   / \
  D   E F   G
</code></pre>
<p>这就是二叉树的经典画。每个节点最多牵着两根线。</p>
<p>二叉树有几个特殊类型：</p>
<table>
<thead>
<tr>
<th>类型</th>
<th>定义</th>
<th>特点</th>
</tr>
</thead>
<tbody><tr>
<td><strong>满二叉树</strong></td>
<td>所有层的节点都满了</td>
<td>第k层有2^k个节点</td>
</tr>
<tr>
<td><strong>完全二叉树</strong></td>
<td>除了最后一层，其他层都满；最后一层从左到右排列</td>
<td>堆就是完全二叉树</td>
</tr>
<tr>
<td><strong>完美二叉树</strong></td>
<td>所有叶子在同一层，所有非叶子都有两个子节点</td>
<td>满二叉树+叶子都在最底层</td>
</tr>
</tbody></table>
<p><strong>二叉树的几个重要性质：</strong></p>
<ul>
<li>第i层最多有 <strong>2^(i-1)</strong> 个节点（根在第1层）</li>
<li>高度为h的二叉树最多有 <strong>2^h - 1</strong> 个节点</li>
<li>叶子节点数 = 度为2的节点数 + 1</li>
</ul>

         1
       /   \
      2     3
     / \   /

后序：左右根 —— 先逛完，最后回来
层序：一层层 —— 从上到下，从左到右

<p><strong>为什么要有四种遍历？</strong> 因为不同的场景需要不同的访问顺序：</p>
<ul>
<li><strong>前序遍历</strong>：复制一棵树。先造根，再造左右。就像盖房子先打地基再盖上面。</li>
<li><strong>中序遍历</strong>：BST中序遍历会得到有序序列。这是BST的核心性质。</li>
<li><strong>后序遍历</strong>：删除一棵树。先删子树，再删根。就像拆房子从顶往下拆。</li>
<li><strong>层序遍历</strong>：BFS搜索、求树的宽度。一层一层扫过去。</li>
</ul>

<h2>深入理解</h2>
<h3>BST删除的"魔术操作"</h3>
<p>删除BST中一个有两个孩子的节点时，为什么用<strong>右子树的最小值</strong>（中序后继）来替换？</p>
<p>想一下：我们要找一个值来替换被删除的节点，这个值必须满足：</p>
<ol>
<li>比左子树所有值都大</li>
<li>比右子树所有值都小</li>
</ol>
<p>右子树的最小值正好满足这两个条件！它就在右子树最左边的那个节点。</p>
<p>同理，用<strong>左子树的最大值</strong>（中序前驱）也可以，效果等价。选哪个都行，一般习惯用后继。</p>
<h3>树的退化问题</h3>
<p>BST有个致命缺陷：如果插入的数据是<strong>有序的</strong>，BST会退化成一个链表！</p>
<pre><code>正常BST：          退化BST（有序插入）：
      5                  1
    /   \                  \
   3     7                  2
  / \   / \                  \
 1   4 6   8                  3
                                 \
                                  4
                                   \
                                    5
</code></pre>
<p>左边高度为2，查找最多比较2次。右边高度为5，查找要比较5次。<strong>本来O(log n)的操作变成了O(n)</strong>。</p>
<p>这就是为什么工业界使用<strong>平衡二叉搜索树</strong>（AVL树、红黑树、B树等）来保证树的高度是O(log n)。下节课我们可能会聊这个话题。</p>

<h2>实际应用</h2>
<h3>1. 文件系统</h3>
<p>你的电脑文件系统就是一棵树：</p>
<pre><code>/
├── Users/
│   └── slouch/
│       ├── Documents/
│       │   └── learn/
│       └── Downloads/
├── Applications/
├── System/
└── tmp/
</code></pre>
<ul>
<li>根目录 <code>/</code> 是树的根</li>
<li>每个文件夹是一个节点</li>
<li>文件是叶子节点</li>
<li>路径 <code>Users/slouch/Documents</code> 就是从根到一个节点的路径</li>
</ul>
<p><code>cd</code> 就是往下走一层，<code>ls</code> 就是看当前节点的所有子节点。</p>
<h3>2. HTML DOM树</h3>
<p>每个网页都是一个DOM树：</p>
<pre><code>&lt;html&gt;
├── &lt;head&gt;
│   ├── &lt;title&gt;
│   └── &lt;meta&gt;
└── &lt;body&gt;
    ├── &lt;h1&gt;
    ├── &lt;p&gt;
    └── &lt;div&gt;
        ├── &lt;span&gt;
        └── &lt;a&gt;
</code></pre>
<p>浏览器的渲染引擎从DOM树的根开始遍历，逐层渲染页面。CSS选择器本质上就是在树中搜索节点。</p>
<h3>3. 数据库索引（B树/B+树）</h3>
<p>数据库索引的核心是<strong>B+树</strong>——BST的进化版。B+树是一种多路搜索树，每个节点可以有多个孩子，而且所有叶子在同一层。</p>
<p><strong>为什么不用BST？</strong> 因为磁盘IO是按"块"读取的。B+树的一个节点大小和磁盘块对齐，一次磁盘IO可以读取整个节点，效率更高。</p>
<p>MySQL、PostgreSQL等关系型数据库的索引底层都是B+树。这也是为什么你的SQL查询加了索引后，从O(n)变成了O(log n)。</p>
<h3>4. 决策树</h3>
<p>机器学习中的决策树，每个内部节点是一个判断条件，每个叶子节点是一个结论：</p>
<pre><code>          今天下雨？
         /         \
      是            否
     /               \
  带伞             有太阳？
                  /       \
                是         否
               /           \
           涂防晒霜      正常出门
</code></pre>
<hr />
<h2>常见误区</h2>
<p>⚠️ <strong>注意这些坑！</strong></p>
<h3>误区1：以为BST总是O(log n)</h3>
<p><strong>错！</strong> BST只有在<strong>平衡</strong>的情况下才是O(log n)。有序插入会让BST退化成链表，所有操作变成O(n)。</p>
<p>✅ <strong>正确做法</strong>：在需要保证性能的场景，使用平衡BST（AVL树、红黑树、Treap等）。Python的<code>dict</code>底层是哈希表而非BST，但Java的<code>TreeMap</code>底层是红黑树。</p>
<h3>误区2：搞混深度和高度</h3>
<p>这是新手最容易晕的地方：</p>
<ul>
<li><strong>深度</strong>：从<strong>根</strong>往下数（根的深度=0）</li>
<li><strong>高度</strong>：从<strong>叶子</strong>往上数（叶子的高度=0）</li>
</ul>
<p>口诀：<strong>深度从上往下跳，高度从下往上爬</strong>。一个节点的深度 = 它到根的边数，高度 = 它到最远叶子的边数。</p>
<h3>误区3：前序/中序/后序记混</h3>
<p>别死记硬背！记住一个核心规则：<strong>前中后指的是"根"什么时候被访问</strong>。</p>
<ul>
<li><strong>前</strong>序：根<strong>前</strong>面访问（根-左-右）</li>
<li><strong>中</strong>序：根<strong>中</strong>间访问（左-根-右）</li>
<li><strong>后</strong>序：根<strong>后</strong>面访问（左-右-根）</li>
</ul>
<p>左右永远是左在前、右在后。变化的只是根的位置。</p>

<h2>动手实践</h2>
<p>现在轮到你动手了！</p>
<h3>练习1：求二叉树的最大深度 ⭐</h3>
<p><strong>描述</strong>：给定一棵二叉树，求它的最大深度（根到最远叶子节点的最长路径上的节点数）。</p>
<p><strong>要求</strong>：</p>
<ul>
<li>实现 <code>max_depth(root)</code> 函数</li>
<li>使用递归方法</li>
</ul>
<p><strong>提示</strong>：一棵树的最大深度 = max(左子树最大深度, 右子树最大深度) + 1</p>

💡 提示（点击展开）<p>递归终止条件：如果节点为空，深度为0。<br />否则返回 <code>1 + max(max_depth(root.left), max_depth(root.right))</code></p>
<h3>练习2：验证二叉搜索树 ⭐⭐</h3>
<p><strong>描述</strong>：给定一棵二叉树，判断它是否是合法的BST。</p>
<p><strong>要求</strong>：</p>
<ul>
<li>实现 <code>is_valid_bst(root)</code> 函数</li>
<li>合法BST：左子树所有节点值 &lt; 根节点值 &lt; 右子树所有节点值</li>
</ul>
<p><strong>提示</strong>：中序遍历BST会得到有序序列。如果中序遍历结果不是递增的，就不是合法BST。</p>

💡 提示（点击展开）<p>方法1：中序遍历后检查结果是否有序。<br />方法2：递归时维护上下界，每个节点的值必须在(min, max)范围内。</p>
<h3>练习3：二叉树的最近公共祖先 ⭐⭐⭐</h3>
<p><strong>描述</strong>：给定一棵二叉树和两个节点p、q，找到它们的最近公共祖先（LCA）。</p>
<p><strong>要求</strong>：</p>
<ul>
<li>实现 <code>lowest_common_ancestor(root, p, q)</code> 函数</li>
<li>最近公共祖先：是p和q的祖先，且在所有公共祖先中深度最深</li>
</ul>
<p><strong>提示</strong>：从根开始递归。如果p和q分别在当前节点的左右两侧，当前节点就是LCA。</p>

💡 提示（点击展开）<p>递归思路：</p>
<ol>
<li>如果当前节点是p或q，返回当前节点</li>
<li>递归查找左子树和右子树</li>
<li>如果左右子树都找到了节点 → 当前节点是LCA</li>
<li>如果只有左子树找到 → LCA在左子树</li>
<li>如果只有右子树找到 → LCA在右子树</li>
</ol>

<h2>小结与思考</h2>
<h3>本节要点</h3>
<ol>
<li><strong>树是分层结构</strong>：根在最上面，叶子在最下面，每个节点可以有多个子节点</li>
<li><strong>二叉树每个节点最多两个孩子</strong>：左孩子和右孩子，有序号之分</li>
<li><strong>四种遍历方式</strong>：前序（根左右）、中序（左根右）、后序（左右根）、层序（逐层）</li>
<li><strong>BST的核心性质</strong>：左 &lt; 根 &lt; 右，中序遍历天然有序</li>
<li><strong>BST的查找/插入/删除</strong>：平均O(log n)，最坏O(n)（退化时）</li>
<li><strong>树的应用无处不在</strong>：文件系统、DOM树、数据库索引、决策树</li>
</ol>
<h3>思考题</h3>
<ol>
<li><p><strong>如果让你用数组来实现一棵完全二叉树，怎么设计？</strong> 提示：根在索引0，节点i的左孩子在2i+1，右孩子在2i+2。这是堆的实现方式，想想为什么可以这样做？</p>
</li>
<li><p><strong>BST的删除操作中，为什么可以用左子树的最大值代替右子树的最小值？</strong> 这两种方式有什么优劣？在什么场景下应该选哪种？</p>
</li>
<li><p><strong>如果我们要频繁地查找"第k大的元素"，BST够好吗？</strong> 可能需要对BST做怎样的改造？</p>
</li>
</ol>
