# 挑战 1：生产者-消费者问题实现

## 背景
生产者-消费者问题是经典的并发编程问题。生产者生成数据并放入缓冲区，消费者从缓冲区取出数据进行处理。当缓冲区满时，生产者必须等待；当缓冲区空时，消费者必须等待。

## 任务要求
使用Java多线程技术实现一个线程安全的生产者-消费者模型，具体要求如下：

### 基本要求
1. 创建一个固定大小的缓冲区（建议大小为5）
2. 实现生产者线程，不断向缓冲区添加随机整数（1-100）
3. 实现消费者线程，不断从缓冲区取出数据并打印
4. 使用适当的同步机制确保线程安全
5. 程序运行10秒后自动停止所有线程

### 进阶要求（选做）
1. 使用`BlockingQueue`接口的实现类（如`ArrayBlockingQueue`）重写解决方案
2. 添加统计功能：记录生产了多少个数据，消费了多少个数据
3. 实现多个生产者和多个消费者（例如2个生产者，3个消费者）

### 提示
- 可以使用`synchronized`关键字配合`wait()`和`notifyAll()`方法
- 也可以使用`java.util.concurrent`包中的高级并发工具
- 注意正确处理线程中断和资源清理

### 验证标准
- 程序不会出现`ArrayIndexOutOfBoundsException`或其他异常
- 缓冲区不会溢出（生产者在缓冲区满时正确等待）
- 不会出现空指针异常（消费者在缓冲区空时正确等待）
- 程序能够正常启动和停止
