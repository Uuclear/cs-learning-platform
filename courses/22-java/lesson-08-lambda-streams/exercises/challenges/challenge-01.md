# 挑战1：电商订单分析系统

## 背景
你是一家电商平台的数据分析师，需要处理大量的订单数据。你的任务是使用 Lambda 表达式和 Stream API 来分析订单数据，提取有价值的业务洞察。

## 数据模型
```java
class Order {
    String customerId;      // 客户ID
    String productName;     // 产品名称
    String category;        // 产品类别
    double amount;          // 订单金额
    int quantity;           // 数量
    LocalDateTime orderDate; // 订单日期
    String status;          // 订单状态 ("completed", "pending", "cancelled")
}
```

## 任务要求

### 基础任务（必须完成）
1. **客户价值分析**：找出消费总额最高的前10名客户，并计算他们的平均订单金额
2. **产品类别分析**：按产品类别统计总销售额、订单数量和平均单价
3. **时间趋势分析**：按月份统计已完成订单的销售总额（假设数据包含多个年份）

### 进阶任务（选做）
4. **客户行为分析**：识别"高价值客户"（定义：订单数量≥5且平均订单金额≥100），并分析他们的购买偏好
5. **异常检测**：找出订单金额异常高的订单（超过该类别平均订单金额3倍的订单）
6. **复购率分析**：计算客户的复购率（有2次以上购买的客户占总客户数的比例）

## 约束条件
- 必须使用 Stream API 和 Lambda 表达式，不能使用传统的 for 循环
- 代码应该具有良好的可读性和性能
- 考虑数据量可能很大，需要合理使用并行流
- 所有操作应该是线程安全的

## 提示
- 使用 `Collectors.groupingBy()` 进行分组统计
- 使用 `Collectors.summingDouble()`, `Collectors.averagingDouble()`, `Collectors.counting()` 等收集器
- 考虑使用 `LocalDateTime` 的方法进行日期处理
- 对于时间趋势分析，可以使用 `orderDate.getMonth()` 或自定义格式化

## 验证标准
- 代码能够正确处理边界情况（空数据、null值等）
- 性能良好，能够处理大量数据
- 代码结构清晰，易于理解和维护
- 充分利用了 Stream API 的特性
