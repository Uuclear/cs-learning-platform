# 挑战1：实现一个简单的日志轮转系统

## 背景
在生产环境中，日志文件会不断增长，需要定期轮转（rotation）以避免单个文件过大。

## 要求
1. 创建一个`LogRotator`类，包含以下功能：
   - 监控指定的日志文件大小
   - 当文件大小超过指定阈值（例如1MB）时，自动进行轮转
   - 轮转规则：将当前日志文件重命名为`logfile.1`，如果已存在则依次递增（`logfile.2`, `logfile.3`...）
   - 最多保留5个历史日志文件，超出的自动删除
   - 创建新的空日志文件继续写入

2. 使用现代Java NIO API实现，包括：
   - `Files.size()`检查文件大小
   - `Files.move()`进行文件重命名
   - `Files.deleteIfExists()`删除旧文件
   - `Files.createFile()`创建新文件

3. 编写测试代码验证功能：
   - 创建一个大日志文件（超过1MB）
   - 调用轮转方法
   - 验证文件是否正确重命名和创建

## 提示
- 考虑线程安全性（如果多个线程同时写日志）
- 处理文件操作异常
- 使用try-with-resources确保资源正确关闭
