# 挑战2：GC调优实战

## 背景
你负责优化一个高并发的消息处理系统，该系统需要处理每秒数千条消息。目前使用默认的JVM参数，但在压力测试中发现GC停顿时间过长，影响了系统的实时性。

## 系统特点
- 高吞吐量：每秒处理5000+消息
- 低延迟要求：99%的消息处理延迟应小于50ms
- 内存占用：堆内存约4GB
- 对象特征：大量短生命周期的对象（消息对象）

## 当前JVM参数
```bash
-Xms4g -Xmx4g -XX:+UseParallelGC
```

## 任务要求
1. **性能分析**：使用jstat或其他工具收集当前GC性能数据
2. **参数调优**：选择合适的GC算法并调整相关参数
3. **基准测试**：使用JMH或其他基准测试工具验证调优效果
4. **结果对比**：提供调优前后的性能对比报告

## 具体步骤
1. 编写一个模拟消息处理的基准测试程序
2. 使用当前参数运行测试，记录GC频率、停顿时间和吞吐量
3. 尝试不同的GC算法（G1 GC、ZGC等）和参数组合
4. 找到最优配置并验证其在不同负载下的稳定性

## 提交内容
- 基准测试代码
- GC日志分析报告
- 最优JVM参数配置
- 性能对比图表和结论

## 评估标准
- GC停顿时间是否满足要求（<50ms）
- 吞吐量是否有所提升
- 内存使用是否合理
- 配置的稳定性和可维护性
