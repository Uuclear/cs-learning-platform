# 挑战任务 2：实现用户认证和授权

## 任务描述
在现有的用户管理 API 基础上，添加基于 JWT（JSON Web Token）的认证和授权功能。

### 功能要求
1. **认证端点**：
   - 创建 `AuthController` 控制器
   - 实现 `POST /api/auth/login` 端点，接收用户名和密码
   - 验证用户凭据并返回 JWT token

2. **JWT 工具类**：
   - 创建 `JwtUtil` 类，包含生成和验证 JWT token 的方法
   - token 应包含用户 ID、用户名和过期时间
   - 使用安全的密钥进行签名

3. **安全配置**：
   - 创建 Spring Security 配置类
   - 配置哪些端点需要认证，哪些可以公开访问
   - 实现 JWT 过滤器，从请求头中提取并验证 token

4. **授权增强**：
   - 修改现有用户 API，只有认证用户才能访问
   - 实现用户只能修改自己的信息（基于用户 ID 的权限检查）
   - 添加管理员角色，管理员可以管理所有用户

5. **错误处理**：
   - 处理认证失败的情况（无效凭据、token 过期等）
   - 返回适当的错误响应和 HTTP 状态码

### 技术要求
- 使用 Spring Security 和 JWT 库
- 密钥应从配置文件中读取，不要硬编码
- 密码必须加密存储（使用 BCrypt）
- 实现无状态认证（每个请求都包含 token）

### 提示
- 可以使用 `spring-boot-starter-security` 和 `jjwt` 依赖
- 考虑创建自定义异常处理器
- 记住 REST API 应该是无状态的

### 验收标准
- 用户可以成功登录并获得有效的 JWT token
- 受保护的端点需要有效的 token 才能访问
- 无效或过期的 token 会被正确拒绝
- 密码安全存储，不会以明文形式保存
- 代码结构清晰，安全逻辑与业务逻辑分离
