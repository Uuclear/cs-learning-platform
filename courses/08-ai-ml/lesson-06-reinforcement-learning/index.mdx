# 强化学习：从AlphaGo到自动驾驶

2016年，AlphaGo 击败李世石，全世界才意识到：机器不靠「背答案」，也能在复杂博弈里自己练出神级策略。强化学习（Reinforcement Learning, RL）干的就是这件事——**在试错里学决策**。

> 💡 **思考问题**：教小孩骑车，你会先讲 50 页力学讲义，还是让他摔几次自己找平衡？RL 更像后者。

---

## 开场白

监督学习像「有标准答案的模考」：给你题和分数。强化学习像「没有标准操作手册的实战」：你动手，环境给反馈（奖励或惩罚），你自己总结什么动作更好。

从 AlphaGo、Dota 智能体，到推荐系统、机器人、自动驾驶，RL 的核心框架始终不变：

- **智能体（Agent）**：做决策的一方  
- **环境（Environment）**：智能体交互的外部世界  
- **状态（State）**、**动作（Action）**、**奖励（Reward）**：描述「在哪、做了什么、好不好」

本课从 MDP 和 Q-learning 入门，再聊探索与利用、DQN 与策略梯度，并用可运行代码把抽象公式落地。

## 核心概念

### 马尔可夫决策过程（MDP）

MDP 是 RL 的数学骨架，包含：

| 要素 | 含义 |
|------|------|
| 状态空间 S | 智能体可能处于的所有状态 |
| 动作空间 A | 可执行的动作集合 |
| 转移 P(s'|s,a) | 执行 a 后进入 s' 的概率 |
| 奖励 R | 即时反馈 |
| 折扣因子 γ | 未来奖励的权重（0~1） |

**马尔可夫性**：未来只依赖当前状态，与更早历史无关（简化后才好算）。

### 策略与价值函数

- **策略 π(a|s)**：在状态 s 选动作 a 的规则  
- **状态价值 V^π(s)**：从 s 出发、按 π 行动的期望总回报  
- **动作价值 Q^π(s,a)**：在 s 执行 a 后的期望总回报  

目标：找到 **最优策略 π\***，使长期累积奖励最大。

### Q-learning（无模型）

不建环境模型，直接迭代 Q 表：

`Q(s,a) ← Q(s,a) + α [ r + γ max Q(s',a') − Q(s,a) ]`

其中 α 为学习率，r 为即时奖励，s' 为下一状态。

## 深入理解

### 探索 vs 利用

- **利用**：选当前看起来最好的动作  
- **探索**：试新动作，可能发现更优策略  

ε-greedy 以概率 ε 随机探索；UCB 根据不确定性主动探索「了解不够」的臂。

### Deep Q-Network（DQN）

用神经网络近似 Q 函数，配合 **经验回放** 与 **目标网络** 稳定训练，让 RL 能处理高维状态（如游戏画面）。

### 策略梯度与 Actor-Critic

- **REINFORCE**：直接优化策略参数  
- **Actor-Critic**：Actor 选动作，Critic 评估好坏，方差更小、收敛更稳

## 实际应用

1. **游戏 AI**：围棋、星际、MOBA 中的序贯决策  
2. **机器人控制**：行走、抓取、导航（仿真中训练再迁移）  
3. **推荐与广告**：把「展示哪条内容」建模为序贯决策，优化长期点击/留存

## 常见误区

### 误区1：「RL 不需要大量试错」

多数算法仍需成千上万次交互。真实场景常用 **仿真环境** 或 **离线 RL** 降低代价。

### 误区2：「算法比奖励设计重要」

糟糕的奖励会导致 **reward hacking**（例如赛车 AI 原地转圈刷速度分）。奖励设计往往决定学到的是不是你真正想要的行为。

### 误区3：「训练集上的高分等于部署可用」

分布偏移、安全约束、延迟反馈都会让实验室策略在生产环境失效，需要约束 RL、模仿学习或人在回路。
