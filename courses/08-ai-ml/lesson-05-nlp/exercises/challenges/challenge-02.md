# 构建简单的情感分析器

## 难度：⭐⭐⭐⭐

## 任务描述

在这个挑战中，你需要构建一个端到端的情感分析系统。你的系统应该能够：

1. 对输入文本进行预处理（分词、停用词移除等）
2. 提取文本特征（使用TF-IDF或其他特征表示方法）
3. 训练一个分类模型来预测情感极性（正面/负面）
4. 对新文本进行情感预测并给出置信度

## 具体要求

### 数据准备

由于我们不能使用外部数据集，你需要：

- 创建一个小型的训练数据集（至少10个正面样本，10个负面样本）
- 确保数据覆盖不同的表达方式和词汇

### 模型实现

你必须从零开始实现一个机器学习模型，不能使用scikit-learn等库。可以选择：

- **逻辑回归**（推荐）：实现梯度下降优化
- **朴素贝叶斯**：基于概率的分类方法
- **支持向量机**：实现简化版的SVM

## 测试用例

使用以下测试文本验证你的系统：

```python
test_texts = [
    "这个产品非常棒，我特别喜欢！",
    "质量太差了，完全不值得购买。",
    "一般般，没有什么特别的感觉。",
    "服务态度很好，解决问题很及时。"

```

## 评估标准

- **准确性**：在测试文本上的情感判断是否合理
- **鲁棒性**：能否处理不同的表达方式和词汇
- **代码质量**：代码结构清晰，注释完整，符合PEP 8规范
- **创新性**：是否有额外的功能或优化（如处理否定词、程度副词等）

## 提示

- 注意中文文本的特殊性（没有明显的词边界）
- 考虑情感词汇的强度（如"非常好" vs "好"）
- 处理否定表达（如"不是很好"应该是负面的）
- 可以使用简单的词典方法作为baseline

## 扩展挑战（可选）

- 添加情感强度预测（1-5星评分）
- 支持不同领域的适应（如电影评论、产品评论等）
- 实现在线学习能力（能够从用户反馈中学习）
