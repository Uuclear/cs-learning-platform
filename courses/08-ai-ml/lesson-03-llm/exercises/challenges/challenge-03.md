# 设计Prompt Engineering模板

**难度**: ⭐⭐⭐

## 描述

在这个挑战中，你需要为不同的应用场景设计有效的提示工程（Prompt Engineering）模板。提示工程是与大语言模型有效交互的关键技能，好的提示模板能够显著提升模型输出的质量、准确性和一致性。

你需要针对至少三种不同的应用场景设计专门的提示模板，并实现一个模板管理系统来动态生成和使用这些模板。

## 要求

### 基本要求
1. 实现一个`PromptTemplateManager`类，包含以下功能：
   - `add_template(self, name, template, description)`: 添加新的提示模板
   - `get_template(self, name)`: 获取指定名称的模板
   - `list_templates(self)`: 列出所有可用模板
   - `render(self, name, **kwargs)`: 渲染模板并填充变量

2. 设计至少三个不同场景的提示模板：
   - **问答系统模板**: 用于知识问答任务
   - **文本摘要模板**: 用于生成文本摘要
   - **创意写作模板**: 用于故事、诗歌等创作任务

3. 每个模板必须包含：
   - 清晰的任务描述
   - 明确的输出格式要求
   - 必要的约束条件
   - 支持变量替换

### 进阶要求（可选）
1. 实现模板版本控制和A/B测试支持
2. 添加模板性能评估功能（基于用户反馈）
3. 实现动态模板优化（基于使用效果自动调整）
4. 支持多语言模板

## 提示

### 模板设计原则
- **明确性**: 清晰描述任务目标和期望输出
- **结构化**: 使用分隔符和标记组织提示内容
- **约束性**: 设置合理的边界条件和限制
- **示例性**: 在需要时提供输入-输出示例

```python
"""

2. 保留关键信息和主要观点
3. 使用简洁清晰的语言
4. 不要添加原文中没有的信息

摘要:
"""
```

### 模板管理器实现
```python
class PromptTemplateManager:
    def __init__(self):
        self.templates = {}
    
    def add_template(self, name, template, description=""):
        """添加模板"""
        self.templates[name] = {
            'template': template,
            'description': description
        }
    
    def render(self, name, **kwargs):
        """渲染模板"""
        if name not in self.templates:
            raise ValueError(f"模板 '{name}' 不存在")
        return self.templates[name]['template'].format(**kwargs)
```
