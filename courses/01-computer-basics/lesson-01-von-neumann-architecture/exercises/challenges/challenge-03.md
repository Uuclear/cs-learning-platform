# 挑战3：模拟一个完整的冯诺依曼计算机

## 难度
⭐⭐⭐⭐

## 描述

你已经学习了冯诺依曼架构的各个组成部分。现在，请用Python实现一个完整的、可交互的冯诺依曼计算机模拟器，包含CPU、内存、输入/输出设备。

## 要求

### 1. 实现完整的CPU

包含以下组件：
- **寄存器**：
  - PC（程序计数器）
  - ACC（累加器）
  - IR（指令寄存器）
  - 通用寄存器 R0, R1, R2, R3

- **指令集**（至少16条指令）：
  - 数据传输：LOAD, STORE, MOVE
  - 算术运算：ADD, SUB, MUL, DIV, INC, DEC
  - 逻辑运算：AND, OR, NOT, XOR
  - 控制流：JUMP, JZ（为零跳转）, JN（为负跳转）, HALT
  - 输入输出：IN, OUT

### 2. 实现内存系统

- 内存大小：256字节
- 支持读写操作
- 实现内存视图显示（类似调试器）

### 3. 实现输入输出设备

- **输入**：从键盘读取数字
- **输出**：在屏幕显示结果
- 使用特殊内存地址或专用指令实现

### 4. 实现汇编器

- 将人类可读的汇编代码转换为机器码
- 支持标签（用于跳转）
- 支持注释

示例汇编代码：
```assembly
; 计算 1+2+3+...+N
; N 由用户输入

        IN          ; 读取N到ACC
        STORE N     ; 保存N
        LOAD ZERO   ; ACC = 0
        STORE SUM   ; SUM = 0

LOOP:   LOAD N      ; 加载N
        JZ DONE     ; 如果N=0，结束
        ADD SUM     ; ACC = N + SUM
        STORE SUM   ; 保存新的和
        LOAD N      ; 加载N
        DEC         ; N = N - 1
        STORE N     ; 保存N
        JUMP LOOP   ; 继续循环

DONE:   LOAD SUM    ; 加载结果
        OUT         ; 输出结果
        HALT        ; 停止

N:      0           ; 变量N
SUM:    0           ; 变量SUM
ZERO:   0           ; 常量0
```

### 5. 实现交互式调试器

功能包括：
- 单步执行（Step）
- 连续执行（Run）
- 设置断点
- 查看寄存器状态
- 查看内存内容
- 修改内存/寄存器

## 进阶功能

1. **图形界面**：使用tkinter或pygame实现可视化
2. **性能分析**：统计指令执行次数、周期数
3. **扩展内存**：实现分页内存管理
4. **中断系统**：模拟硬件中断
5. **多程序**：实现简单的进程切换

## 思考

1. 你的模拟器和真实CPU有什么主要区别？
2. 如果要提高模拟速度，你会怎么做？
3. 现代CPU有哪些你的模拟器没有的特性？（流水线、缓存、分支预测等）
4. 冯诺依曼架构的局限性是什么？（提示：冯诺依曼瓶颈）
