# 闭包与作用域链

## 开场白：JavaScript中最神秘的概念

同学们好！今天我们来探讨JavaScript中最有名也最让人困惑的概念——**闭包**（Closure）和**作用域链**（Scope Chain）。如果你觉得这些概念难以理解，别担心，你不是一个人！就连经验丰富的开发者也会在这些概念上犯迷糊。

想象一下，闭包就像一个背着小书包的旅行者。这个旅行者（函数）从家乡（定义的地方）出发，无论走到哪里（执行的地方），都始终带着家乡的记忆（外部作用域的变量）。这就是闭包的魔力——它让函数能够"记住"并访问其创建时所在环境的变量。

在本课程中，我们将一步步揭开闭包和作用域链的神秘面纱，通过生活化的例子、清晰的代码演示和实际应用场景，让你真正掌握这些核心概念。

## 核心概念：词法作用域与作用域链

### 什么是词法作用域？

JavaScript采用**词法作用域**（Lexical Scoping），也称为**静态作用域**。这意味着函数的作用域在**代码编写时**就已经确定了，而不是在运行时动态决定的。

```javascript
const name = "全局变量";

function outer() {
  const name = "外层函数变量";
  
  function inner() {
    const name = "内层函数变量";
    console.log(name); // 输出什么？
  }
  
  return inner;
}

const myFunc = outer();
myFunc(); // 输出: "内层函数变量"
```

在这个例子中，`inner`函数在定义时就确定了它的作用域。即使我们在`outer`函数外部调用`myFunc()`，`inner`函数仍然按照词法作用域规则查找变量。

### 作用域链的工作原理

当JavaScript引擎需要查找一个变量时，它会沿着**作用域链**（Scope Chain）向上查找：

1. **首先**在当前函数的作用域中查找
2. **然后**在外层函数的作用域中查找
3. **最后**在全局作用域中查找
4. **如果都找不到**，则抛出ReferenceError

作用域链就像一条链条，每个环节都是一个作用域对象。这条链条在函数创建时就固定下来了。

### 变量提升（Hoisting）

在理解作用域之前，我们还需要了解**变量提升**的概念。JavaScript会将变量和函数声明"提升"到其作用域的顶部：

```javascript
console.log(x); // undefined (不是ReferenceError!)
var x = 5;

// 上面的代码实际上等价于：
var x; // 声明被提升
console.log(x); // undefined
x = 5; // 赋值留在原地
```

需要注意的是：
- `var`声明会被完全提升（声明和初始化）
- `let`和`const`声明也会被提升，但在声明前访问会抛出ReferenceError（暂时性死区）
- 函数声明会被完全提升，但函数表达式不会

## 代码演示：闭包的创建与工作原理

### 示例1：基础闭包

让我们通过具体的代码来理解闭包是如何工作的：

```javascript
function createCounter() {
  let count = 0; // 私有变量
  
  return function() {
    count++; // 闭包：内部函数访问外部函数的变量
    console.log(`计数器当前值: ${count}`);
    return count;
  };
}

const counter1 = createCounter();
const counter2 = createCounter();

counter1(); // 计数器当前值: 1
counter1(); // 计数器当前值: 2
counter2(); // 计数器当前值: 1 (独立的闭包)
```

这里的关键点是：
- `count`变量被"封闭"在返回的匿名函数中
- 每次调用`createCounter()`都会创建一个新的闭包环境
- 不同的闭包实例之间互不影响

### 示例2：闭包与循环

闭包在循环中经常会产生意外的结果，让我们看看如何正确处理：

```javascript
// 错误的方式 - 所有回调都引用同一个i
for (var i = 0; i < 3; i++) {
  setTimeout(() => {
    console.log(`错误: ${i}`); // 输出: 错误: 3, 错误: 3, 错误: 3
  }, 100);
}

// 正确的方式1 - 使用let (块级作用域)
for (let i = 0; i < 3; i++) {
  setTimeout(() => {
    console.log(`正确(let): ${i}`); // 输出: 正确(let): 0, 1, 2
  }, 100);
}

// 正确的方式2 - 使用立即执行函数(IIFE)
for (var i = 0; i < 3; i++) {
  (function(index) {
    setTimeout(() => {
      console.log(`正确(IIFE): ${index}`); // 输出: 正确(IIFE): 0, 1, 2
    }, 100);
  })(i);
}
```

## 深入理解：闭包的内存管理

### 闭包如何影响内存？

闭包会让外部作用域的变量一直保持在内存中，即使外部函数已经执行完毕。这是因为内部函数仍然持有对这些变量的引用。

```javascript
function heavyComputation() {
  const largeArray = new Array(1000000).fill('data'); // 大数组
  const importantData = "关键数据";
  
  return function() {
    // 只使用importantData，但largeArray也被保留
    return importantData;
  };
}

const result = heavyComputation();
// largeArray仍然在内存中，因为闭包可能需要它
```

### 避免内存泄漏

为了避免不必要的内存占用，我们应该：

1. **只引用需要的变量**
2. **及时断开不需要的引用**
3. **使用弱引用（WeakMap/WeakSet）**

```javascript
// 更好的做法：只捕获需要的数据
function betterHeavyComputation() {
  const largeArray = new Array(1000000).fill('data');
  const importantData = "关键数据";
  
  // 提前提取需要的数据
  const neededData = importantData;
  
  return function() {
    return neededData; // largeArray可以被垃圾回收
  };
}
```

## 实际应用：闭包的经典使用场景

### 1. 模块模式（Module Pattern）

闭包是实现模块模式的基础，可以创建私有变量和方法：

```javascript
const Calculator = (function() {
  let currentValue = 0; // 私有变量
  
  return {
    add: function(value) {
      currentValue += value;
      return this;
    },
    subtract: function(value) {
      currentValue -= value;
      return this;
    },
    getValue: function() {
      return currentValue;
    },
    reset: function() {
      currentValue = 0;
      return this;
    }
  };
})();

Calculator.add(10).subtract(3).getValue(); // 7
```

### 2. 工厂函数（Factory Functions）

闭包可以用来创建配置化的函数：

```javascript
function createValidator(rules) {
  return function(value) {
    // rules被闭包捕获
    for (const rule of rules) {
      if (!rule.test(value)) {
        return false;
      }
    }
    return true;
  };
}

const emailValidator = createValidator([
  { test: (v) => v.includes('@'), message: "必须包含@" },
  { test: (v) => v.length > 5, message: "长度必须大于5" }
]);

console.log(emailValidator("test@example.com")); // true
console.log(emailValidator("invalid")); // false
```

### 3. 事件处理器和回调

闭包在事件处理中非常有用，可以保存上下文信息：

```javascript
function createButtonHandler(buttonId, message) {
  return function() {
    console.log(`按钮 ${buttonId} 被点击: ${message}`);
  };
}

// 为多个按钮创建不同的处理器
document.getElementById('btn1').onclick = createButtonHandler('btn1', '欢迎!');
document.getElementById('btn2').onclick = createButtonHandler('btn2', '再见!');
```

## 常见误区与陷阱

### 误区1：闭包很慢

很多人认为闭包会影响性能。实际上，现代JavaScript引擎对闭包进行了大量优化。闭包的性能开销通常可以忽略不计，除非在极端情况下。

### 误区2：所有嵌套函数都是闭包

严格来说，只有当内部函数**访问了外部函数的变量**时，才形成闭包。如果内部函数没有引用外部变量，那么它只是一个普通的嵌套函数。

```javascript
function outer() {
  const x = 1;
  
  function inner1() {
    console.log("我没有访问外部变量"); // 不是闭包
  }
  
  function inner2() {
    console.log(x); // 访问了外部变量x，这是闭包
  }
  
  return [inner1, inner2];
}
```

### 误区3：闭包总是导致内存泄漏

闭包本身不会导致内存泄漏。内存泄漏通常是因为：
- 忘记移除事件监听器
- 意外的全局变量引用
- DOM节点的循环引用

合理的闭包使用是安全的。
