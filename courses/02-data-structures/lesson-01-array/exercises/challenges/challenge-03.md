# 挑战3：合并两个有序数组

## 难度
⭐⭐⭐

## 描述

给你两个按**非递减顺序**（即升序，允许相等）排列的整数数组 `nums1` 和 `nums2`，另有两个整数 `m` 和 `n`，分别表示 `nums1` 和 `nums2` 的元素数目。

请你**合并** `nums2` 到 `nums1` 中，使合并后的数组同样按**非递减顺序**排列。

**注意**：`nums1` 的长度等于 `m + n`，其中后 `n` 个元素设为 0（这是为了容纳 `nums2` 的元素）。你需要**原地**修改 `nums1`，而不是返回新数组。

## 输入

- `nums1`: 整数数组，长度 = m + n，前 m 个是有效数据，后 n 个是0
- `m`: nums1 的有效元素个数
- `nums2`: 整数数组，长度 = n
- `n`: nums2 的元素个数

## 输出

原地修改 `nums1`，使其包含合并后的有序数组。

## 示例

**示例 1:**
```
输入: nums1 = [1, 2, 3, 0, 0, 0], m = 3, nums2 = [2, 5, 6], n = 3
输出: [1, 2, 2, 3, 5, 6]
解释: 合并 [1, 2, 3] 和 [2, 5, 6] 得到 [1, 2, 2, 3, 5, 6]
```

**示例 2:**
```
输入: nums1 = [1], m = 1, nums2 = [], n = 0
输出: [1]
解释: nums2 为空，不需要合并
```

**示例 3:**
```
输入: nums1 = [0], m = 0, nums2 = [1], n = 1
输出: [1]
解释: nums1 为空，直接把 nums2 放入
```

## 约束条件

- nums1.length == m + n
- nums2.length == n
- 0 <= m, n <= 200
- 1 <= m + n <= 200
- -10^9 <= nums1[i], nums2[j] <= 10^9

## 提示

- 如果从前往后合并，需要移动元素，时间复杂度较高
- **关键思路**：从后往前合并！
- 设置三个指针：
  - `i`：指向 nums1 有效数据的末尾（m-1）
  - `j`：指向 nums2 的末尾（n-1）
  - `k`：指向 nums1 的末尾（m+n-1，填充位置）
- 比较 `nums1[i]` 和 `nums2[j]`，把大的放到 `nums1[k]`，然后相应指针前移

## 进阶思考

1. 为什么从后往前比从前往后更好？
2. 如果两个数组都是降序排列，算法需要如何修改？
3. 如果要求合并到新的数组（不原地修改），代码会更简单吗？
4. 如果 nums1 没有预留空间（长度就是m），该如何处理？

### 思路

**关键洞察：从后往前填充**

如果从前往后合并：
- 需要把 nums1 的元素后移，给 nums2 的元素腾位置
- 这样会导致频繁的元素移动，效率低

如果从后往前合并：
- nums1 后面本来就是空的（填充了0）
- 我们只需要把较大的元素放到后面即可
- 不需要移动任何元素！

**算法步骤**：
1. 初始化三个指针：`i = m-1`, `j = n-1`, `k = m+n-1`
2. 比较 `nums1[i]` 和 `nums2[j]`
3. 把较大的放到 `nums1[k]`，相应指针前移
4. 重复直到所有元素都放置完毕

### 代码

```python
def merge(nums1, m, nums2, n):
    """
    合并两个有序数组
    时间复杂度: O(m + n)
    空间复杂度: O(1)
    """
    # 从后往前的三个指针
    i = m - 1      # nums1 有效数据的末尾
    j = n - 1      # nums2 的末尾
    k = m + n - 1  # nums1 的末尾（填充位置）
    
    # 从后往前填充
    while i >= 0 and j >= 0:
        if nums1[i] > nums2[j]:
            nums1[k] = nums1[i]
            i -= 1
        else:
            nums1[k] = nums2[j]
            j -= 1
        k -= 1
    
    # 如果 nums2 还有剩余，复制到前面
    # （如果 nums1 有剩余，已经在正确位置，不需要处理）
    while j >= 0:
        nums1[k] = nums2[j]
        j -= 1
        k -= 1

### 复杂度分析

- **时间复杂度**: O(m + n)
  - 每个元素最多被访问和移动一次
- **空间复杂度**: O(1)
  - 只使用了三个指针变量
  - 原地修改，没有使用额外数组

### 为什么从后往前更好？

**从前往后的问题**：
```
nums1: [1, 2, 3, 0, 0, 0]
nums2: [2, 5, 6]

比较 1 和 2，1较小，应该放在位置0（已经在）
比较 2 和 2，相等，2应该放在位置1，但位置1已经有2了！
→ 需要把 [2,3] 后移，才能放入 nums2 的 2
```

这样会导致频繁的元素移动，最坏情况下是 O((m+n)²)。

**从后往后的优势**：
```
nums1: [1, 2, 3, 0, 0, 0]
              ↑     ↑
              i     k
nums2: [2, 5, 6]
              ↑
              j

比较 3 和 6，6较大，放到位置k（末尾）
不需要移动任何元素！
```

       6 > 3，nums1[5]=6

       5 > 3，nums1[4]=5

       3 > 2，nums1[3]=3

       2 >= 2，nums1[2]=2

nums2 已用完，nums1 剩余元素 [1,2] 已在正确位置

```
