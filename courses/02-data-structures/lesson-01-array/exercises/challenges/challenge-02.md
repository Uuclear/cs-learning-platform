# 挑战2：旋转数组

## 难度
⭐⭐

## 描述

给定一个数组，将数组中的元素向右轮转 `k` 个位置，其中 `k` 是非负数。

**向右轮转 1 位** 意味着：最后一个元素移到数组开头，其他元素依次后移一位。

例如：
- 原数组: `[1, 2, 3, 4, 5]`
- 向右轮转1位: `[5, 1, 2, 3, 4]`
- 向右轮转2位: `[4, 5, 1, 2, 3]`

## 输入

- `nums`: 整数数组，长度范围 [1, 10^5]
- `k`: 非负整数，轮转次数，范围 [0, 10^5]

## 输出

原地修改输入数组，不需要返回值。

## 示例

**示例 1:**
```
输入: nums = [1, 2, 3, 4, 5, 6, 7], k = 3
输出: [5, 6, 7, 1, 2, 3, 4]
解释:
向右轮转 1 步: [7, 1, 2, 3, 4, 5, 6]
向右轮转 2 步: [6, 7, 1, 2, 3, 4, 5]
向右轮转 3 步: [5, 6, 7, 1, 2, 3, 4]
```

**示例 2:**
```
输入: nums = [-1, -100, 3, 99], k = 2
输出: [3, 99, -1, -100]
解释:
向右轮转 1 步: [99, -1, -100, 3]
向右轮转 2 步: [3, 99, -1, -100]
```

## 约束条件

- 1 <= nums.length <= 10^5
- -2^31 <= nums[i] <= 2^31 - 1
- 0 <= k <= 10^5

## 提示

- 注意：k 可能大于数组长度！轮转 n 次等于回到原样（n是数组长度）
- 解法1：使用额外数组，简单但空间复杂度 O(n)
- 解法2：环状替换，空间复杂度 O(1)，但逻辑稍复杂
- 解法3：**三次翻转法**，空间 O(1) 且代码简洁（推荐学习）

2. 如果要求向左轮转，代码需要如何修改？
3. 三次翻转法的原理是什么？为什么有效？

### 思路

**解法1：使用额外数组（最容易理解）**
- 创建一个新数组
- 原数组索引 `i` 的元素，放到新数组的 `(i + k) % n` 位置
- 把新数组复制回原数组

**解法2：三次翻转法（最优雅）**
- 原理：把数组分成两部分，翻转三次实现轮转
- 步骤：
  1. 翻转整个数组
  2. 翻转前 k 个元素
  3. 翻转剩余元素

例如：`[1, 2, 3, 4, 5]`, k=2
1. 翻转全部: `[5, 4, 3, 2, 1]`
2. 翻转前2个: `[4, 5, 3, 2, 1]`
3. 翻转后3个: `[4, 5, 1, 2, 3]` ✅

### 代码

```python
# 解法1：使用额外数组
# 时间复杂度: O(n)
# 空间复杂度: O(n)
def rotate_extra_array(nums, k):
    n = len(nums)
    k = k % n  # 处理 k > n 的情况
    
    # 创建新数组
    temp = [0] * n
    
    # 放置元素到新位置
    for i in range(n):
        temp[(i + k) % n] = nums[i]
    
    # 复制回原数组
    for i in range(n):
        nums[i] = temp[i]


# 解法2：三次翻转法（推荐）
# 时间复杂度: O(n)
# 空间复杂度: O(1)
def rotate_reverse(nums, k):
    n = len(nums)
    k = k % n  # 处理 k > n 的情况
    
    def reverse(arr, start, end):
        """翻转数组的 [start, end] 部分"""
        while start < end:
            arr[start], arr[end] = arr[end], arr[start]
            start += 1
            end -= 1
    
    # 三次翻转
    reverse(nums, 0, n - 1)      # 1. 翻转全部
    reverse(nums, 0, k - 1)      # 2. 翻转前k个
    reverse(nums, k, n - 1)      # 3. 翻转剩余


# 测试
if __name__ == "__main__":
    # 测试用例
    test_cases = [
        ([1, 2, 3, 4, 5, 6, 7], 3, [5, 6, 7, 1, 2, 3, 4]),
        ([-1, -100, 3, 99], 2, [3, 99, -1, -100]),
        ([1, 2], 3, [2, 1]),  # k > n 的情况
    ]
    
    for nums, k, expected in test_cases:
        nums_copy = nums.copy()
        rotate_reverse(nums_copy, k)
        print(f"nums={nums}, k={k}")
        print(f"结果: {nums_copy}, 期望: {expected}")
        assert nums_copy == expected, "测试失败！"
        print("✅ 通过\n")
```

### 复杂度分析

| 解法 | 时间复杂度 | 空间复杂度 | 说明 |
|------|-----------|-----------|------|
| 额外数组 | O(n) | O(n) | 简单易懂，但用额外空间 |
| 三次翻转 | O(n) | O(1) | 空间最优，代码简洁 |

### 三次翻转法原理

为什么三次翻转有效？

假设数组分成两部分：A（前 n-k 个）和 B（后 k 个）
- 原数组: `[A][B]`
- 目标: `[B][A]`

翻转过程：
1. 翻转全部: `[A][B]` → `[B'][A']` （B'表示B的逆序，A'表示A的逆序）
2. 翻转B': `B'` → `B`
3. 翻转A': `A'` → `A`

最终结果: `[B][A]` ✅
