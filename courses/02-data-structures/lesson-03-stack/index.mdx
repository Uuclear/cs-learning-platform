## 核心概念

### 什么是栈？

**栈**是一种**受限**的线性数据结构。它的特点是：**只能在一端进行插入和删除操作**。

这一端叫做**栈顶**（top），另一端叫做**栈底**（bottom）。

栈遵循一个铁律：**LIFO —— Last In, First Out（后进先出）**。

### LIFO原则：后进先出

LIFO 就像一堆盘子：

```
      ┌───┐   ← 栈顶（最后放上去的）
      │ 3 │   ← pop() 先弹它
      ├───┤
      │ 2 │
      ├───┤
      │ 1 │   ← 最早放进来的
      └───┘   ← 栈底
```

- **push（入栈）**：把盘子放在最上面
- **pop（出栈）**：从最上面拿走一个
- **peek（查看栈顶）**：看一眼最上面是什么，但不拿走
- **isEmpty（判空）**：看看这摞盘子是不是空的

四个生活类比帮你理解：

| 场景 | push | pop | LIFO的体现 |
|------|------|-----|-----------|
| 盘子堆 | 放盘子到顶上 | 从顶上拿走 | 最后放的先被用 |
| 子弹夹 | 装入子弹 | 射出一颗子弹 | 最后装的先射出 |
| 电梯 | 最后进来的人站在最靠门的位置 | 先出去 | 后进先出 |
| 撤销操作 | 记录你最新的修改 | 撤销最新修改 | 最近做的先撤回 |

### 栈的四种基本操作

```
操作名    | 描述           | 返回值          | 时间复杂度
----------|----------------|-----------------|----------
push(x)   | 把x放入栈顶     | 无              | O(1)
pop()     | 移除并返回栈顶   | 栈顶元素        | O(1)
peek()    | 查看栈顶不移除   | 栈顶元素        | O(1)
isEmpty() | 判断栈是否为空   | true / false    | O(1)
```

注意：所有操作都是 **O(1)**！这是因为我们只操作栈顶，不需要遍历。

## 代码演示

让我们通过代码来理解这个概念：

### 示例1: 栈的基本实现（数组版 vs 链表版）

栈可以用两种方式实现：**数组（列表）** 或 **链表**。两种都能实现LIFO，但各有特点。

```python
# 代码见: code/examples/example-01-stack-basics.py

# 数组版核心逻辑：
class ArrayStack:
    def __init__(self):
        self.items = []
    def push(self, item):
        self.items.append(item)      # 放最后
    def pop(self):
        return self.items.pop()      # 拿最后
    def peek(self):
        return self.items[-1]        # 看最后
    def is_empty(self):
        return len(self.items) == 0
```

**运行结果**：
```
数组版栈演示
初始状态: 栈为空? True, 大小: 0

--- 入栈操作 ---
  push('盘子1') → 栈: ['盘子1']
  push('盘子2') → 栈: ['盘子1', '盘子2']
  push('盘子3') → 栈: ['盘子1', '盘子2', '盘子3']

--- 查看栈顶 ---
  peek() → '盘子3'

--- 出栈操作 ---
  pop() → 弹出 '盘子3', 栈: ['盘子1', '盘子2']
  pop() → 弹出 '盘子2', 栈: ['盘子1']
  pop() → 弹出 '盘子1', 栈: []
  pop() → 栈为空，无法出栈！
```

链表版也是同样的逻辑，只不过用 `top` 指针代替数组索引。链表版的好处是**动态大小不会满**，数组版（Python列表）的好处是**更简洁、缓存友好**。

### 示例2: 括号匹配算法

括号匹配是栈最经典的应用之一。检查代码中的 `()`、`[]`、`{}` 是否正确配对。

```python
# 代码见: code/examples/example-02-bracket-match.py

def check_brackets(expression):
    stack = Stack()
    for char in expression:
        if char in "([{":
            stack.push(char)    # 左括号入栈
        elif char in ")]}":
            if stack.is_empty():
                return False    # 右括号多了！
            top = stack.pop()
            if not is_matching_pair(top, char):
                return False    # 不匹配！
    return stack.is_empty()     # 栈空才说明都匹配了
```

**运行结果**：
```
检查表达式: {[()]}
  位置0: '{' → 入栈, 栈: ['{']
  位置1: '[' → 入栈, 栈: ['{', '[']
  位置2: '(' → 入栈, 栈: ['{', '[', '(']
  位置3: ')' ← 匹配 '(', 弹出
  位置4: ']' ← 匹配 '[', 弹出
  位置5: '}' ← 匹配 '{', 弹出
  ✓ 所有括号都匹配！

检查表达式: ([)]
  位置0: '(' → 入栈
  位置1: '[' → 入栈
  位置2: ')' ← 不匹配 '['！
  ✗ 不匹配
```

**原理**：左括号进栈，右括号检查栈顶。这样**最内层的括号最先被匹配**，完美符合嵌套结构！

### 示例3: 表达式求值（中缀转后缀）

计算器是怎么算出 `3 + 4 * 2 = 11` 的？它先把人写的"中缀表达式"转成"后缀表达式"（也叫**逆波兰表达式**）。

```
中缀: 3 + 4 * 2    →  后缀: 3 4 2 * +
中缀: (3 + 4) * 2  →  后缀: 3 4 + 2 *
```

```python
# 代码见: code/examples/example-03-expression-eval.py

# 中缀转后缀（Shunting Yard算法）：
# 1. 数字直接输出
# 2. 左括号入栈
# 3. 右括号：弹到左括号为止
# 4. 运算符：把优先级>=自己的都弹出再入栈
```

**运行结果**：
```
中缀表达式: 3 + 4 * 2
后缀表达式: 3 4 2 * +

计算后缀:
  数字 '3' → 入栈
  数字 '4' → 入栈
  数字 '2' → 入栈
  4.0 * 2.0 = 8.0 → 入栈
  3.0 + 8.0 = 11.0 → 入栈
最终结果: 11  ✓
```

**后缀表达式怎么算？** 遇到数字入栈，遇到运算符弹出两个数计算后入栈。不需要考虑优先级，从左到右一遍搞定！

## 深入理解

### 栈的工作原理

栈本质上就是对"一端"的读写做了限制。你可以把它理解成**只允许从一头进出的隧道**：

```
入 → [ A | B | C ] ← 出（只能从这里）
```

因为只能从一头操作，所以**最后进来的必然最先出去**——这就是LIFO的必然结果。

### 数组实现 vs 链表实现

| 对比维度 | 数组实现 | 链表实现 |
|---------|---------|---------|
| 空间利用率 | 高（连续内存） | 较低（每个节点有额外指针） |
| 扩容 | 需要（Python自动处理） | 不需要 |
| 缓存友好性 | 好（连续内存） | 差（分散内存） |
| 代码简洁度 | 简单 | 稍复杂 |
| 典型场景 | 日常使用 | 嵌入式/需要精确控制 |

**推荐**：日常编程中，**用数组（Python的list）实现栈**。简单、高效、内置支持。

### 时间复杂度总结

| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| push | O(1) | 直接放到末尾 |
| pop | O(1) | 直接从末尾拿走 |
| peek | O(1) | 看一眼末尾 |
| isEmpty | O(1) | 检查长度 |
| 搜索元素 | O(n) | 需要遍历（不是栈的标准操作） |

### 与队列的对比

| 特性 | 栈（Stack） | 队列（Queue） |
|------|------------|--------------|
| 存取原则 | LIFO（后进先出） | FIFO（先进先出） |
| 操作端 | 一端（栈顶） | 两端（头进尾出或尾进头出） |
| 类比 | 盘子堆 | 排队买票 |
| 典型操作 | push/pop | enqueue/dequeue |
| 典型场景 | 撤销、递归、括号 | 任务调度、BFS |

## 实际应用

### 应用场景1: 函数调用栈

这是栈在计算机中最核心的应用！每次你调用一个函数，系统就会在**调用栈**上压入一个"栈帧"（stack frame），里面存着：

- 函数的局部变量
- 函数的参数
- 函数执行完后该返回的地址

```
调用 main()     → 栈: [main帧]
  调用 foo()    → 栈: [main帧, foo帧]
    调用 bar()  → 栈: [main帧, foo帧, bar帧]
    bar()返回   → 栈: [main帧, foo帧]      ← 回到foo继续执行
  foo()返回     → 栈: [main帧]            ← 回到main继续执行
```

这就是为什么**递归太深会"栈溢出"**（Stack Overflow）——栈的空间是有限的，每层递归都要压一个栈帧，栈帧太多了就装不下了！

### 应用场景2: 浏览器前进/后退

浏览器用**两个栈**来实现导航：

```
访问 A → B → C → D
历史栈: [A, B, C, D]    ← 当前在D

点"后退"到C:
历史栈: [A, B, C]    前进栈: [D]
当前页面: C

再点"后退"到B:
历史栈: [A, B]       前进栈: [C, D]
当前页面: B

点"前进"到C:
历史栈: [A, B, C]    前进栈: [D]
当前页面: C
```

### 应用场景3: 撤销操作（Undo）

编辑器里的 Ctrl+Z 也是栈的应用：

```
你依次做了: 打字A → 删除B → 打字C → 格式化D

Ctrl+Z（撤销）:
  第1次: 撤销"格式化D"  ← 最近的先撤
  第2次: 撤销"打字C"
  第3次: 撤销"删除B"
  第4次: 撤销"打字A"
```

每次操作被记录并压入"撤销栈"，撤销时从栈顶弹。

## 常见误区

⚠️ **注意这些坑！**

### 误区1: "栈顶就是索引0"

**错误理解**：
> 我用列表实现栈，push是append，pop应该pop(0)才对吧？

**真相**：
Python的 `list.pop()` 默认弹出**最后一个元素**（索引-1），这才是栈顶！`pop(0)` 弹出的是第一个元素，那就变成FIFO（队列）了。

✅ **正确做法**：
```python
stack = []
stack.append(x)   # push
x = stack.pop()   # pop，弹出最后一个
```

### 误区2: "对空栈pop不会出错"

**错误理解**：
> 我直接pop就行，不用检查isEmpty。

**真相**：
对空栈pop会抛出异常！Python中 `[]` 上调用 `pop()` 会抛出 `IndexError: pop from empty list`。

✅ **正确做法**：
```python
if not stack.is_empty():
    item = stack.pop()
else:
    print("栈为空！")
```
或者用try-except捕获异常。

### 误区3: "栈只能存数字"

**错误理解**：
> 栈不就是存数字的吗？

**真相**：
栈可以存**任何类型**的数据：数字、字符串、对象、甚至另一个栈！栈不关心里面装什么，只关心存取顺序。

✅ **正确做法**：
```python
stack = []
stack.append(42)           # 数字
stack.append("hello")      # 字符串
stack.append([1, 2, 3])    # 列表
stack.append({"key": "val"})  # 字典
```

## 小结与思考

### 本节要点

1. **栈遵循LIFO（后进先出）原则** —— 就像一叠盘子，最后放的先被拿走

2. **栈有四种基本操作** —— push（入栈）、pop（出栈）、peek（查看栈顶）、isEmpty（判空），全部是O(1)

3. **栈可以用数组或链表实现** —— 数组版简单高效，链表版动态大小

4. **栈的应用无处不在** —— 函数调用栈、括号匹配、表达式求值、浏览器导航、撤销操作

5. **栈是"受限但强大"的数据结构** —— 限制存取方向，反而带来了简洁和高效
