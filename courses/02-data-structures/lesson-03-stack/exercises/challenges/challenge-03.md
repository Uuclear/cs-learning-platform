# 挑战3: 每日温度问题

## 难度
⭐⭐⭐⭐

## 描述
给定一个每天温度的列表，对于每一天，返回需要等待多少天才能遇到一个更高的温度。如果之后没有更高的温度，输出0。

这是一个经典的"下一个更大元素"问题的变体，非常适合用单调栈来解决！

## 输入
一个整数列表 `temperatures`，表示每天的温度值。

## 输出
一个整数列表，每个位置的值表示需要等待的天数。

## 示例

**示例 1:**
```
输入: [73, 74, 75, 71, 69, 72, 76, 73]
输出: [1,  1,  4,  2,  1,  1,  0,  0]
解释:
- 第0天(73°): 第1天(74°)更高 → 等1天
- 第1天(74°): 第2天(75°)更高 → 等1天
- 第2天(75°): 第6天(76°)更高 → 等4天
- 第3天(71°): 第5天(72°)更高 → 等2天
- 第5天(72°): 第6天(76°)更高 → 等1天
- 第6天和第7天之后没有更高的 → 0
```

**示例 2:**
```
输入: [30, 40, 50, 60]
输出: [1,  1,  1,  0]
解释: 温度每天都升高
```

**示例 3:**
```
输入: [60, 50, 40, 30]
输出: [0,  0,  0,  0]
解释: 温度每天下降，永远不会遇到更高的
```

## 约束条件
- 1 <= len(temperatures) <= 10^5
- 30 <= temperatures[i] <= 100

## 提示
- 维护一个"单调递减栈"：栈中存储的是索引，对应的温度从高到低
- 当遇到一个比栈顶温度更高的日子时，说明栈顶那天等的人等到了！
- 弹出栈顶，记录等待天数 = 当前索引 - 栈顶索引
- 重复这个过程直到栈顶温度 >= 当前温度

### 进阶思考
- 为什么叫"单调栈"？栈中元素保持了什么单调性？
- 如果不等更高的温度，而是求"下一个更大的数"，算法有什么不同？

### 思路
用单调递减栈：从前往后遍历温度，栈中保存尚未找到更高温度的日期索引。当遇到更高温时，说明栈中这些低温度日期的人等到了！计算等待天数并弹出。

### 代码
```python
def daily_temperatures(temperatures):
    """
    计算每一天需要等待多少天才能遇到更高温度
    使用单调栈优化，时间复杂度O(n)
    """
    n = len(temperatures)
    result = [0] * n       # 初始化：默认都是0（没有更高的）
    stack = []             # 单调栈：存储索引，温度递减

    print(f"温度列表: {temperatures}")
    print("-" * 50)

    for i, temp in enumerate(temperatures):
        print(f"\n第{i}天: {temp}°")
        # 如果当前温度比栈顶对应天数的温度更高
        while stack and temperatures[stack[-1]] < temp:
            prev_index = stack.pop()
            wait_days = i - prev_index
            result[prev_index] = wait_days
            print(f"  第{prev_index}天({temperatures[prev_index]}°)等到了! 等待{wait_days}天")

        stack.append(i)
        print(f"  当前栈: {[(idx, temperatures[idx]) for idx in stack]}")

    print(f"\n结果: {result}")
    return result


if __name__ == "__main__":
    # 测试
    test1 = [73, 74, 75, 71, 69, 72, 76, 73]
    daily_temperatures(test1)
    # 预期输出: [1, 1, 4, 2, 1, 1, 0, 0]

# 输出:
# 温度列表: [73, 74, 75, 71, 69, 72, 76, 73]
# --------------------------------------------------
#
# 第0天: 73°
#   当前栈: [(0, 73)]
#
# 第1天: 74°
#   第0天(73°)等到了! 等待1天
#   当前栈: [(1, 74)]
#
# 第2天: 75°
#   第1天(74°)等到了! 等待1天
#   当前栈: [(2, 75)]
#
# 第3天: 71°
#   当前栈: [(2, 75), (3, 71)]
#
# ...
#
# 结果: [1, 1, 4, 2, 1, 1, 0, 0]
```

### 复杂度分析
- 时间复杂度: O(n)，每个元素最多入栈和出栈各一次
- 空间复杂度: O(n)，最坏情况下栈中有n个元素
