# 链表：手拉手的小朋友们

想象一下，有一群小朋友在操场上排队，每个小朋友都拉着前面和后面小朋友的手。这就是链表的生动比喻！今天我们就来学习这个有趣的数据结构。

> 💡 **思考问题**: 如果让你管理一队随时可能有人加入或离开的队伍，你会选择用数组还是链表？为什么？

---

## 开场白

还记得上节课我们学的数组吗？数组就像一排整齐的座位，每个位置都有固定的编号。但是，如果我们要在中间插入一个新元素，就需要把后面所有的元素都往后挪一位，这可真麻烦！

这时候，链表就派上用场了！链表就像一队手拉手的小朋友，他们不需要站在固定的位置上。当有新朋友要加入时，只需要让旁边的小朋友松开手，然后拉住新朋友的手就行了。删除某个小朋友也很简单，只需要让他两边的朋友直接拉手就好。

这种灵活的结构就是我们今天要学习的链表！

---

## 核心概念

链表是一种线性数据结构，但它与数组不同，链表中的元素在内存中不是连续存储的，而是通过指针（引用）连接在一起。

### 链表的基本组成

链表由一系列**节点（Node）**组成，每个节点包含两个部分：
1. **数据域（Data）**：存储实际的数据
2. **指针域（Pointer/Reference）**：存储下一个节点的地址

### 单链表 vs 双链表 vs 循环链表

#### 单链表（Singly Linked List）
- 每个节点只有一个指向下一个节点的指针
- 只能从前往后遍历
- 内存占用较少

#### 双链表（Doubly Linked List）  
- 每个节点有两个指针：一个指向前一个节点，一个指向后一个节点
- 可以双向遍历
- 内存占用较多，但操作更灵活

#### 循环链表（Circular Linked List）
- 最后一个节点的指针指向第一个节点，形成环
- 适合某些特定场景，如约瑟夫问题

## 代码演示

让我们通过代码来理解这个概念：

### 示例1: 单链表的基本实现

```python
#!/usr/bin/env python3
# -*- coding: utf-8 -*-
"""
单链表实现示例

链表就像一队手拉手的小朋友，每个小朋友（节点）只知道下一个小朋友是谁，
但不知道前面的小朋友是谁。这就是单链表的特点！
"""

class ListNode:
    """链表节点类"""
    def __init__(self, val=0):
        self.val = val          # 节点存储的值
        self.next = None        # 指向下一个节点的指针


class SinglyLinkedList:
    """单链表类"""
    def __init__(self):
        self.head = None        # 头节点，指向链表的第一个节点

    def append(self, val):
        """在链表末尾添加节点"""
        new_node = ListNode(val)
        if not self.head:
            # 如果链表为空，新节点就是头节点
            self.head = new_node
        else:
            # 找到链表的最后一个节点
            current = self.head
            while current.next:
                current = current.next
            current.next = new_node

    def insert_at_head(self, val):
        """在链表头部插入节点"""
        new_node = ListNode(val)
        new_node.next = self.head
        self.head = new_node

    def delete(self, val):
        """删除第一个值为val的节点"""
        if not self.head:
            return False

        # 如果要删除的是头节点
        if self.head.val == val:
            self.head = self.head.next
            return True

        # 查找要删除的节点
        current = self.head
        while current.next and current.next.val != val:
            current = current.next

        # 如果找到了要删除的节点
        if current.next:
            current.next = current.next.next
            return True

        return False

    def find(self, val):
        """查找值为val的节点，返回索引位置"""
        current = self.head
        index = 0
        while current:
            if current.val == val:
                return index
            current = current.next
            index += 1
        return -1  # 未找到

    def display(self):
        """显示链表内容"""
        elements = []
        current = self.head
        while current:
            elements.append(str(current.val))
            current = current.next
        return " -> ".join(elements) + " -> None"
```

**运行结果**：
```
=== 单链表演示 ===
初始链表:  -> None
添加1,2,3后: 1 -> 2 -> 3 -> None
在头部插入0后: 0 -> 1 -> 2 -> 3 -> None
查找元素2的位置: 2
查找元素5的位置: -1
删除元素2后: 0 -> 1 -> 3 -> None
删除头节点0后: 1 -> 3 -> None
```

这个示例展示了单链表的基本操作：在末尾添加、在头部插入、删除指定值、查找元素位置。注意删除头节点的特殊处理！

### 示例2: 双链表的实现

```python
#!/usr/bin/env python3
# -*- coding: utf-8 -*-
"""
双链表实现示例

双链表就像一队手拉手的小朋友，每个小朋友（节点）不仅知道下一个小朋友是谁，
还知道前面的小朋友是谁。这样就可以双向移动了！
"""

class DoublyListNode:
    """双链表节点类"""
    def __init__(self, val=0):
        self.val = val          # 节点存储的值
        self.next = None        # 指向下一个节点的指针
        self.prev = None        # 指向前一个节点的指针


class DoublyLinkedList:
    """双链表类"""
    def __init__(self):
        self.head = None        # 头节点
        self.tail = None        # 尾节点

    def append(self, val):
        """在链表末尾添加节点"""
        new_node = DoublyListNode(val)
        if not self.head:
            # 链表为空
            self.head = new_node
            self.tail = new_node
        else:
            # 连接到当前尾节点
            new_node.prev = self.tail
            self.tail.next = new_node
            self.tail = new_node

    def prepend(self, val):
        """在链表头部添加节点"""
        new_node = DoublyListNode(val)
        if not self.head:
            # 链表为空
            self.head = new_node
            self.tail = new_node
        else:
            # 连接到当前头节点
            new_node.next = self.head
            self.head.prev = new_node
            self.head = new_node

    def delete(self, val):
        """删除第一个值为val的节点"""
        current = self.head
        while current:
            if current.val == val:
                # 如果是头节点
                if current == self.head:
                    self.head = current.next
                    if self.head:
                        self.head.prev = None
                    else:
                        self.tail = None  # 链表变空
                # 如果是尾节点
                elif current == self.tail:
                    self.tail = current.prev
                    self.tail.next = None
                # 如果是中间节点
                else:
                    current.prev.next = current.next
                    current.next.prev = current.prev
                return True
            current = current.next
        return False

    def display_forward(self):
        """正向显示链表"""
        elements = []
        current = self.head
        while current:
            elements.append(str(current.val))
            current = current.next
        return " <-> ".join(elements) + " <-> None"

    def display_backward(self):
        """反向显示链表"""
        elements = []
        current = self.tail
        while current:
            elements.append(str(current.val))
            current = current.prev
        return " <-> ".join(elements) + " <-> None"
```

**运行结果**：
```
=== 双链表演示 ===
初始链表(正向):  <-> None
初始链表(反向):  <-> None
添加1,2,3后(正向): 1 <-> 2 <-> 3 <-> None
添加1,2,3后(反向): 3 <-> 2 <-> 1 <-> None
在头部添加0后(正向): 0 <-> 1 <-> 2 <-> 3 <-> None
在头部添加0后(反向): 3 <-> 2 <-> 1 <-> 0 <-> None
查找元素2的位置: 2
查找元素5的位置: -1
删除元素2后(正向): 0 <-> 1 <-> 3 <-> None
删除元素2后(反向): 3 <-> 1 <-> 0 <-> None
删除头节点0后(正向): 1 <-> 3 <-> None
删除头节点0后(反向): 3 <-> 1 <-> None
```

双链表的优势在于可以双向遍历，删除操作也更容易，因为可以直接访问前驱节点。

### 示例3: 链表的经典操作

```python
#!/usr/bin/env python3
# -*- coding: utf-8 -*-
"""
链表常见操作示例

这里展示一些链表的经典操作，比如反转链表、检测环、合并链表等。
这些是面试中经常遇到的问题！
"""

class ListNode:
    """链表节点类"""
    def __init__(self, val=0):
        self.val = val
        self.next = None


def reverse_linked_list(head):
    """
    反转链表
    
    时间复杂度: O(n)
    空间复杂度: O(1)
    """
    prev = None
    current = head
    
    while current:
        next_temp = current.next  # 保存下一个节点
        current.next = prev       # 反转当前节点的指针
        prev = current           # 移动prev指针
        current = next_temp      # 移动current指针
    
    return prev  # prev现在是新的头节点


def has_cycle(head):
    """
    检测链表是否有环（Floyd's Cycle Detection Algorithm）
    
    使用快慢指针：快指针每次走两步，慢指针每次走一步
    如果有环，快指针最终会追上慢指针
    """
    if not head or not head.next:
        return False
    
    slow = head
    fast = head.next
    
    while slow != fast:
        if not fast or not fast.next:
            return False
        slow = slow.next
        fast = fast.next.next
    
    return True


def merge_two_sorted_lists(list1, list2):
    """
    合并两个有序链表
    
    时间复杂度: O(m + n)
    空间复杂度: O(1)
    """
    dummy = ListNode(0)  # 虚拟头节点
    current = dummy
    
    while list1 and list2:
        if list1.val <= list2.val:
            current.next = list1
            list1 = list1.next
        else:
            current.next = list2
            list2 = list2.next
        current = current.next
    
    # 连接剩余的节点
    current.next = list1 if list1 else list2
    
    return dummy.next
```

**运行结果**：
```
=== 链表操作演示 ===

1. 反转链表:
原链表: 1 -> 2 -> 3 -> 4 -> 5 -> None
反转后: 5 -> 4 -> 3 -> 2 -> 1 -> None

2. 检测环:
无环链表: 1 -> 2 -> 3 -> None
是否有环: False
有环链表: 1 -> 2 -> 3 -> 4 -> 2 (形成环)
是否有环: True

3. 合并有序链表:
链表A: 1 -> 3 -> 5 -> 7 -> None
链表B: 2 -> 4 -> 6 -> 8 -> None
合并后: 1 -> 2 -> 3 -> 4 -> 5 -> 6 -> 7 -> 8 -> None
```

这些经典操作展示了链表的强大功能。特别是快慢指针技巧，在很多链表问题中都非常有用！

## 深入理解

### 工作原理

链表的核心思想是**动态连接**。每个节点只关心自己的数据和下一个节点的位置，这种设计使得插入和删除操作变得非常高效。

当你在链表中间插入一个新节点时，只需要修改相邻节点的指针，而不需要像数组那样移动大量元素。同样，删除节点也只需要调整指针即可。

### 时间/空间复杂度

| 操作 | 数组 | 单链表 | 双链表 |
|------|------|--------|--------|
| 访问元素 | O(1) | O(n) | O(n) |
| 在末尾插入 | O(1) | O(n) | O(1) |
| 在开头插入 | O(n) | O(1) | O(1) |
| 在中间插入 | O(n) | O(1)* | O(1)* |
| 删除元素 | O(n) | O(1)* | O(1)* |

*注：O(1)的前提是已经知道要操作的位置

### 与其他概念的对比

| 特性 | 链表 | 数组 |
|------|------|------|
| 内存布局 | 分散存储，通过指针连接 | 连续存储 |
| 大小 | 动态增长，理论上无限 | 固定大小（静态数组）或需要重新分配（动态数组） |
| 插入/删除 | O(1)（已知位置） | O(n) |
| 随机访问 | 不支持，O(n) | 支持，O(1) |
| 内存开销 | 每个元素额外存储指针 | 无额外开销 |

## 实际应用

### 应用场景1: LRU缓存

LRU（Least Recently Used）缓存是一种常用的缓存淘汰策略。它需要快速地将最近使用的元素移动到头部，并在缓存满时删除尾部元素。双链表配合哈希表是实现LRU缓存的经典方案。

### 应用场景2: 浏览器前进后退

浏览器的历史记录功能可以用双链表实现。每个页面是一个节点，向前浏览就是在链表中向后移动，向后浏览就是向前移动。这样可以轻松实现前进后退功能。

### 应用场景3: 音乐播放列表

音乐播放器的播放列表通常使用链表实现，特别是循环播放模式。链表可以轻松实现下一首、上一首、随机播放等功能，而且插入新歌曲或删除歌曲都很高效。

## 常见误区

⚠️ **注意这些坑！**

### 误区1: 忘记处理空链表

很多初学者在实现链表操作时，忘记了处理空链表的情况，导致程序崩溃。

✅ **正确做法**: 在每个操作开始时，先检查链表是否为空（`head is None`），并做相应的处理。

### 误区2: 删除头节点时忘记更新头指针

删除头节点是一个特殊情况，因为需要更新链表的头指针。如果忘记更新，会导致内存泄漏或者无法正确访问链表。

✅ **正确做法**: 删除头节点时，要将`head`指向原来的第二个节点（`head = head.next`）。

### 误区3: 忘记释放内存（在C/C++中）

在手动管理内存的语言中，删除节点后要记得释放内存，否则会造成内存泄漏。

✅ **正确做法**: 在C/C++中，删除节点后要调用`free()`或`delete`。在Python/Java等有垃圾回收的语言中，只要没有引用指向该节点，它就会被自动回收。

## 小结与思考

### 本节要点

- 链表是一种通过指针连接的线性数据结构
- 单链表只能单向遍历，双链表可以双向遍历
- 链表的插入和删除操作时间复杂度为O(1)（已知位置）
- 链表不支持随机访问，访问元素需要O(n)时间
- 快慢指针是解决链表问题的重要技巧

### 思考题

1. 为什么链表的插入操作说是O(1)，但实际上很多时候还是需要O(n)的时间？

3. 如果要实现一个支持在任意位置高效插入和删除的数据结构，你会如何设计？链表和数组各有什么优缺点？
