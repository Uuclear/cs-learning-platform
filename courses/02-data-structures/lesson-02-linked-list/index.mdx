# 链表：手拉手的小朋友们

上节课我们学了**数组**——一群人按编号排排坐，每个人紧紧挨着。但生活中还有一种完全不同的排队方式，想象一下这个场景……

> 🎵 "找呀找呀找朋友，找到一个好朋友，敬个礼呀握个手，你是我的好朋友……"
>
> 幼儿园里，小朋友们手拉手排成一队，**每个人可以站在任何位置**，只要牵着前后两个人的手就行。

---

## 开场白

想象一下这两个排队场景：

**场景一（数组）**：军训站军姿

```
[小明] [小红] [小刚] [小丽] [小强]
  0      1      2      3      4
```
所有人必须紧紧挨着，中间不能留空。你想插队到第2个位置？那小红、小刚、小丽、小强都得**往后挪一步**。累不累！

**场景二（链表）**：小朋友手拉手

```
小明 --- 小红 --- 小刚 --- 小丽 --- 小强
```
小朋友们可以分散站在操场的任何位置，只要牵着前后两个人的手，队伍就"连"起来了。你想插队？

```
小明 --- 小红 --- 你 --- 小刚 --- 小丽 --- 小强
```
只需要小红放开小刚的手，牵起你的手，然后你牵起小刚的手。**其他人完全不用动！**

这就是链表的核心思想：**用指针（手拉手）把分散在内存各处的节点连接起来**。

## 核心概念

### 什么是链表？

**链表（Linked List）**是一种线性数据结构，由一系列**节点（Node）**组成。每个节点包含两部分：

1. **数据域**：存储实际数据（就像小朋友手里的玩具）
2. **指针域**：存储指向下一个节点的引用（就像小朋友伸出的手）

```
┌─────────┬─────────┐     ┌─────────┬─────────┐     ┌─────────┬─────────┐
│ 数据: 10│ next ───┼────>│ 数据: 20│ next ───┼────>│ 数据: 30│ next ───┼────> None
└─────────┴─────────┘     └─────────┴─────────┘     └─────────┴─────────┘
```

和数组对比一下：

| 特征 | 数组 | 链表 |
|------|------|------|
| 内存布局 | **连续**，紧紧挨着 | **不连续**，分散各处 |
| 大小 | 固定（动态数组可扩容但需要复制） | 灵活，随时增删 |
| 访问方式 | 索引直接访问 O(1) | 从头遍历 O(n) |
| 插入/删除 | 需要移动元素 O(n) | 只改指针 O(1) |

### 链表的基本结构

让我们用代码看看一个链表节点长什么样：

```python
class Node:
    def __init__(self, data):
        self.data = data    # 数据域：小朋友的玩具
        self.next = None    # 指针域：牵着下一个小朋友的手
```

就这么简单！一个数据，一个指针，链表的核心结构就这两个东西。

## 代码演示

### 示例1: 单向链表的基本操作

下面是一个完整的单向链表实现，包含创建、插入、删除、查找等操作：

```python
class Node:
    def __init__(self, data):
        self.data = data
        self.next = None

class SinglyLinkedList:
    def __init__(self):
        self.head = None      # 头节点指针

    def append(self, data):
        """在链表尾部添加元素"""
        new_node = Node(data)
        if self.head is None:
            self.head = new_node
        else:
            current = self.head
            while current.next is not None:
                current = current.next
            current.next = new_node

    def prepend(self, data):
        """在头部插入（O(1)操作）"""
        new_node = Node(data)
        new_node.next = self.head
        self.head = new_node

    def remove(self, data):
        """删除第一个匹配到的元素"""
        if self.head is None:
            return
        if self.head.data == data:
            self.head = self.head.next
            return
        current = self.head
        while current.next is not None:
            if current.next.data == data:
                current.next = current.next.next  # 跳过被删的节点
                return
            current = current.next

    def to_list(self):
        """转换为列表方便查看"""
        result = []
        current = self.head
        while current is not None:
            result.append(current.data)
            current = current.next
        return result

# 测试
ll = SinglyLinkedList()
ll.append(10)
ll.append(20)
ll.append(30)
print(f"追加后: {ll.to_list()}")

ll.prepend(5)
print(f"头部插入5: {ll.to_list()}")

ll.remove(20)
print(f"删除20: {ll.to_list()}")
```

**运行结果**：
```
追加后: [10, 20, 30]
头部插入5: [5, 10, 20, 30]
删除20: [5, 10, 30]
```

**解读**：
- `append` 需要遍历到尾节点才能追加，时间复杂度 O(n)
- `prepend` 只需要改头指针，时间复杂度 O(1) —— 链表的优势！
- `remove` 需要找到目标位置，但一旦找到，删除操作只需要改一个指针，O(1)

### 示例2: 双向链表——能往前走能往后退

单向链表有个缺点：**只能从前往后走**。想从后往前走？对不起，走不回去。

双向链表给每个节点多加了一个`prev`指针，指向前一个节点：

```python
class DoublyNode:
    def __init__(self, data):
        self.data = data
        self.next = None      # 后继（指向下一个）
        self.prev = None      # 前驱（指向上一个）

class DoublyLinkedList:
    def __init__(self):
        self.head = None
        self.tail = None      # 多了尾指针！

    def append(self, data):
        """双向链表的append是O(1)！因为有tail指针"""
        new_node = DoublyNode(data)
        if self.head is None:
            self.head = self.tail = new_node
        else:
            new_node.prev = self.tail    # 新节点的前驱指向旧尾
            self.tail.next = new_node    # 旧尾的后继指向新节点
            self.tail = new_node         # 更新尾指针

    def forward(self):
        """从头到尾遍历"""
        result = []
        current = self.head
        while current:
            result.append(current.data)
            current = current.next
        return result

    def backward(self):
        """从尾到头遍历（单向链表做不到的！）"""
        result = []
        current = self.tail
        while current:
            result.append(current.data)
            current = current.prev
        return result

# 测试
dll = DoublyLinkedList()
for c in ['A', 'B', 'C', 'D']:
    dll.append(c)

print(f"正向: {dll.forward()}")
print(f"反向: {dll.backward()}")
```

**运行结果**：
```
正向: ['A', 'B', 'C', 'D']
反向: ['D', 'C', 'B', 'A']
```

**解读**：
- 双向链表的`append`是**O(1)**！因为有`tail`指针，不需要遍历
- 支持反向遍历，这在很多场景非常有用
- 代价：每个节点多了一个`prev`指针，内存开销更大

### 示例3: 链表 vs 数组的性能实测

光说不练假把式，让我们用实际数据对比一下两者的性能差异：

```python
import time

# 对比：在头部连续插入10000个元素
n = 10000

# 链表：每次O(1)，总O(n)
start = time.perf_counter()
result_ll = []
class Node:
    def __init__(self, d): self.data = d; self.next = None

head = None
for i in range(n):
    new = Node(i)
    new.next = head
    head = new
ll_time = time.perf_counter() - start

# 数组：每次插入到开头O(n)，总O(n²)
arr = []
start = time.perf_counter()
for i in range(n):
    arr.insert(0, i)
arr_time = time.perf_counter() - start

print(f"头部插入 {n} 次:")
print(f"  链表耗时: {ll_time*1000:.2f} ms")
print(f"  数组耗时: {arr_time*1000:.2f} ms")
print(f"  链表快 {arr_time/ll_time:.0f} 倍")
```

**运行结果**（具体数值因机器而异，趋势一致）：
```
头部插入 10000 次:
  链表耗时: 1.23 ms
  数组耗时: 12.45 ms
  链表快 10 倍
```

**解读**：
- 数组在头部插入时，每次需要把后面所有元素往后挪，是 O(n) 操作
- 链表在头部插入只需要改一个指针，是 O(1) 操作
- 10000次下来，链表快了整整10倍！

## 深入理解

### 链表的类型

我们已经认识了两种链表，其实还有一种：

| 类型 | 结构 | 特点 | 示意图 |
|------|------|------|--------|
| **单向链表** | 每个节点有一个next指针 | 只能从前往后走 | A → B → C → None |
| **双向链表** | 每个节点有next和prev两个指针 | 可以双向行走 | A ↔ B ↔ C ↔ None |
| **循环链表** | 尾节点的next指向头节点 | 从任何节点出发都能回到起点 | A → B → C → A（闭环） |

**循环链表**的特点是最后一个节点的`next`指向头节点，形成一个环。这种结构常用于：
- 操作系统的**循环调度**（Round Robin）
- 音乐播放器的**循环播放**
- 游戏的**回合制**轮流

### 工作原理：插入和删除的本质

**插入操作的本质**就是改指针：

```
插入前:  A ──→ B
         
插入C在A和B之间:
         
第一步:  C ──→ B       （C的next指向B）
第二步:  A ──→ C       （A的next指向C）
         
结果:    A ──→ C ──→ B
```

注意：**顺序很重要**！如果先改A的next指向C，那就找不到B了，链表就"断"了。

**删除操作的本质**就是跳过要删的节点：

```
删除前:  A ──→ B ──→ C

跳过B:   A ─────────→ C

结果:    A ──→ C      （B被"跳过"了，没人牵它的手了）
```

### 时间/空间复杂度汇总

| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| **访问第i个元素** | O(n) | 需要从头节点一步一步走 |
| **查找值为x的元素** | O(n) | 需要遍历检查每个节点 |
| **头部插入** | O(1) | 只需要改头指针 |
| **尾部插入** | O(n) / O(1)* | 单向链表需遍历到尾；双向链表有tail是O(1) |
| **已知位置的插入** | O(1) | 只改两个指针 |
| **已知位置的删除** | O(1) | 只改两个指针 |

> \* 这里说的O(n)是指需要遍历找到插入/删除位置，而插入/删除本身是O(1)。如果已经知道了位置，链表操作就是O(1)。

### 链表 vs 数组：全面对比

| 特性 | 数组 | 链表 |
|------|------|------|
| **内存布局** | 连续 | 不连续 |
| **随机访问** | ✅ O(1) | ❌ O(n) |
| **头部插入** | ❌ O(n) | ✅ O(1) |
| **尾部插入** | ✅ O(1)* | ❌ O(n) / ✅ O(1)† |
| **中间插入/删除** | ❌ O(n) 需要移动 | ✅ O(1) 已知位置 |
| **内存开销** | 小（无额外指针） | 大（每个节点多1-2个指针） |
| **缓存友好** | ✅ 连续内存，CPU缓存命中率高 | ❌ 节点分散，缓存命中率低 |
| **大小灵活** | ❌ 固定大小 | ✅ 动态伸缩 |

> * 数组尾部插入是摊还O(1)（动态数组扩容时需要复制）
> † 双向链表因为有tail指针，尾部插入是O(1)

### 什么时候用数组，什么时候用链表？

- **选数组**：频繁随机访问、尾部追加、对内存敏感、数据量已知
- **选链表**：频繁在中间插入/删除、头部频繁操作、数据量不确定

## 实际应用

链表在真实世界中无处不在，让我们看两个例子：

### 应用场景1：浏览器的"前进/后退"按钮

你用过浏览器的后退和前进按钮吗？背后的数据结构就是**双向链表**！

```
[首页] ↔ [搜索页] ↔ [文章页] ↔ [评论页]
               ↑
             当前页面
```

- 点击"后退"：沿着`prev`指针往前走
- 点击"前进"：沿着`next`指针往前走
- 访问新页面：在当前节点后面插入新节点，截断后面的历史记录

用数组能做到吗？可以，但需要在当前位置后面插入和删除，效率低得多。

### 应用场景2：操作系统的内存管理

操作系统管理空闲内存块时，常用**链表**来维护：

```
空闲块1 (64KB) ──→ 空闲块2 (128KB) ──→ 空闲块3 (32KB) ──→ ...
```

- 程序申请内存：遍历链表找到足够大的空闲块
- 程序释放内存：把释放的块插回链表

由于内存块是分散的，而且需要频繁地合并和分割，链表天然适合这种场景。如果用数组，每次合并内存块都要移动大量数据，效率极低。

## 常见误区

⚠️ **注意这些坑！**

### 误区1: "链表的插入和删除都是O(1)"

很多人一知半解地说链表插入删除快，但其实——

- **插入/删除操作本身**确实是O(1)，只需要改几个指针
- 但**找到插入/删除的位置**需要O(n)的遍历

所以完整地说：
- **已知位置**的插入/删除：O(1)
- **不知道位置**，需要从值查找再插入/删除：O(n) + O(1) = O(n)

✅ **正确理解**: 链表的优势在于"已知位置的插入/删除"，而不是"找到位置的速度"。

### 误区2: "链表比数组省内存"

恰恰相反！链表的每个节点都需要额外的指针空间：

```
数组:  [10][20][30][40]  ← 只有数据

链表:  [10|→]──[20|→]──[30|→]──[40|None]
        ↑ 指针也要占内存！
```

在64位系统中，每个指针占8字节。一个存4字节整数的链表节点，实际需要4+8=12字节。内存开销比数组大得多。

✅ **正确理解**: 链表内存更大，但它的大小是**灵活的**。

### 误区3: "删除链表节点只需要改一个指针"

很多人以为删除节点只需要让前一个节点的next跳过后面的节点。但如果你的链表还需要维护`tail`指针，或者要删除的是**头节点**或**尾节点**，那就需要额外处理：

```python
# 删头节点：需要更新head
# 删尾节点：需要更新tail（单向链表需要遍历找到倒数第二个！）
```

✅ **正确做法**: 考虑所有边界情况：空链表、单节点链表、头/尾节点的特殊处理。

### 练习1: 打印链表的中间节点 ⭐

给定一个单向链表，找出它的中间节点。如果链表有偶数个节点，返回中间两个中的后一个。

**要求**：
- 只遍历链表一次
- 不能先计算长度再找中间位置

**示例**：
```
输入: 1 → 2 → 3 → 4 → 5
输出: 3（中间节点）

输入: 1 → 2 → 3 → 4
输出: 3（偶数取后一个）
```

<details>
<summary>💡 提示（点击展开）</summary>

试试"快慢指针"的方法：两个人同时从起点出发，一个人每次走一步，另一个每次走两步。当快的人走到终点时，慢的人刚好走到中间！这就是著名的"龟兔赛跑"技巧。

</details>

### 练习2: 反转链表 ⭐⭐

实现一个函数，将单向链表反转。

**要求**：
- 原地反转，不使用额外的链表
- 处理空链表和单节点链表的情况

**示例**：
```
输入: 1 → 2 → 3 → 4 → None
输出: 4 → 3 → 2 → 1 → None
```

<details>
<summary>💡 提示（点击展开）</summary>

需要三个指针：`prev`（前一个）、`current`（当前）、`next_node`（下一个）。遍历链表时，让每个节点的`next`反过来指向`prev`。关键：先保存`next_node`，不然会丢失后面的节点！

</details>

### 练习3: 合并两个有序链表 ⭐⭐⭐

给定两个**升序**排列的链表，合并成一个新的升序链表。

**要求**：
- 复用原有节点，不创建新节点
- 保持O(m + n)的时间复杂度

**示例**：
```
输入: l1 = 1 → 3 → 5 → None
      l2 = 2 → 4 → 6 → None
输出: 1 → 2 → 3 → 4 → 5 → 6 → None
```

<details>
<summary>💡 提示（点击展开）</summary>

用一个"哨兵"节点（虚拟头节点）来简化边界处理。每次比较l1和l2当前节点的值，把较小的那个接到结果链表后面。一个链表走完后，把另一个直接接上。

</details>

## 小结与思考

### 本节要点

1. **链表的本质**是用指针把分散的节点连接起来，节点在内存中**不连续**存储
2. **单向链表**只能从头往后走，**双向链表**可以前后走，**循环链表**首尾相连形成环
3. **插入和删除**操作本身是O(1)（已知位置时），但**查找和遍历**是O(n)
4. **数组 vs 链表**没有绝对优劣，关键看使用场景：数组适合随机访问，链表适合频繁插入/删除

### 思考题

1. 如果让你实现一个支持O(1)头部插入、O(1)尾部插入、O(1)尾部删除的数据结构，你会怎么做？（提示：双向链表+头尾指针）

2. 链表能不能支持二分查找？如果不能，为什么？如果能，需要做什么改进？（提示：跳表Skip List就是答案）

3. 假设你需要频繁在链表中间插入元素，但每次都需要先找到插入位置。在这种情况下，链表和数组哪个实际更快？（提示：考虑缓存友好性和实际常数因子）
