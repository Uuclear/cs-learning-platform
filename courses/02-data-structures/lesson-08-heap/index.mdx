# 堆：优先级队列的秘密

> 医院急诊室里，心跳停止的病人一定比感冒咳嗽的病人先得到救治吗？这就是堆要解决的问题——不是谁先来谁先服务，而是谁重要谁先服务。

---

## 开场白

同学们好！今天我们来聊聊一种特别的数据结构——**堆**（Heap）。

说到"堆"，你可能想到一堆沙子、一堆书，或者一堆没洗的衣服。但计算机科学里的堆可不是"随便堆在一起"的意思，它有着严格的规矩和秩序。

**想象一个场景：**

你深夜11点打开外卖软件，发现你的订单前面已经有47个人在排队。但是这时候来了一个"加急单"——一位医生刚做完10小时手术，急需补充能量。外卖平台该怎么做？

**让加急单排到47个人后面？** 那医生可能饿晕了。

**把加急单直接插到最前面？** 那后面的47个人要炸锅了。

**正确答案是：用一个"优先级队列"！** 重要程度高的自动往前排，这才是合理的方案。而堆，就是实现优先级队列的最佳数据结构。

**思考一下：**
- 如果我们用普通的数组来存储优先级队列，插入新元素要怎么做？
- 如果每次都扫描整个数组找最大值，效率会怎样？
- 有没有一种结构，既能快速找到最大值，又能快速插入新元素？

> 💡 **思考问题**: 队列是"先进先出"，栈是"后进先出"，那"按优先级出"的数据结构应该长什么样？它的时间复杂度能做到比O(n)更好吗？

## 核心概念

### 堆的定义

堆是一种**完全二叉树**，它满足一个特殊性质——**堆序性**。

**什么是完全二叉树？** 还记得我们上节课学的树吗？完全二叉树就是一种"尽量填满"的树——除了最后一层，其他层都是满的，而且最后一层的节点都靠左排列。就像一个排队买奶茶的队伍，前面的人站满了，后面的人只能从左到右排。

**什么是堆序性？** 这才是堆的灵魂。根据堆序性的不同，堆分为两种：

#### 最大堆（Max Heap）

```
        30
       /  \
     20    25
    /  \   /
   10  15  5
```

规则很简单：**每个节点的值都大于或等于其子节点的值**。

所以根节点一定是整个堆中的最大值。这就像一家公司的组织架构——CEO的工资最高，每个部门经理的工资都比手下高。

#### 最小堆（Min Heap）

```
        5
       /  \
     10    25
    /  \   /
   30  15 20
```

和最大堆相反：**每个节点的值都小于或等于其子节点的值**。

根节点是最小值。这就好比医院急诊——病情最轻的在最外面，病情最重的（优先级最高）的在最里面等着被处理。

### 堆的两个关键性质

1. **结构性质**：堆是一棵完全二叉树。这使得我们可以用**数组**来表示堆，不需要指针！
2. **堆序性质**：父节点和子节点之间的大小关系（最大堆：父>=子；最小堆：父<=子）。

**重要提醒**：堆**不保证**左右子节点之间的大小关系！上面最大堆的例子中，根节点30的左子节点是20，右子节点是25，20<25。堆只关心父子关系，不关心兄弟关系。

## 代码演示

让我们通过代码来理解堆的具体实现。

### 示例1：堆的基本实现

首先，我们手写一个最大堆，理解核心的上浮和下沉操作：

```python
# 示例1：堆的基本实现（最大堆）
# 用数组实现最大堆，包含插入、删除、上浮、下沉操作

class MaxHeap:
    """
    最大堆实现
    核心思想：每个节点的值都大于等于其子节点的值
    数组表示：父节点i的子节点在2*i+1和2*i+2
    """

    def __init__(self):
        # 用列表存储堆的元素，索引0是根节点
        self.heap = []

    def parent(self, i):
        """返回父节点的索引"""
        return (i - 1) // 2

    def left_child(self, i):
        """返回左子节点的索引"""
        return 2 * i + 1

    def right_child(self, i):
        """返回右子节点的索引"""
        return 2 * i + 2

    def has_parent(self, i):
        """判断是否有父节点"""
        return self.parent(i) >= 0

    def has_left_child(self, i):
        """判断是否有左子节点"""
        return self.left_child(i) < len(self.heap)

    def has_right_child(self, i):
        """判断是否有右子节点"""
        return self.right_child(i) < len(self.heap)

    def swap(self, i, j):
        """交换两个位置的元素"""
        self.heap[i], self.heap[j] = self.heap[j], self.heap[i]

    def sift_up(self, i):
        """
        上浮操作：将新插入的元素向上调整到正确位置
        如果比父节点大，就和父节点交换，直到满足堆性质
        """
        while self.has_parent(i) and self.heap[i] > self.heap[self.parent(i)]:
            self.swap(i, self.parent(i))
            i = self.parent(i)

    def sift_down(self, i):
        """
        下沉操作：将根节点向下调整到正确位置
        如果比子节点小，就和较大的子节点交换
        """
        while self.has_left_child(i):
            # 先找出较大的子节点
            larger = self.left_child(i)
            if self.has_right_child(i) and self.heap[self.right_child(i)] > self.heap[larger]:
                larger = self.right_child(i)

            # 如果当前节点已经是最大的，就不用下沉了
            if self.heap[i] > self.heap[larger]:
                break

            # 否则和较大的子节点交换
            self.swap(i, larger)
            i = larger

    def insert(self, value):
        """
        插入元素：放到数组末尾，然后上浮到正确位置
        时间复杂度：O(log n)
        """
        self.heap.append(value)
        self.sift_up(len(self.heap) - 1)

    def extract_max(self):
        """
        删除并返回最大值（根节点）
        把最后一个元素移到根，然后下沉
        时间复杂度：O(log n)
        """
        if len(self.heap) == 0:
            return None

        if len(self.heap) == 1:
            return self.heap.pop()

        # 把最大值（根）保存下来
        max_value = self.heap[0]
        # 把最后一个元素移到根
        self.heap[0] = self.heap.pop()
        # 从根开始下沉
        self.sift_down(0)

        return max_value

    def peek(self):
        """查看最大值但不删除"""
        if len(self.heap) == 0:
            return None
        return self.heap[0]

    def size(self):
        """返回堆的大小"""
        return len(self.heap)

    def is_empty(self):
        """判断堆是否为空"""
        return len(self.heap) == 0

    def display(self):
        """打印堆的内容"""
        print(f"堆的内容: {self.heap}")


# ========== 测试代码 ==========
if __name__ == "__main__":
    print("=== 最大堆基本操作演示 ===\n")

    # 创建一个最大堆
    heap = MaxHeap()

    # 插入元素
    print("--- 插入元素 ---")
    values = [10, 20, 5, 30, 15, 25]
    for v in values:
        heap.insert(v)
        print(f"插入 {v} -> ", end="")
        heap.display()

    print()
    print("--- 查看最大值（不删除） ---")
    print(f"当前最大值: {heap.peek()}")
    heap.display()

    print()
    print("--- 依次删除最大值 ---")
    while not heap.is_empty():
        max_val = heap.extract_max()
        print(f"删除最大值 {max_val} -> ", end="")
        heap.display()

# 输出:
# === 最大堆基本操作演示 ===
#
# --- 插入元素 ---
# 插入 10 -> 堆的内容: [10]
# 插入 20 -> 堆的内容: [20, 10]
# 插入 5 -> 堆的内容: [20, 10, 5]
# 插入 30 -> 堆的内容: [30, 20, 5, 10]
# 插入 15 -> 堆的内容: [30, 20, 5, 10, 15]
# 插入 25 -> 堆的内容: [30, 20, 25, 10, 15, 5]
#
# --- 查看最大值（不删除） ---
# 当前最大值: 30
# 堆的内容: [30, 20, 25, 10, 15, 5]
#
# --- 依次删除最大值 ---
# 删除最大值 30 -> 堆的内容: [25, 20, 5, 10, 15]
# 删除最大值 25 -> 堆的内容: [20, 15, 5, 10]
# 删除最大值 20 -> 堆的内容: [15, 10, 5]
# 删除最大值 15 -> 堆的内容: [10, 5]
# 删除最大值 10 -> 堆的内容: [5]
# 删除最大值 5 -> 堆的内容: []
```

**核心要点**：

- **插入**：把新元素放到数组末尾，然后**上浮**到正确位置。就像新来的员工先站在最外面，然后看他能力——比他上级强就换位子。
- **删除最大值**：把根节点（最大值）拿走，把末尾元素搬到根，然后**下沉**。就像CEO退休了，让实习生暂时坐CEO的位置，然后一路降职直到合适的位置。

### 示例2：堆排序算法

堆不仅能找最大值，还能用来排序！堆排序是效率极高的排序算法：

```python
# 示例2：堆排序算法
# 利用堆的性质实现O(n log n)的排序算法

def build_max_heap(arr):
    """
    把无序数组构建成最大堆
    从最后一个非叶子节点开始，依次下沉
    时间复杂度：O(n) - 神奇！不是O(n log n)
    """
    n = len(arr)
    # 最后一个非叶子节点的索引是 (n//2) - 1
    for i in range(n // 2 - 1, -1, -1):
        sift_down(arr, n, i)


def sift_down(arr, n, i):
    """
    下沉操作：将索引i的元素向下调整
    arr: 数组
    n: 堆的有效大小
    i: 当前节点索引
    """
    largest = i
    left = 2 * i + 1   # 左子节点
    right = 2 * i + 2  # 右子节点

    # 如果左子节点比当前节点大
    if left < n and arr[left] > arr[largest]:
        largest = left

    # 如果右子节点比当前节点大
    if right < n and arr[right] > arr[largest]:
        largest = right

    # 如果最大值不是当前节点，交换并继续下沉
    if largest != i:
        arr[i], arr[largest] = arr[largest], arr[i]
        sift_down(arr, n, largest)


def heap_sort(arr):
    """
    堆排序算法
    1. 构建最大堆
    2. 反复把堆顶（最大值）和末尾交换
    3. 缩小堆的范围，重新调整
    时间复杂度：O(n log n)
    空间复杂度：O(1) - 原地排序！
    """
    n = len(arr)

    # 第一步：构建最大堆
    build_max_heap(arr)
    print(f"构建最大堆后: {arr}")

    # 第二步：反复提取最大值
    for i in range(n - 1, 0, -1):
        # 把堆顶（最大值）和当前末尾交换
        arr[0], arr[i] = arr[i], arr[0]
        # 缩小堆的范围，对新的堆顶下沉
        sift_down(arr, i, 0)
        print(f"  提取最大值后: {arr}")

    return arr


# ========== 测试代码 ==========
if __name__ == "__main__":
    print("=== 堆排序算法演示 ===\n")

    # 测试：一般数组
    print("--- 测试：[12, 11, 13, 5, 6, 7] ---")
    arr = [12, 11, 13, 5, 6, 7]
    print(f"原始数组: {arr}")
    heap_sort(arr)
    print(f"排序结果: {arr}")

# 输出:
# === 堆排序算法演示 ===
#
# --- 测试：[12, 11, 13, 5, 6, 7] ---
# 原始数组: [12, 11, 13, 5, 6, 7]
# 构建最大堆后: [13, 11, 12, 5, 6, 7]
#   提取最大值后: [12, 11, 7, 5, 6, 13]
#   提取最大值后: [11, 6, 7, 5, 12, 13]
#   提取最大值后: [7, 6, 5, 11, 12, 13]
#   提取最大值后: [6, 5, 7, 11, 12, 13]
#   提取最大值后: [5, 6, 7, 11, 12, 13]
# 排序结果: [5, 6, 7, 11, 12, 13]
```

**堆排序的精妙之处**：

1. **原地排序**：不需要额外空间，直接在原数组上操作
2. **稳定的O(n log n)**：不像快速排序在最坏情况下退化到O(n²)，堆排序不管输入怎样都是O(n log n)
3. **建堆只要O(n)**：这听起来反直觉——n个元素一个一个插入要O(n log n)，但自底向上建堆只要O(n)！

### 示例3：Top K问题

这是面试中的高频题目——从海量数据中找出最大（或最小）的K个数：

```python
# 示例3：Top K问题（用最小堆解决）
# 经典问题：从海量数据中找出最大的K个数

import heapq


def top_k_largest(arr, k):
    """
    方法一：使用最小堆找出最大的K个数
    核心思路：维护一个大小为K的最小堆
    - 当堆未满时，直接入堆
    - 当堆满时，新元素比堆顶大才替换堆顶
    - 最终堆里的元素就是最大的K个
    时间复杂度：O(n log k)，空间复杂度：O(k)
    适合：n很大，k很小的场景
    """
    # Python的heapq是最小堆
    min_heap = []

    for num in arr:
        if len(min_heap) < k:
            # 堆没满，直接推入
            heapq.heappush(min_heap, num)
        elif num > min_heap[0]:
            # 新元素比堆顶（最小值）大，替换
            heapq.heapreplace(min_heap, num)

    return sorted(min_heap, reverse=True)  # 从大到小返回


# ========== 测试代码 ==========
if __name__ == "__main__":
    print("=== Top K问题演示 ===\n")

    # 测试数据：一个成绩列表
    scores = [85, 92, 78, 95, 88, 76, 90, 93, 82, 87]

    print("--- 场景：找出前3名最高分 ---")
    print(f"所有成绩: {scores}")

    result = top_k_largest(scores, 3)
    print(f"前3名最高分: {result}")

    print()
    print("--- 大规模数据测试：100万个数中找Top 10 ---")
    import random
    import time

    large_data = [random.randint(1, 1000000) for _ in range(1000000)]

    # 堆方法
    start = time.time()
    result = top_k_largest(large_data, 10)
    heap_time = time.time() - start
    print(f"堆方法耗时: {heap_time:.4f}秒")

    # 全排序方法
    start = time.time()
    result = sorted(large_data, reverse=True)[:10]
    sort_time = time.time() - start
    print(f"全排序方法耗时: {sort_time:.4f}秒")

    print(f"\n堆方法比全排序快约 {sort_time/heap_time:.1f} 倍！")

# 输出:
# === Top K问题演示 ===
#
# --- 场景：找出前3名最高分 ---
# 所有成绩: [85, 92, 78, 95, 88, 76, 90, 93, 82, 87]
# 前3名最高分: [95, 93, 92]
#
# --- 大规模数据测试：100万个数中找Top 10 ---
# 堆方法耗时: 0.0500秒
# 全排序方法耗时: 0.5000秒
#
# 堆方法比全排序快约 10.0 倍！
```

**为什么最小堆能找最大的K个？** 这听起来有点反直觉。想象一下：你有一个能装K个球的篮子，你只关心篮子里最大的那些球。每次来个新球，如果它比篮子里最小的球还大，就把最小的踢出去。最后篮子里留下的自然就是最大的K个。而**最小堆的堆顶就是最小的那个**，完美匹配这个需求！

## 深入理解

### 堆的数组表示：为什么不需要指针？

还记得二叉树通常怎么表示吗？每个节点有left和right指针。但堆不一样——因为堆是**完全二叉树**，节点之间有规律的索引关系：

```
数组: [30, 20, 25, 10, 15, 5]

对应的树结构：
         30 (索引0)
        /      \
    20(索引1)  25(索引2)
    /   \       /
10(3) 15(4)  5(5)
```

核心公式（从0开始索引）：
- **父节点**：`(i - 1) // 2`
- **左子节点**：`2 * i + 1`
- **右子节点**：`2 * i + 2`

这个公式就像数学公式一样精确，不需要额外的指针空间。这就是堆用数组实现比链表树更节省内存的原因。

### 上浮和下沉的可视化

上浮和下沉是堆操作的"双引擎"，理解它们就理解了堆的全部：

**上浮（Sift-Up）**：新元素加入后，从下往上看
```
        30               30               30
       /  \             /  \             /  \
     20    25    →    20    35    →    35    25
    /  \   /         /  \   /         /  \   /
   10  15  5        10  15  5        10  15  5

插入35后上浮：35>25，交换  35>30，交换  到顶了，结束
```

**下沉（Sift-Down）**：根节点被替换后，从上往下看
```
        5                5                25
       /  \             /  \             /  \
     20    25    →    30    25    →    30    20
    /  \   /         /  \   /         /  \   /
   10  15  35       10  15  35       10  15  35

根换成5后下沉：5<35（右更大），交换  5<25（右更大），交换  到叶子了，结束
```

### 时间复杂度分析

| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| 查找最大/最小值 | O(1) | 直接看堆顶 |
| 插入元素 | O(log n) | 上浮最多走树的高度 |
| 删除最大/最小值 | O(log n) | 下沉最多走树的高度 |
| 建堆（Heapify） | O(n) | 自底向上，底层节点下沉很快 |
| 堆排序 | O(n log n) | n次删除，每次O(log n) |

**为什么建堆是O(n)而不是O(n log n)？**

这是堆最反直觉的结论之一。直觉上，n个元素每个都要O(log n)，总的是O(n log n)。但自底向上建堆时：

- 一半节点（叶子）根本不需要下沉
- 四分之一的节点最多下沉1层
- 八分之一的节点最多下沉2层
- ...

把所有层加起来，数学上可以证明总操作次数是O(n)。这个结论在面试中经常被问到，记住它！

### 堆 vs 二叉搜索树 vs 数组

| 特性 | 堆 | 二叉搜索树 | 有序数组 |
|------|-----|-----------|---------|
| 查找最值 | O(1) | O(log n) | O(1) |
| 插入 | O(log n) | O(log n) | O(n) |
| 删除最值 | O(log n) | O(log n) | O(n) |
| 查找任意元素 | O(n) | O(log n) | O(log n) |
| 遍历有序 | O(n log n) | O(n) | O(1) |

**结论**：如果你只需要频繁获取最值和插入元素，堆是最优选择。但如果需要高效查找任意元素，二叉搜索树更合适。堆不擅长"查找中间的值"。

## 实际应用

### 应用场景1：操作系统任务调度

你的电脑同时运行着浏览器、微信、音乐播放器、IDE……操作系统是怎么安排CPU时间片的？答案就是**优先级队列（堆）**！

```
任务队列（按优先级）：
🔴 键盘输入响应 (优先级: 100)  ← 最高，不能卡顿
🟡 视频播放 (优先级: 80)      ← 高，保证流畅
🟢 文件下载 (优先级: 50)      ← 中，后台运行
🔵 系统更新 (优先级: 10)      ← 低，空闲时执行
```

操作系统用堆来管理这些任务，每次从堆顶取出优先级最高的任务执行。这样就能保证你的鼠标点击永远比后台下载更先得到响应。

### 应用场景2：Top K 热搜词

想象你是微博的后端工程师，每秒钟有上万条热搜数据涌入。产品经理问你："能实时显示Top 10热搜吗？"

**方案对比**：

- **全排序方案**：把所有热搜按热度排序，取前10。每秒处理上万条数据？CPU会冒烟的。
- **堆方案**：维护一个大小为10的最小堆，每条新数据来的时候和堆顶比一比。每秒轻松处理，CPU在摸鱼。

这就是为什么大厂都在用堆——不是因为程序员喜欢炫技，而是因为在大规模数据面前，O(n log k)就是比O(n log n)优雅太多。

### 应用场景3：Dijkstra最短路径算法

图论中的经典算法Dijkstra也用到了堆（优先队列）。每次从堆中取出距离最短的节点进行扩展，用堆来优化可以将时间复杂度从O(V²)降到O((V+E) log V)。这个我们在后续的算法课程中会详细讲解。

## 常见误区

⚠️ **注意这些坑！**

### 误区1：堆就是二叉搜索树

这是新手最常犯的错误。堆和二叉搜索树完全不同：

- **堆**：只保证父子关系（父>=子 或 父<=子），不保证左右子节点之间的大小关系
- **二叉搜索树**：左子树所有节点 < 根 < 右子树所有节点

```
❌ 错误想法：堆的中序遍历是有序的
✅ 事实：堆的中序遍历没有任何意义，只有删除堆顶才能有序输出
```

### 误区2：堆的数组中元素是有序的

很多新手看到堆用数组表示，以为数组本身就是有序的。大错特错！

```python
heap = [30, 20, 25, 10, 15, 5]
# 这是一个合法的最大堆
# 但数组本身并不是有序的！25 > 20 但25在20后面
```

堆的数组只保证父子索引对应关系，不保证全局有序。堆只保证一个方向的关系——从上到下递减（最大堆）。

### 误区3：Top K问题用最大堆

找最大的K个数，直觉告诉你应该用最大堆？错了！

- **找最大的K个数**：用**最小堆**，堆顶是最小的，新元素比堆顶大就替换
- **找最小的K个数**：用**最大堆**，堆顶是最大的，新元素比堆顶小就替换

记住这个口诀："**找大用小，找小用大**"——找最大的K个数用最小堆，找最小的K个数用最大堆。

### 练习1：实现最小堆 ⭐

在示例1的MaxHeap基础上，修改代码实现一个最小堆（MinHeap）。要求：

- 支持 insert 和 extract_min 操作
- 验证 extract_min 返回的是从小到大排序的序列

**要求**：
- 只需要把比较符号反过来（> 改成 <）
- 变量名 larger 改成 smaller

<details>
<summary>💡 提示（点击展开）</summary>

其实就是把 MaxHeap 中的比较符号全部反过来：
- sift_up: `self.heap[i] < self.heap[self.parent(i)]` 时上浮
- sift_down: 找较小的子节点，`self.heap[i] > self.heap[smaller]` 时停止下沉
- extract_max 改名为 extract_min

```python
class MinHeap:
    def __init__(self):
        self.heap = []

    def sift_up(self, i):
        while self.has_parent(i) and self.heap[i] < self.heap[self.parent(i)]:
            self.swap(i, self.parent(i))
            i = self.parent(i)

    def sift_down(self, i):
        while self.has_left_child(i):
            smaller = self.left_child(i)
            if self.has_right_child(i) and self.heap[self.right_child(i)] < self.heap[smaller]:
                smaller = self.right_child(i)
            if self.heap[i] < self.heap[smaller]:
                break
            self.swap(i, smaller)
            i = smaller

    # insert 和 extract_min 的逻辑和最大堆类似
```

把两个数组的所有元素都放入最小堆，然后逐个弹出：

```python
import heapq

def merge_arrays(arr1, arr2):
    # 把两个数组的所有元素放入堆
    heap = arr1 + arr2
    heapq.heapify(heap)  # O(n) 建堆

    # 依次弹出最小元素
    result = []
    while heap:
        result.append(heapq.heappop(heap))
    return result

print(merge_arrays([1, 3, 5], [2, 4, 6]))
# 输出: [1, 2, 3, 4, 5, 6]
```

### 练习3：实现一个优先级队列 ⭐⭐⭐

设计一个优先级队列类 PriorityQueue，支持以下操作：

**要求**：
- `push(item, priority)`: 插入一个元素及其优先级（数字越大优先级越高）
- `pop()`: 取出优先级最高的元素
- `peek()`: 查看优先级最高的元素
- 内部使用 heap 实现

```python
pq = PriorityQueue()
pq.push("买牛奶", 3)
pq.push("救火", 10)
pq.push("写作业", 1)
print(pq.pop())  # 应该输出: "救火"
```

<details>
<summary>💡 提示（点击展开）</summary>

```python
import heapq


class PriorityQueue:
    """
    优先级队列实现
    优先级用数字表示，数字越大优先级越高
    """

    def __init__(self):
        # Python的heapq是最小堆，用负优先级来模拟最大堆
        self._queue = []
        self._index = 0  # 用于处理优先级相同的情况

    def push(self, item, priority):
        """
        插入元素
        存储格式: (-优先级, 索引, 元素)
        - 负优先级: 让heapq这个最小堆表现为最大堆
        - 索引: 保证相同优先级时按插入顺序处理
        """
        heapq.heappush(self._queue, (-priority, self._index, item))
        self._index += 1

    def pop(self):
        """取出优先级最高的元素"""
        if not self._queue:
            raise IndexError("优先级队列为空")
        return heapq.heappop(self._queue)[2]  # 返回元素部分

    def peek(self):
        """查看优先级最高的元素"""
        if not self._queue:
            return None
        return self._queue[0][2]

    def is_empty(self):
        return len(self._queue) == 0

    def size(self):
        return len(self._queue)


# 测试
if __name__ == "__main__":
    pq = PriorityQueue()
    pq.push("买牛奶", 3)
    pq.push("救火", 10)
    pq.push("写作业", 1)
    pq.push("回邮件", 5)

    print("--- 按优先级出队 ---")
    while not pq.is_empty():
        item = pq.pop()
        print(f"处理: {item}")

# 输出:
# --- 按优先级出队 ---
# 处理: 救火
# 处理: 回邮件
# 处理: 买牛奶
# 处理: 写作业
```

## 小结与思考

### 本节要点

1. **堆是一种完全二叉树**，满足堆序性（最大堆：父>=子；最小堆：父<=子）
2. **堆用数组表示**，通过索引公式找到父子节点，不需要指针
3. **插入和删除都是O(log n)**，通过上浮和下沉操作维护堆性质
4. **堆排序是O(n log n)的原地排序**，建堆只要O(n)
5. **Top K问题用堆解决**：找最大K个用最小堆，找最小K个用最大堆
6. **Python内置heapq模块**提供了最小堆的实现

### 思考题

1. 如果我们有一个大小为n的堆，执行n次extract_max操作后得到的是一个有序数组吗？这说明堆和排序之间有什么关系？

2. 为什么堆排序不是稳定排序（相同元素的相对顺序可能改变）？什么样的排序算法是稳定的？

3. 在Top K问题中，如果K接近n（比如n=100万，K=99万），用堆还高效吗？有没有更好的方案？
