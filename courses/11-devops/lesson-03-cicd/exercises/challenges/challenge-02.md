# 挑战 02：设计多环境部署策略

## 难度：⭐⭐⭐⭐

## 背景
你的公司有一个关键的 Web 应用，需要支持三种不同的部署环境：
- **开发环境 (development)**：用于日常开发和测试
- **预发布环境 (staging)**：用于集成测试和用户验收
- **生产环境 (production)**：面向真实用户

每个环境有不同的要求和风险承受能力。

## 任务
基于 `code/examples/example-02-deployment-strategies.py`，扩展实现一个 **智能部署策略选择器**，能够根据以下条件自动选择最合适的部署策略：

1. **开发环境**：使用直接部署（简单快速）
2. **预发布环境**：使用蓝绿部署（确保稳定性）
3. **生产环境**：使用金丝雀发布（最小化风险）

## 具体要求

### 功能要求
- 创建一个 `DeploymentManager` 类，接收环境名称作为参数
- 实现 `select_strategy()` 方法，根据环境返回相应的部署策略对象
- 实现 `deploy()` 方法，执行完整的部署流程
- 添加环境特定的配置（如流量百分比、测试时长等）

### 技术要求
- 使用面向对象设计，保持代码的可扩展性
- 支持添加新的环境类型而不需要修改现有代码（开闭原则）
- 包含详细的错误处理和日志记录
- 所有代码必须包含中文注释

### 验证要求
编写测试代码验证以下场景：
1. 开发环境部署（应该快速完成）
2. 预发布环境部署（应该模拟蓝绿切换）
3. 生产环境部署（应该模拟渐进式金丝雀发布）
4. 无效环境名称（应该抛出适当的异常）

## 扩展挑战（可选）
- 添加配置文件支持（JSON 或 YAML），允许外部定义环境策略
- 实现部署回滚功能，当新版本出现问题时自动回退到旧版本
- 添加部署状态监控，实时报告部署进度和健康状况

## 提示
参考 `code/solutions/solution-02.py` 中的策略模式实现，但需要扩展为支持环境感知的部署管理器。
