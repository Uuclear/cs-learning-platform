# 挑战 01：实现自定义 CI/CD 流水线阶段

## 难度：⭐⭐⭐

## 背景
你正在为一个 Python Web 应用开发 CI/CD 流水线。现有的流水线包含基本的 lint、test、build 和 deploy 阶段，但缺少安全扫描阶段。

## 任务
在 `code/examples/example-01-pipeline-stages.py` 的基础上，添加一个 **安全扫描** 阶段，该阶段应该：

1. 在测试阶段之后、构建阶段之前执行
2. 模拟运行安全扫描工具（如 bandit、safety 等）
3. 如果发现高危漏洞，则停止整个流水线
4. 包含适当的中文注释说明安全扫描的重要性

## 要求
- 使用标准库，不要引入外部依赖
- 保持原有的代码风格和结构
- 安全扫描阶段应该有 80% 的成功率（模拟真实情况）
- 添加详细的中文注释解释每个步骤

## 验证
运行修改后的脚本，确保它能正确处理以下场景：
- 安全扫描通过：流水线继续执行到部署
- 安全扫描失败：流水线在安全扫描阶段停止

## 提示
参考 `code/solutions/solution-01.py` 中的 PipelineStage 类设计模式，但你需要直接修改示例文件而不是创建新类。
