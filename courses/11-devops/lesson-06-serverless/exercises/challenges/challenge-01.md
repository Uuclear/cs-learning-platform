# 挑战 1：设计 Serverless 图片处理服务

## 背景
你需要为一个电商平台设计一个 Serverless 图片处理服务。当用户上传产品图片到 S3 存储桶时，系统需要自动：
1. 验证图片格式和大小
2. 生成不同尺寸的缩略图（大、中、小）
3. 应用水印
4. 将处理后的图片存储到另一个 S3 存储桶

## 要求
1. 使用事件驱动架构，S3 上传事件触发 Lambda 函数
2. 实现完整的错误处理和日志记录
3. 考虑冷启动对用户体验的影响
4. 设计合理的目录结构和文件命名规范

## 思考问题
- 如何处理大图片文件的内存限制？
- 如何确保图片处理的原子性和一致性？
- 如果图片处理失败，如何实现重试机制？
- 如何监控和度量服务的性能？

## 扩展挑战
- 添加 CDN 缓存失效逻辑
- 实现基于标签的图片分类功能
- 添加图片质量检测和自动优化
