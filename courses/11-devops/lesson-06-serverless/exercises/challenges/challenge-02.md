# 挑战 2：构建 Serverless 实时聊天应用

## 背景
你需要构建一个实时聊天应用的后端，支持：
- 用户注册和身份验证
- 一对一私聊和群组聊天
- 消息历史记录存储
- 在线状态检测
- 推送通知

## 要求
1. 使用 WebSocket API Gateway + Lambda 构建实时通信
2. 使用 DynamoDB 存储用户信息和消息历史
3. 实现基于 JWT 的身份验证
4. 设计事件驱动的消息广播机制

## 架构设计要点
- 连接管理：如何处理 WebSocket 连接的建立、保持和断开
- 消息路由：如何将消息正确路由到目标用户或群组
- 状态同步：如何维护用户的在线/离线状态
- 扩展性：如何支持大量并发连接

## 性能考虑
- WebSocket 连接的内存和连接数限制
- 消息广播的延迟优化
- 数据库读写性能和成本控制
- 冷启动对新用户连接体验的影响

## 安全要求
- 防止未授权访问
- 消息内容的安全传输
- 防止 DDoS 攻击
- 敏感数据的加密存储

## 扩展挑战
- 添加文件上传和分享功能
- 实现消息撤回和编辑功能
- 添加端到端加密
- 集成第三方通知服务（邮件、短信）
