# 挑战 2：实现简单的 Pod 调度模拟器

## 难度：⭐⭐⭐⭐⭐

### 背景
Kubernetes 调度器是集群的核心组件，负责将 Pods 分配到合适的节点上。调度过程包含两个主要阶段：过滤（Filtering）和评分（Scoring）。你的任务是实现一个简化的调度模拟器，理解调度算法的工作原理。

### 要求

#### 1. 基础功能
- 实现 `Node` 类，包含 CPU、内存容量和已分配资源
- 实现 `Pod` 类，包含资源请求和节点选择器（nodeSelector）
- 实现基本的过滤逻辑：检查节点是否有足够资源容纳 Pod
- 实现基本的评分逻辑：基于资源利用率选择最佳节点

#### 2. 高级功能
- 支持多种调度策略：
  - **LeastRequested**: 选择资源请求最少的节点（负载最轻）
  - **MostAllocated**: 选择资源分配最多的节点（提高资源利用率）
  - **BalancedAllocation**: 平衡 CPU 和内存的分配比例
- 实现亲和性（Affinity）和反亲和性（Anti-Affinity）支持
- 支持污点（Taints）和容忍（Tolerations）
- 实现抢占（Preemption）逻辑：当没有合适节点时，驱逐低优先级 Pod

#### 3. 扩展功能
- 添加可视化界面显示调度结果
- 支持批量调度和并发调度
- 实现自定义调度插件接口
- 添加性能监控和调度延迟统计

### 输入/输出格式

#### 输入示例
```python
# 节点定义
nodes = [
    {"name": "node-1", "cpu": 4000, "memory": 8192, "labels": {"zone": "a"}},
    {"name": "node-2", "cpu": 8000, "memory": 16384, "labels": {"zone": "b"}}
]

# Pod 定义
pods = [
    {"name": "web-1", "cpu": 500, "memory": 512, "nodeSelector": {"zone": "a"}},
    {"name": "db-1", "cpu": 2000, "memory": 4096}
]
```

#### 输出示例
```json
{
  "scheduled": {
    "web-1": "node-1",
    "db-1": "node-2"
  },
  "unscheduled": [],
  "metrics": {
    "total_pods": 2,
    "scheduled_pods": 2,
    "average_scheduling_time_ms": 15.2
  }
}
```

### 测试用例
你需要处理以下场景：
1. **资源充足**: 所有 Pods 都能成功调度
2. **资源不足**: 部分 Pods 无法调度
3. **节点选择器约束**: Pod 只能在特定标签的节点上运行
4. **资源碎片**: 虽然总资源足够，但没有单个节点能容纳大 Pod
5. **抢占场景**: 高优先级 Pod 需要驱逐低优先级 Pod
