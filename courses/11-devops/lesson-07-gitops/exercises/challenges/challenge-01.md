# 挑战 1：实现简单的 GitOps 协调器

## 背景
你被要求实现一个简化的 GitOps 协调器，它能够监控 Git 仓库中的配置变更并自动应用到目标环境。

## 要求
1. 创建一个 `GitOpsReconciler` 类，具有以下方法：
   - `__init__(self, git_repo_path: str, target_env_path: str)`：初始化协调器
   - `detect_changes(self) -> bool`：检测 Git 仓库是否有变更
   - `reconcile(self)`：执行协调操作，同步目标环境到期望状态

2. 协调器应该能够处理以下场景：
   - 新服务的创建
   - 现有服务的更新（镜像版本、副本数等）
   - 服务的删除

3. 使用 JSON 文件存储状态：
   - Git 仓库中的 `desired.json` 存储期望状态
   - 目标环境中的 `actual.json` 存储实际状态

## 提示
- 期望状态和实际状态都包含服务列表，每个服务有 `name`、`image`、`replicas` 字段
- 协调过程需要比较两个状态的差异并执行相应的操作
- 不需要实际部署服务，只需更新 `actual.json` 文件即可

## 验证
运行你的实现并验证以下场景：
1. 初始状态为空，添加新服务
2. 更新现有服务的镜像版本
3. 删除不再需要的服务
4. 同时进行多个变更

## 扩展思考
- 如何处理协调过程中的错误？
- 如何实现幂等性（多次执行产生相同结果）？
- 如何添加健康检查机制？
