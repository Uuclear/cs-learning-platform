# 挑战 2：设计多环境 GitOps 策略

## 背景
你的团队正在管理三个环境：开发（development）、预发布（staging）和生产（production）。每个环境都有不同的配置需求和部署策略。

## 要求
1. 设计一个 GitOps 目录结构来管理多环境配置
2. 为每个环境定义不同的部署策略：
   - 开发环境：自动同步，允许快速迭代
   - 预发布环境：手动同步，需要代码审查
   - 生产环境：手动同步，需要多级审批

3. 实现环境特定的配置覆盖机制：
   - 共享的基础配置
   - 环境特定的覆盖配置
   - 敏感数据的安全处理方案

## 具体任务
### 任务 1：目录结构设计
创建以下目录结构并说明每个部分的作用：
```
gitops-repo/
├── base/                  # 基础配置
├── environments/
│   ├── development/       # 开发环境配置
│   ├── staging/          # 预发布环境配置  
│   └── production/       # 生产环境配置
└── secrets/              # 加密的敏感数据
```

### 任务 2：部署策略配置
为每个环境创建部署策略配置文件，包含：
- 同步模式（自动/手动）
- 健康检查配置
- 回滚策略
- 通知设置

### 任务 3：配置合并逻辑
实现一个配置合并函数，能够：
- 加载基础配置
- 应用环境特定的覆盖
- 处理敏感数据（使用占位符或加密引用）

## 验证
1. 验证配置合并逻辑是否正确处理覆盖
2. 验证不同环境的部署策略是否符合要求
3. 验证敏感数据是否得到适当保护

## 扩展思考
- 如何处理跨环境的依赖关系？
- 如何实现蓝绿部署或金丝雀发布？
- 如何与现有的 CI/CD 流水线集成？
- 如何监控和告警 GitOps 状态？
