# 挑战 2: Terraform 状态管理和漂移修复自动化

## 难度：⭐⭐⭐⭐

## 背景
在大型组织中，基础设施经常被手动修改（违反 IaC 原则），导致 Terraform 状态与实际基础设施出现漂移。你需要设计一个自动化解决方案来检测和修复这些漂移。

## 要求
创建一个 Python 脚本，模拟 Terraform 的状态管理和漂移修复过程：

1. **状态跟踪系统**：实现一个类来管理当前状态和期望状态
2. **漂移检测算法**：能够识别三种类型的漂移：
   - 资源缺失（配置中有但实际没有）
   - 资源意外存在（实际有但配置中没有）
   - 属性漂移（资源存在但属性不匹配）
3. **自动修复策略**：
   - 对于缺失资源：生成创建计划
   - 对于意外资源：提供删除或导入选项
   - 对于属性漂移：生成更新计划
4. **报告生成**：输出详细的漂移报告，包括影响分析
5. **安全机制**：在执行任何破坏性操作前要求确认

## 验证标准
- [ ] 能够正确检测所有三种漂移类型
- [ ] 提供清晰的修复建议和影响分析
- [ ] 实现安全确认机制防止意外删除
- [ ] 支持批量处理多个资源
- [ ] 代码结构清晰，易于扩展到真实云提供商

## 示例场景
```python
# 模拟当前基础设施状态
current_state = {
    "vpc-main": {"type": "aws_vpc", "cidr": "10.0.0.0/16", "tags": {"env": "prod"}},
    "ec2-web": {"type": "aws_instance", "instance_type": "t2.micro"}  # 手动降级了实例类型
}

# 期望的配置状态
desired_state = {
    "vpc-main": {"type": "aws_vpc", "cidr": "10.0.0.0/16", "tags": {"env": "production"}},
    "ec2-web": {"type": "aws_instance", "instance_type": "t3.medium"},
    "rds-db": {"type": "aws_db_instance", "engine": "postgres"}  # 缺失的数据库
}
```
