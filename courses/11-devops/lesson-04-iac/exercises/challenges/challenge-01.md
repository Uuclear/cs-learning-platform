# 挑战 1: 多环境基础设施部署

## 难度：⭐⭐⭐

## 背景
你的团队需要为三个不同环境（开发、测试、生产）部署相似但配置不同的 AWS 基础设施。每个环境都需要：
- VPC 和子网
- 安全组
- EC2 实例
- RDS 数据库实例

## 要求
使用 Terraform 创建一个模块化的解决方案，满足以下条件：

1. **创建可重用的 VPC 模块**：接受 CIDR 块、区域和环境名称作为参数
2. **创建 Web 应用模块**：包含 EC2 实例和安全组，根据环境调整实例类型
3. **使用工作区（Workspaces）或变量文件**：为每个环境提供不同的配置
4. **实现适当的标签策略**：所有资源都应有 Environment 标签
5. **状态管理**：为每个环境使用独立的远程状态存储

## 验证标准
- [ ] 模块可以被不同环境重用而无需修改代码
- [ ] 开发环境使用 t2.micro 实例，生产环境使用 m5.large 实例
- [ ] 所有资源都有正确的 Environment 标签
- [ ] 可以独立地对每个环境执行 terraform plan 和 apply
- [ ] 状态文件正确分离，避免环境间冲突

## 提示
- 考虑使用 `terraform workspace` 或单独的目录结构
- 利用变量和输出来实现模块间的依赖传递
- 生产环境应该有更严格的安全组规则
