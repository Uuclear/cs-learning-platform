数组：排排坐，吃果果

🎵 "排排坐，吃果果，你一个，我一个..."

小时候排队分水果的场景，其实和计算机里的数组一模一样！

开场白

想象一下这个场景：

你走进一家网红奶茶店，发现队伍排得老长。有趣的是，每个人都站在地上的方格里，方格上写着编号：0号、1号、2号...就像这样：

[小明] [小红] [小刚] [小丽] [小强]
  0      1      2      3      4

这时候你想找排在第3位的小丽，直接看地上的数字"3"就行了，不用从头数。但如果有人插队或者离开，后面的人就得整体移动——想想就累！

这就是数组的本质：连续排队，编号定位。

💡 思考问题: 为什么数组的编号从0开始，而不是1呢？（提示：这和内存地址有关哦！）

核心概念

什么是数组？

数组（Array）是计算机中最基础的数据结构，简单来说就是一组相同类型的数据，在内存中连续排列。

用生活化的比喻：

  • 🏢 就像一栋公寓楼：每个房间大小一样，门牌号连续（101、102、103...）
  • 🅿️ 就像停车场：每个车位连续排列，编号清晰
  • 📚 就像图书馆书架：同一类书连续摆放，按编号查找

数组的三大特征

  1. 连续存储：所有元素在内存中紧紧挨着，中间不插空
  2. 相同类型：就像停车场只停轿车，不会突然冒出个自行车
  3. 索引访问：通过编号（索引）直接找到元素，就像按门牌号找人

数组的基本操作

1. 访问元素 - O(1) ⚡

数组的超能力：随机访问。只要知道索引，瞬间就能找到元素。

数组: [10, 20, 30, 40, 50]
索引:  0   1   2   3   4

访问索引2的元素 → 直接拿到30！

为什么这么快？因为计算机有个公式：

元素地址 = 起始地址 + 索引 × 元素大小

就像你知道公寓楼从101开始，那105号房就是：101 + 4 = 105，不用挨家挨户敲门问。

2. 修改元素 - O(1) ⚡

找到位置后直接替换，也是瞬间完成。

修改前: [10, 20, 30, 40, 50]
把索引2改成99
修改后: [10, 20, 99, 40, 50]

3. 插入元素 - O(n) 🐌

这是最麻烦的操作！想象一下排队买奶茶，你想插队到第2个位置：

插入前: [小明] [小红] [小刚] [小丽]
           0      1      2      3

小红、小刚、小丽都要往后挪一位！

插入后: [小明] [你] [小红] [小刚] [小丽]
           0     1     2      3      4

最坏情况下（插在开头），所有人都要移动，所以是O(n)。

4. 删除元素 - O(n) 🐌

和插入类似，删除后后面的人要往前补位。

删除前: [小明] [小红] [小刚] [小丽]
删掉小红

小刚、小丽都要往前挪！

删除后: [小明] [小刚] [小丽] [空]

数组的优缺点

特性 优点 缺点
访问速度 ⚡ 超快！O(1) -
修改速度 ⚡ 超快！O(1) -
插入删除 - 🐌 慢！O(n)
内存占用 紧凑，无额外开销 大小固定（静态数组）
缓存友好 👍 连续存储，CPU缓存命中率高 -

动态数组（Dynamic Array）

静态数组最大的痛点：大小固定。就像你租了个10平米的单间，结果东西越来越多...

动态数组就是来解决这个问题的！它的核心思想：不够用了就搬家到更大的地方。

扩容机制：

  1. 当数组满了，申请一块更大的内存（通常是原来的2倍）
  2. 把所有元素搬运过去
  3. 释放原来的内存
  4. 继续愉快地添加元素
原数组: [A, B, C, D]  （满了！）
        ↓ 扩容 ×2
新数组: [A, B, C, D, _, _, _, _]  （宽敞！）
                          ↑
                       可以继续添加

Python的list、Java的ArrayList、C++的vector，都是动态数组的实现。

示例2: 手动实现动态数组

# ============================================
# 示例2：手动实现动态数组
# 理解Python列表背后的扩容机制
# ============================================

class DynamicArray:
    """
    动态数组的实现
    核心思想：容量不够时，搬家到更大的地方
    """
    
    def __init__(self, initial_capacity=4):
        """
        初始化动态数组
        capacity: 当前容量（能装多少）
        size: 当前大小（装了多少）
        """
        self.capacity = initial_capacity
        self.size = 0
        self.data = [None] * self.capacity  # 预分配空间
        print(f"🎉 创建了容量为 {self.capacity} 的动态数组")
    
    def append(self, value):
        """
        在末尾添加元素
        如果满了，先扩容再添加
        """
        # 检查是否需要扩容
        if self.size >= self.capacity:
            self._resize()
        
        self.data[self.size] = value
        self.size += 1
        print(f"  添加 {value}，当前大小: {self.size}/{self.capacity}")
    
    def _resize(self):
        """
        扩容操作：容量翻倍
        这就像搬家到更大的房子
        """
        old_capacity = self.capacity
        self.capacity *= 2  # 容量翻倍
        
        # 创建新的大数组
        new_data = [None] * self.capacity
        
        # 把旧数据搬过去
        for i in range(self.size):
            new_data[i] = self.data[i]
        
        self.data = new_data
        print(f"  🚚 扩容！{old_capacity} → {self.capacity}")
    
    def get(self, index):
        """获取指定索引的元素"""
        if 0 <= index < self.size:
            return self.data[index]
        raise IndexError("索引越界啦！")
    
    def __str__(self):
        """打印数组内容"""
        return f"[{', '.join(str(self.data[i]) for i in range(self.size))}]"


# 测试动态数组
print("=== 测试动态数组 ===")
arr = DynamicArray(initial_capacity=4)

print("\n开始添加元素...")
for i in range(10):
    arr.append(i * 10)

print(f"\n最终数组: {arr}")
print(f"最终大小: {arr.size}")
print(f"最终容量: {arr.capacity}")

# 访问元素
print(f"\n访问索引3的元素: {arr.get(3)}")

# 场景2：实现一个简单的栈（Stack）
print("=== 场景2：用数组实现栈 ===")

class Stack:
    """
    栈：后进先出（LIFO）
    就像一叠盘子，最后放的先拿
    """
    def __init__(self):
        self.items = []  # 用数组存储
    
    def push(self, item):
        """入栈：放在最上面"""
        self.items.append(item)
        print(f"  入栈: {item}，当前栈: {self.items}")
    
    def pop(self):
        """出栈：拿走最上面的"""
        if not self.items:
            return None
        item = self.items.pop()
        print(f"  出栈: {item}，当前栈: {self.items}")
        return item

# 测试栈

深入理解

为什么数组访问是 O(1)？

这是数组最强大的特性！秘密在于内存的连续性。

想象内存是一条长长的街道，数组就像一排连续的房子：

内存地址:  1000   1004   1008   1012   1016
          ┌─────┬─────┬─────┬─────┬─────┐
          │  10 │  20 │  30 │  40 │  50 │
          └─────┴─────┴─────┴─────┴─────┘
            0     1     2     3     4
            ↑
          索引

如果每个整数占4字节，那么：

  • 索引0的元素在地址 1000 + 0×4 = 1000
  • 索引1的元素在地址 1000 + 1×4 = 1004
  • 索引2的元素在地址 1000 + 2×4 = 1008

公式：地址 = 基地址 + 索引 × 元素大小

计算机只需要一次简单的数学运算，就能直接算出元素位置，不需要逐个查找。这就是为什么数组访问是常数时间 O(1)。

为什么插入删除是 O(n)？

因为数组元素必须连续存储！

插入时，后面的元素都要后移：

插入前: [A] [B] [C] [D]
           ↑ 在B后面插入X
插入时: [A] [B] [ ] [C] [D]  ← C、D后移
插入后: [A] [B] [X] [C] [D]

删除时，后面的元素都要前移补位：

删除前: [A] [B] [C] [D]
           ↑ 删除B
删除时: [A] [ ] [C] [D]  ← 出现空洞
删除后: [A] [C] [D] [ ]  ← C、D前移

最坏情况下（操作第一个元素），需要移动n个元素，所以是O(n)。

动态数组的摊还分析

动态数组扩容时，需要把所有元素搬到新地方，这个操作是O(n)。那为什么说append是**O(1)**呢？

这就是**摊还分析（Amortized Analysis）**的魔法：

容量4的数组，扩容到8：
  添加1: [1, _, _, _]      - 直接放，1步
  添加2: [1, 2, _, _]      - 直接放，1步
  添加3: [1, 2, 3, _]      - 直接放，1步
  添加4: [1, 2, 3, 4]      - 直接放，1步
  添加5: 扩容！搬4个元素 + 放新元素 = 5步
  添加6: [..., 6, _, _, _] - 直接放，1步
  添加7: [..., 6, 7, _, _] - 直接放，1步
  添加8: [..., 6, 7, 8, _] - 直接放，1步
  添加9: 扩容！搬8个元素 + 放新元素 = 9步

总共添加9个元素，操作次数：1+1+1+1+5+1+1+1+1+9 = 17

平均每次添加：17/9 ≈ 2次操作，接近常数时间！

数组 vs 链表

特性 数组 链表
访问元素 ⚡ O(1) 🐌 O(n)
插入/删除 🐌 O(n) ⚡ O(1)（已知位置）
内存布局 连续 分散
缓存友好 👍 高 👎 低
大小灵活 需扩容 天然灵活

选择建议：

  • 需要频繁随机访问 → 选数组
  • 需要频繁插入删除 → 选链表
  • 关注CPU缓存性能 → 选数组（现代CPU loves连续内存！）

常见误区

⚠️ 注意这些坑！

误区1: 数组索引从0开始，但我以为是1

错误代码：

arr = [10, 20, 30]
print(arr[1])  # 以为是10？其实是20！

为什么会这样：

  • 索引0表示"偏移0个位置"，就是开头
  • 这是C语言的传统，被大多数语言继承
  • 计算公式：地址 = 基地址 + 索引 × 大小，索引为0时就是基地址本身

✅ 正确做法：

arr = [10, 20, 30]
print(arr[0])  # 10，第一个元素
print(arr[1])  # 20，第二个元素
print(arr[-1]) # 30，Python支持负数索引（最后一个）

误区2: 插入元素后，数组长度自动变？

错误理解：

# C语言的静态数组
int arr[5] = {1, 2, 3, 4, 5};
arr[5] = 6;  // ❌ 越界！数组大小固定是5

真相：

  • 静态数组（C/Java的基本数组）大小固定不变
  • 动态数组（Python list、Java ArrayList）会自动扩容
  • 但扩容是有代价的（需要搬家）

✅ 正确做法：

# Python - 动态数组，会自动处理
arr = [1, 2, 3, 4, 5]
arr.append(6)  # ✅ 没问题，自动扩容

# 如果你知道大概需要多大，可以预先分配
arr = [None] * 1000  # 预先分配1000个位置

误区3: 数组和列表是同一个东西

混淆点：Python的list、Java的ArrayList都叫"列表"，但底层是动态数组。

真正的链表（Linked List）是这样的：

数组: [A][B][C][D]  ← 连续存储，通过索引访问

链表: [A]→[B]→[C]→[D]  ← 分散存储，通过指针连接

✅ 正确理解：

  • Python list = 动态数组（支持O(1)随机访问）
  • 链表 = 真正的链表（插入删除O(1)，但访问O(n)）
  • Java: ArrayList是动态数组，LinkedList是链表

小结与思考

本节要点

  1. 数组的本质：连续内存存储，通过索引直接访问
  2. 时间复杂度：
    • 访问/修改：⚡ O(1) —— 数组的超能力！
    • 插入/删除：🐌 O(n) —— 需要移动元素
  3. 动态数组：自动扩容，摊还时间复杂度O(1)
  4. 适用场景：频繁随机访问、数据量相对固定
  5. 不适用场景：频繁插入删除、数据大小变化剧烈
