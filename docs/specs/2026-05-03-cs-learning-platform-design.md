## 1. 项目概述

### 1.1 愿景

打造一个**通俗易懂、幽默风趣、深入浅出**的CS知识学习平台。采用**教授人设**（知识储备巨大、语言通俗易懂、口吻幽默风趣），让复杂的计算机科学概念变得生动有趣。

### 1.2 目标受众

混合层级学习者：
- 编程零基础初学者（高中生/大一新生）
- 有一定基础的学生（大二/大三CS专业）
- 进阶学习者（准备面试/转行工程师）

### 1.3 规模

第一期：50节课，覆盖10个CS核心模块

## 2. 系统架构

### 2.1 Agent协作模式

参考Anthropic和OpenAI的Harness Engineering最佳实践：

```
┌─────────────────────────────────────────────────────────────┐
│                      主Agent (协调者)                        │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐     │
│  │ 项目初始化│→│ 课程规划 │→│ 任务分配 │→│ 质量审核 │     │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘     │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                      子Agent (执行者)                        │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐     │
│  │ 课件开发 │→│ 代码编写 │→│ 练习设计 │→│ 自测提交 │     │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘     │
└─────────────────────────────────────────────────────────────┘
```

## 3. 内容规范

### 3.1 教授人设要求

- **语言风格**: 通俗易懂，避免学术黑话
- **口吻**: 幽默风趣，适当使用比喻和类比
- **结构**: 循序渐进，从具体到抽象
- **互动**: 每节课包含"思考题"和"动手实践"

### 3.2 课件模板结构

1. **开场白** - 幽默引入，激发兴趣
2. **核心概念** - 深入浅出讲解
3. **代码演示** - 可运行的实例
4. **深入理解** - 原理剖析
5. **实际应用** - 真实场景
6. **常见误区** - 避坑指南
7. **动手实践** - 编程练习
8. **小结与思考** - 总结+思考题

### 3.3 技术栈

- **框架**: Next.js 14 (App Router)
- **语言**: TypeScript
- **样式**: Tailwind CSS + shadcn/ui
- **内容**: MDX (Markdown + JSX)
- **代码运行**: Sandpack (浏览器内运行代码)
- **图表**: Mermaid (流程图、架构图)
- **部署**: Vercel

## 5. Agent工作流程

### 5.1 初始化阶段（主Agent）

1. 创建项目结构
2. 编写AGENTS.md工作指南
3. 制定完整课程大纲
4. 创建课件模板

### 5.2 开发阶段（子Agent）

1. 读取AGENTS.md了解规范
2. 读取课程大纲明确目标
3. 开发单节课件（课件+代码+练习）
4. 自测验证
5. 提交PR

### 5.3 审核阶段（主Agent）

1. 审核内容质量
2. 检查风格一致性
3. 验证代码可运行
4. 合并或反馈修改
