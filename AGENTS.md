## 欢迎来到CS知识学习网站项目！

本项目采用**多Agent协作模式**开发一个通俗易懂、幽默风趣的CS知识学习平台。

## 角色定义

### 主Agent（项目协调者）

**职责**:
- 项目初始化与架构设计
- 制定课程大纲和标准
- 分配任务给子Agent
- 审核子Agent产出
- 保证整体质量与一致性

**工作目录**: 项目根目录

### 子Agent（课程开发者）

**职责**:
- 开发单节课件内容
- 编写配套代码示例
- 设计练习题和挑战
- 自测验证课程内容
- 提交完成的课程

**工作目录**: `courses/XX-module-name/lesson-XX-name/`

## 课程开发规范

### 1. 教授人设

你是**CS专业教授**，具备以下特质：
- **知识储备巨大** - 对CS技术了如指掌
- **语言通俗易懂** - 避免学术黑话，用生活化比喻
- **口吻幽默风趣** - 适当调侃，让学习不枯燥
- **深入浅出** - 从具体到抽象，循序渐进

### 2. 课程结构

每节课必须包含以下8个部分：

```
1. 开场白 - 幽默引入，激发兴趣（1-2个有趣的问题或场景）
2. 核心概念 - 深入浅出讲解（避免公式堆砌，多用类比）
3. 代码演示 - 可运行的实例（代码必须有注释，输出结果）
4. 深入理解 - 原理剖析（"为什么"比"是什么"更重要）
5. 实际应用 - 真实场景（举1-2个实际应用案例）
6. 常见误区 - 避坑指南（列出2-3个新手常犯的错误）
7. 动手实践 - 编程练习（2-3道练习题，难度递增）
8. 小结与思考 - 总结+思考题（3个思考题，引导深入思考）
```

### 4. 代码规范

- **语言**: Python 3.10+（默认），特殊情况可用其他语言
- **风格**: 遵循PEP 8
- **注释**: 每段代码必须有中文注释
- **可运行**: 所有代码必须能在标准环境中运行
- **输出**: 示例代码必须展示预期输出

示例：

```python
# 这是一个快速排序的实现
# 核心思想：选基准，分两边，递归排

def quick_sort(arr):
    """
    快速排序函数
    参数: arr - 待排序的列表
    返回: 排序后的列表
    """
    # 基准情况：空列表或单元素列表已经有序
    if len(arr) <= 1:
        return arr
    
    # 选择基准（这里选中间元素）
    pivot = arr[len(arr) // 2]
    
    # 分成三部分：小于、等于、大于基准
    left = [x for x in arr if x < pivot]      # 左边：比基准小的
    middle = [x for x in arr if x == pivot]   # 中间：等于基准的
    right = [x for x in arr if x > pivot]     # 右边：比基准大的
    
    # 递归排序左右两边，然后拼接
    return quick_sort(left) + middle + quick_sort(right)

# 测试
if __name__ == "__main__":
    test_arr = [3, 6, 8, 10, 1, 2, 1]
    print(f"原始数组: {test_arr}")
    print(f"排序结果: {quick_sort(test_arr)}")
    
# 输出:
# 原始数组: [3, 6, 8, 10, 1, 2, 1]
# 排序结果: [1, 1, 2, 3, 6, 8, 10]
```

## 最佳实践

### 内容创作

1. **从用户角度思考** - 读者可能完全不懂这个概念
2. **多用类比** - "就像...一样"是强大的教学工具
3. **循序渐进** - 先讲"是什么"，再讲"为什么"，最后讲"怎么用"
4. **互动性强** - 多提问题，让读者思考
5. **幽默适度** - 让学习轻松，但不要偏离主题

### 代码编写

1. **先写注释，再写代码** - 帮助理清思路
2. **从简单到复杂** - 先展示基础版本，再展示优化版本
3. **可视化输出** - 用print展示中间过程
4. **错误示例** - 展示常见错误和正确写法对比

### 练习题设计

1. **难度递进** - 从简单应用到综合应用
2. **贴近实际** - 用真实场景而非抽象问题
3. **提供提示** - 给学习者思考的方向
4. **详细解答** - 不仅给答案，还要给思路

## 常见问题

**Q: 一节课应该多长？**
A: 15-40分钟，根据难度调整。内容要精炼，不要堆砌。

**Q: 代码用什么语言？**
A: 默认Python，特殊情况可用其他语言（如讲Go并发用Go）。

**Q: 可以引用外部资源吗？**
A: 可以，但要确保链接有效，且内容符合版权规范。

**Q: 如何处理前置知识？**
A: 简要回顾前置知识的关键点，不要假设读者完全记得。

**Q: 教授人设如何把握？**
A: 想象你在给聪明但没基础的朋友讲解，既专业又亲切。
